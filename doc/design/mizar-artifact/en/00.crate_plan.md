# Crate Plan: mizar-artifact

> Canonical language: English. Japanese companion:
> [../ja/00.crate_plan.md](../ja/00.crate_plan.md).

## Responsibility

Owned behavior:

- stable published artifact projections for downstream packages, LSP,
  documentation, and AI tooling;
- artifact-facing schemas, including `ModuleSummary`, `RegistrationSummary`,
  `ProofWitnessRef`, and `VerifiedArtifact`;
- canonical serialization and artifact hash inputs shared by published schemas;
- artifact-store layout, atomic writes, validating readers, and manifest
  transactions;
- proof witness references by content hash and provenance fields needed to
  reproduce or inspect published artifacts.

Out of scope:

- raw `SurfaceAst`, typed AST, core IR, control-flow IR, VC IR, ATP problem, or
  kernel-state dumps;
- internal cache records, cache-key ownership, cluster-db storage, proof-reuse
  validation, or scheduler state;
- proof authority, proof search, kernel acceptance, and policy decisions;
- producer-owned projection logic inside resolver, checker, VC, kernel, or proof
  crates.

## Exit Criteria

Hard gates:

- no blocking/high specification inconsistency remains;
- no source behavior remains outside `doc/spec`, `.miz` tests, traceability
  metadata, or task-specific module specs;
- documented artifact/source responsibility boundaries are preserved;
- no raw IR, cache record, scheduler state, proof authority, or kernel
  acceptance behavior is owned by this crate;
- each completed task has its own commit;
- remaining gaps are classified as `deferred`, `external_dependency_gap`, or
  human-owned.

Verification commands:

- For every Rust source task:
  - `cargo test -p mizar-artifact`
  - `cargo clippy -p mizar-artifact --all-targets -- -D warnings`
  - `cargo fmt --check`
- For task 1's new workspace member:
  - `cargo check --workspace`
- For broad crate completion:
  - `cargo clippy --all-targets --all-features -- -D warnings`
  - `cargo test`

Review expectations:

- review-only agent for implementation specification and documentation;
- review-only agent for test sufficiency;
- review-only agent for full implementation;
- review-only agent for source/documentation consistency;
- final crate completion review with hard-gate status and quality score at
  least 90/100.
