## 目的

この文書は、trusted kernel が consume する final proof certificate format を定義する。

これは pipeline phase 13-14 を詳細化する。ATP backend、deterministic pre-ATP discharge、Mizar-side proof reconstruction は evidence を生成してよいが、kernel が受理するのは normalized かつ replayable な certificate だけである。

## 設計判断

### Certificate Top Level

certificate は canonical かつ versioned object である。

```text
Certificate
  version
  target_vc
  kernel_profile
  imported_axioms
  imported_theorems
  generated_clauses
  substitutions
  resolution_trace
  derived_facts
  final_goal
```

`target_vc` は certificate を 1 つの canonical verification condition に bind する。`kernel_profile` は accepted certificate schema、clause encoding、hash algorithm、有効な built-in checkers を記録する。

### Trust Scope

kernel が check するのは次だけである。

- certificate parsing と schema validation
- stable id と content hash による imported fact availability
- SAT clause well-formedness
- substitution、alpha-conversion、free-variable side conditions
- resolution trace replay
- checked clauses から final goal が導出されること

kernel は proof search、premise selection、overload resolution、cluster expansion、coercion insertion、global mutable compiler state 参照を行ってはならない。

### Clause Representation

clause serialization は canonical でなければならない。

- variables は normalization 後の canonical variable id で order する。
- literals は polarity、predicate/functor symbol id、arity、normalized argument encoding の順で order する。
- duplicate literals は certificate normalization 中に除去する。
- tautological clauses は reject するか、kernel profile に従って明示的な `tautology` marker で表す。
- stable hash は canonical byte encoding と schema version から計算する。

map/set iteration order が clause id や clause hash に影響してはならない。

### Resolution Trace

各 resolution step は次を記録する。

```text
ResolutionStep
  step_id
  parent_clause_a
  parent_clause_b
  pivot_literal
  generated_clause
```

kernel は両 parent を lookup し、pivot が反対 polarity で出現することを check し、parents を resolve し、結果を canonicalize して `generated_clause` と比較する。

parent reference は imported clauses、generated clauses、または earlier resolution steps だけを指す。forward reference は malformed certificate である。

### Substitution Rule

substitution は certificate 内の明示的 entry である。

```text
SubstitutionStep
  source_term
  target_term
  binder_context
  freshness_witnesses
  free_variable_constraints
```

kernel は [16.substitution_and_binding.md](./16.substitution_and_binding.md) で定義した通り、capture avoidance、alpha-conversion、free-variable constraints、binder normalization を検証する。

### Imported Facts

imported axioms と theorems は次で参照する。

- package id
- module path
- exported item id
- statement fingerprint
- policy が要求する accepted proof status

imported theorem が unavailable、fingerprint mismatch、または current kernel profile が許さない弱い policy でしか accepted されていない場合、certificate は invalid である。

### Kernel Rejection Semantics

rejection category は stable API value である。

| Category | Meaning |
|---|---|
| `malformed_certificate` | schema、version、reference、canonical encoding failure |
| `invalid_substitution` | capture、alpha-conversion、freshness、free-variable rule failure |
| `invalid_sat_proof` | clause または resolution replay failure |
| `unresolved_symbol` | referenced symbol、theorem、axiom、VC が unavailable |
| `timeout` | checking が deterministic resource budget を超過 |
| `resource_exhaustion` | memory、clause count、trace length、recursion limit 超過 |

`timeout` と `resource_exhaustion` は proof acceptance ではない。obligation は unverified のまま残る。

## インターフェイス定義

```rust
struct Certificate {
    version: CertificateVersion,
    target_vc: VcFingerprint,
    kernel_profile: KernelProfile,
    imported_axioms: Vec<ImportedFactRef>,
    imported_theorems: Vec<ImportedFactRef>,
    generated_clauses: Vec<Clause>,
    substitutions: Vec<SubstitutionStep>,
    resolution_trace: Vec<ResolutionStep>,
    derived_facts: Vec<DerivedFact>,
    final_goal: GoalRef,
}
```

## 制約と前提

- certificate checking は同一 certificate bytes と dependency artifacts に対して deterministic でなければならない。
- certificate parsing は trusted boundary の一部であり、小さく、total で、resource-bounded でなければならない。
- backend-specific proof format は kernel acceptance 前にこの format へ translate する。
- certificate hash は canonical content だけから計算し、file path、timestamp、backend runtime log を含めない。
