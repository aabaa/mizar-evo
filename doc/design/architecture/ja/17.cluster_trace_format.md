# Architecture: Cluster Trace Format

> Canonical language: English. English canonical version: [../en/17.cluster_trace_format.md](../en/17.cluster_trace_format.md).

## 目的

この文書は cluster / registration expansion の replayable trace format を定義する。

cluster search は kernel の外側で行う。kernel と artifact consumer が受け取るのは、どの cluster または registration step が使われたかを説明する explicit trace だけである。

## Design Decisions

### Cluster Step

各 cluster expansion step は次の形で表す。

```text
ClusterStep
  source_type
  applied_cluster
  generated_attribute
  generated_type
  dependency
```

`dependency` は、その expansion を正当化する theorem、definition、registration、mode、attribute、または prior cluster step を記録する。

### Chain Trace

transitive expansion は explicit ordered chain として保存する。

```text
A -> B
B -> C
C -> D
```

trace はすべての intermediate step を保持しなければならない。artifact は、元の replay steps を content-addressed reference として保存しない限り、chain を `A -> D` に collapse してはならない。

### Deterministic Traversal

cluster traversal order は次で固定する。

1. source type canonical id
2. cluster origin module path
3. declaration source order
4. generated attribute canonical id
5. registration fingerprint

worker completion order、hash-map iteration order、cache insertion order が selected trace に影響してはならない。

### Loop Detection

loop detection は deterministic である。

- active expansion stack 内の cycle は `cluster_loop` として reject する。
- すでに derived された fact の繰り返しは canonical fingerprint が一致した後にのみ ignore する。
- maximum expansion depth と maximum generated fact count は verifier settings であり cache keys に含める。
- maximum 超過は bounded failure であり、required facts を silent に truncate してはならない。

### Trace Consumers

trace consumer は次を行ってよい。

- diagnostics と artifact validation のため cluster derivations を replay する。
- generated type facts を VC に含める。
- dependency slices を fingerprint する。
- `@show_resolution` explanation を表示する。

trace consumer は trace に存在しない追加 cluster を infer してはならない。

## Interface Definitions

```rust
struct ClusterTrace {
    source_item: ItemId,
    steps: Vec<ClusterStep>,
    derived_facts: Vec<ClusterFact>,
    traversal_profile: ClusterTraversalProfile,
}

struct ClusterStep {
    source_type: TypeFingerprint,
    applied_cluster: ClusterFingerprint,
    generated_attribute: AttributeFingerprint,
    generated_type: TypeFingerprint,
    dependency: ClusterDependencyRef,
}
```

## Constraints and Assumptions

- cluster trace は semantic artifact であり deterministic hash に含める。
- coherence proof が accepted されていない registration は cluster trace から不可視である。
- hidden transitive expansion と implicit cluster insertion は禁止する。
- fail tests は infinite chain、cyclic registration、unintended coercion、hidden transitive expansion を含める。
