## 目的

この文書は、evo2 における失敗の分類・伝播・順序を固定します。

安定した失敗の意味論は、再現可能なビルド・信頼できる LSP の挙動・不健全性の防止を重視する回帰テストに必要です。

## 設計判断

### 失敗の分類

実装は、少なくとも次を区別しなければなりません。

- `parse_error`
- `resolve_error`
- `type_error`
- `overload_ambiguity`
- `cluster_loop`
- `atp_timeout`
- `certificate_rejection`
- `kernel_rejection`

これらの分類は機械可読であり、文言の変更に対して安定です。

カーネル／evidence の失敗は、2 層の分類を用います。`FailureCategory` は、
ビルドの伝播と診断の順序に用いるフェーズレベルのクラスを記録します。
legacy envelope の `malformed_certificate`、`unsupported_certificate_format`、
`invalid_substitution`、`invalid_sat_refutation` のような、より具体的な棄却理由は、
失敗レコードの安定した構造化された詳細として保存します。

### フェーズの所有

各失敗は、それを健全に検出できる最も早いフェーズが所有者になります。

例:

- 不正な構文は構文解析器の失敗であり、型の失敗ではない。
- 未解決のシンボルはリゾルバの失敗であり、ATP の失敗ではない。
- 曖昧なオーバーロードはオーバーロード解決の失敗であり、カーネルの失敗ではない。
- 不正な kernel evidence または不正な legacy certificate は evidence の棄却である。
- 不正な解決の再生はカーネルの棄却である。

### 決定的なエラー順序

エラーは正準的な順序で出力します。

1. パッケージ ID とモジュールパスによるソースファイル
2. ソース範囲
3. フェーズ順
4. 重大度
5. 診断コード
6. 安定した詳細キー

並列検証の完了順が、診断の順序に影響してはなりません。

### 失敗の伝播

失敗は依存する作業だけをブロックします。

- 構文解析エラーは、同一ファイルの意味論フェーズをブロックする。
- インターフェイスの失敗は、その失敗したインターフェイスを必要とするインポート側をブロックする。
- 失敗した VC は、同一モジュール内の独立した VC の失敗を隠してはならない。
- ATP のタイムアウトは、別の受理済み kernel evidence result がない限り、義務を open のままにする。
- evidence／カーネルの棄却は、リリースポリシーのもとで警告にならない。

ブロックされた作業は明示的にブロック済みとして表現し、黙って省いてはなりません。

### 健全性の優先

分類が曖昧な場合、実装は不健全な受理を防ぎ、最も精密な棄却境界を保つ分類を選びます。

失敗を成功に変えるために、フォールバック推論・隠れた型強制・暗黙の cluster 挿入・未検査の ATP 結果を用いてはなりません。

## インターフェイス定義

```rust
enum FailureCategory {
    ParseError,
    ResolveError,
    TypeError,
    OverloadAmbiguity,
    ClusterLoop,
    AtpTimeout,
    CertificateRejection,
    KernelRejection,
}

struct FailureRecord {
    category: FailureCategory,
    phase: PipelinePhase,
    owner: SemanticOwnerId,
    source_range: Option<SourceRange>,
    diagnostic_code: DiagnosticCode,
    stable_detail_key: String,
    rejection_reason: Option<RejectionReason>,
}
```

## 制約と前提

- 診断テキストは改善してよいが、分類・フェーズの所有・安定したコードの変更には互換性のレビューが必要である。
- タイムアウトと資源枯渇は、受理しないという結果である。
- 失敗のスナップショットは回帰テストに含める。
- 定理証明の成功は、それより早い決定的な失敗を無視することに依存してはならない。
