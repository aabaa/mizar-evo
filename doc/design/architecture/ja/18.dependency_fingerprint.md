# アーキテクチャ: 依存フィンガープリント

> 正本は英語です。英語版: [../en/18.dependency_fingerprint.md](../en/18.dependency_fingerprint.md)。

## 目的

この文書は、安定した増分再ビルドのための依存フィンガープリントと依存スライスを定義します。

目的は、変更に影響される意味論的な作業だけを再ビルドしつつ、決定的なアーティファクトと証明受理を保つことです。

## 設計判断

### フィンガープリントの対象

ビルドシステムは、少なくとも次をフィンガープリントします。

- 定理ステートメントと受理済みの証明状態
- 定義と定義的な展開境界
- 記法宣言と構文解析への効果
- cluster
- registration
- mode
- 属性

証明本体は、定理ステートメントとは別にフィンガープリントします。これにより、受理済みの定理ステートメントが変わらず、証明の実装だけが変わった場合に、下流モジュールを再ビルドしません。

### 依存スライス

各定理と検証条件は、依存スライスを保存します。

```text
DependencySlice
  used_theorems
  used_definitions
  used_clusters
  used_notations
  used_modes
  used_attributes
```

スライスは、構文解析・解決・型検査・elaboration・VC 生成・ATP 変換・証明書受理に使われたものを正確に記録します。

### 安定したフィンガープリント入力

フィンガープリントには、次を含めます。

- 正準的な意味論の内容
- スキーマバージョン
- 言語エディション
- 依存向けのポリシーフィールド
- 正規化された由来 ID
- 可視性が依存する場合の受理済みの証明状態

フィンガープリントには、次を含めません。

- コメント
- 構文に影響する記法以外のソースの整形
- タイムスタンプ
- ローカルの絶対パス
- ワーカー ID
- バックエンドのランタイム実行時間

### 再ビルドのトリガ

この表は、下流の意味論的な再ビルドと、ローカルなアーティファクトの更新を区別します。証明本体のみの編集は、ローカルな証明 witness や実装ハッシュの再ビルドを必要とする場合がありますが、エクスポートされた定理ステートメントと受理済みの証明状態が変わらない限り、下流モジュールを無効化してはなりません。

| 変更 | 下流の意味論的な再ビルド | ローカルなアーティファクトの更新 |
|---|---|---|
| コメントのみ | no | no |
| 定理の証明本体のみ | no | yes（証明 witness または実装ハッシュが変わった場合） |
| 定理ステートメント | yes | yes |
| cluster の意味論 | yes | yes |
| 記法の構文解析結果 | yes | yes |
| registration の coherence 証明状態 | その registration を見られるインポート側について yes | yes |
| 診断の文言のみ | no | yes（診断スナップショットのみ） |

### キャッシュの検証

キャッシュヒットが有効なのは、依存スライス内のすべてのフィンガープリントが一致し、有効な検証器ポリシーが記録されたポリシーと互換である場合だけです。

スライスが欠落している、または未対応のスキーマで生成された場合、ビルドは影響を受けるフェーズを保守的に再実行しなければなりません。

## インターフェイス定義

```rust
struct DependencySlice {
    owner: SemanticOwnerId,
    used_theorems: Vec<TheoremFingerprint>,
    used_definitions: Vec<DefinitionFingerprint>,
    used_clusters: Vec<ClusterFingerprint>,
    used_notations: Vec<NotationFingerprint>,
    used_modes: Vec<ModeFingerprint>,
    used_attributes: Vec<AttributeFingerprint>,
}
```

## 制約と前提

- フィンガープリントの計算は、正準的な順序と正準的なシリアライズを用いる。
- 増分再ビルドは最適化であり、完全なクリーンビルドが基準となる挙動である。
- 欠落した依存データを「依存なし」と解釈してはならない。
- 依存スライスは、決定的ビルドのテストにおける回帰スナップショットに含める。
