# Source/Spec Correspondence Audit: mizar-core

> Canonical language: English. Japanese companion:
> [../ja/source_spec_audit.md](../ja/source_spec_audit.md).

Task 22 audits the public `mizar-core` surface after task 21. It does not
change source behavior, public APIs, `.miz` fixtures, expectations, or runtime
semantics. The audit records where the implemented public surface is specified,
tested, guarded, or intentionally deferred.

## Scope And Method

The inventory covers every current `pub mod` export in
`crates/mizar-core/src/lib.rs`, every top-level public item in
`crates/mizar-core/src/*.rs`, and public newtypes/tables generated by the
crate-local `dense_id!`, `string_key!`, and `table!` macros. Public methods are
grouped under their owning public types because the module specs describe
builder, table, accessor, lowering, and rendering APIs at that level.

Module specifications audited:

- [core_ir.md](./core_ir.md)
- [binder_normalization.md](./binder_normalization.md)
- [elaborator.md](./elaborator.md)
- [control_flow.md](./control_flow.md)

Result: No source/spec drift is observed for implemented explicit-payload APIs.
No blocking `source_undocumented_behavior`, `design_drift`, or
`boundary_violation` remains in the public surface. Remaining unavailable
coverage is classified below as `external_dependency_gap` or `deferred` work
instead of being treated as implemented behavior.

## Crate Module Exports

`src/lib.rs` exports exactly these public modules:

- `binder_normalization`
- `control_flow`
- `core_ir`
- `elaborator`

Evidence: `crates/mizar-core/tests/lint_policy.rs` checks this list through
`public_semantic_modules_have_owning_specs`, keeps the crate off frontend and
downstream VC/proof/kernel boundaries, and now checks that this audit mentions
each public module/source/spec path.

## Source-Undocumented Behavior Pass

The audit treated source behavior as inventory only until it was matched to a
module spec, active Rust test, lint guard, traceability row, or explicit gap
classification. Every current top-level public item is grouped in the module
inventories above and has a correspondence row tying its promised behavior to
spec and test evidence.

No `source_undocumented_behavior` remains for the current public surface. The
items that cannot be covered by active source-derived `.miz` or snapshot tests
are not undocumented source behavior; they are unavailable seams classified in
the follow-up register below.
