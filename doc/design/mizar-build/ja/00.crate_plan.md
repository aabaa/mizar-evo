# Crate Plan: mizar-build

> 正本は英語です。英語版:

## 責務

所有する挙動:

- pipeline phase 0 の workspace 計画: package/workspace manifest、lockfile
  検証、依存グラフ解決、決定的な `BuildPlan` 生成。
- `BuildPlan` と source layout または dependency artifact summary からの
  phase-0 module-index 構築。
- 並列検証と増分 invalidation で使う versioned task graph。
- scheduling、決定的な result collation、resource budget、協調的
  cancellation、有界 failure propagation、決定的 artifact commit ordering。
- 実行前に `mizar-cache` の判断を消費でき、clean-build equivalence を保つ
  cache-aware scheduling seam。

範囲外:

- build request、CLI/watch/LSP entry point、build session、event stream、
  phase-service registry、`salsa` query/database 境界。これらはすべて
  `mizar-driver` が所有する。
- raw IR storage、output handle、snapshot-handle rehydration。これらは
  `mizar-ir` の仕事である。
- `mizar-cache` の `CacheKey` 構築、dependency fingerprint 構築、
  cache-store compatibility check、proof-reuse validation。
- artifact schema ownership、manifest transaction 内部、proof witness
  publication token、producer-owned artifact projection。
- proof authority、proof search、ATP/backend winner selection、kernel
  acceptance、proof-status promotion。

`mizar-build` は、検証済み cache hit の後で task execution を skip してよい。
ただし hit は optimization only である。semantic acceptance、proof
authority、あるいは task、artifact record、backend result、externally
attested proof をより trusted な status に昇格する理由になってはならない。

`mizar-build` は `mizar-driver` に依存してはならない。依存方向は
driver-to-build である。

## 既知の gap と drift

`external_dependency_gap` は、利用できない upstream/downstream integration
境界を記録するための、この repository / user request で要求された分類である。
これはこの task では protocol taxonomy を拡張するが、planned `mizar-build`
work ではなく external seam の欠落を記録する場合にだけ使う。

| ID | Class | Evidence | Action |
|---|---|---|---|
| BUILD-G-001 | `design_drift` | autonomous crate protocol は `doc/design/mizar-build/en/00.crate_plan.md` を要求するが、この crate には TODO/module spec しかなかった。 | task 0 でこの英語 plan と日本語 companion を追加する。 |
| BUILD-G-002 | `external_dependency_gap` | `mizar-driver` request/session/registry/salsa tasks は open で、crate も存在しない。 | `mizar-build` を entry-point agnostic に保ち、driver-owned API や driver dependency を追加しない。 |
| BUILD-G-003 | `external_dependency_gap` | `mizar-ir` が存在せず、output storage、sealed handle、snapshot rehydration が利用できない。 | scheduler tests では synthetic in-memory task output を使い、placeholder IR API を作らない。 |
| BUILD-G-004 | `external_dependency_gap` | real producer artifact publication token と full phase-15 emission input が公開されていない。 | 可能な場合は commit-order boundary だけを実装し、real producer publication integration は deferred として、token を創作しない。 |
| BUILD-G-005 | `deferred` | `mizar-cache` は存在し、`CacheKey`、dependency fingerprint、proof-reuse validation を所有するが、build-side consumer seam はまだ task 18 のままである。 | task 18 で `mizar-build` consumer seam を実装し、必要な tests では mock decision を使う。cache internals を再実装しない。 |
| BUILD-G-006 | `external_dependency_gap` | full resolver/checker/VC/proof/kernel/driver integration は end-to-end real build に利用できない。 | producer/consumer seam が存在するまで、Wave B tests は synthetic に保つ。 |

Tests: 既存 behavior tests と、moved APIs に対する source/spec・bilingual
audits。

Affected design files: module-boundary report、TODO status、file が移動する
場合の module table updates。

Affected source files: private module splits のみ。必要に応じて unchanged
public exports と tests。

Completion condition: 必要な split が behavior-preserving で、review・verified
済みであり、crate hard gates が closeout 可能な状態である。

Forbidden behavior: module move 内で API exposure、behavior cleanup、
diagnostic change、schema change、unrelated refactor を行わない。

## Exit criteria

Hard gates:

- blocking/high specification inconsistency が残っていない。
- `mizar-build` source behavior が module specs または記録済み
  external/deferred gap list なしに残っていない。
- source、tests、英語 docs、日本語 companion docs が同期している。
- 利用できない downstream integration は stub ではなく
  `external_dependency_gap` または `deferred` として分類されている。
- crate が `mizar-driver` に依存していない。
- cache-aware scheduling は optimization-only execution-skip seam として
  文書化・実装されている。
- cache hits と artifact records が proof trust や semantic acceptance を
  昇格しない。
- task graph、scheduler、resource budget、cancellation、failure propagation、
  deterministic commit boundary、cache seam behavior が focused tests または
  明示的 deferred reasons で覆われている。
- review-only spec/test/full/source-doc reviews に unresolved blocking/high
  findings が残っていない。
- required verification commands が通るか、未実行 command が明示的に正当化
  されている。

Verification commands:

```text
cargo test -p mizar-build
cargo clippy -p mizar-build --all-targets -- -D warnings
cargo fmt --check
```

scheduler/cache/artifact/commit boundaries に触れた場合は、利用可能な隣接
crate checks も実行する:

```text
cargo test -p mizar-cache
cargo test -p mizar-artifact
cargo test -p mizar-vc
cargo test -p mizar-proof
cargo test -p mizar-driver
```

crate completion では以下を実行する:

```text
cargo fmt --check
cargo clippy --all-targets --all-features -- -D warnings
cargo test
```

Review expectations:

- 各 task は AGENTS.md の spec/documentation、test sufficiency、full
  implementation、source/documentation consistency の review loop に従う。
- 残る low/medium notes は concrete deferred reason を添えて記録するか、
  closeout 前に修正する。
- crate completion には read-only quality score 90/100 以上が必要である。
