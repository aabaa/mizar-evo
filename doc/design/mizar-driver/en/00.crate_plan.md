# Crate Plan: mizar-driver

> Canonical language: English. Japanese companion:
> [../ja/00.crate_plan.md](../ja/00.crate_plan.md).

## Responsibility

Owned behavior:

- build request lifecycle for batch, watch, and LSP-originated builds;
- `BuildSession` creation around one current source/dependency snapshot;
- current-snapshot publication guards that reject obsolete watch/LSP results;
- deterministic phase-service registration and lookup;
- the driver-owned query boundary that wraps pure phase services and keeps phase
  crates free of query-engine dependencies;
- phase-0 bootstrap through `mizar-build`, task-graph submission to the
  `mizar-build` scheduler, and cancellation requests with supported
  supersession expressed through `mizar-build` cancellation policy;
- protocol-agnostic build events for CLI progress, diagnostics readiness, phase
  completion, stale-result suppression, and artifact-boundary handoff;
- CLI batch entry point and watch-mode orchestration over the same driver API.

Out of scope:

- phase semantics, type checking, name resolution, VC generation, proof
  acceptance, trusted status, kernel acceptance, and proof-policy selection;
- cache compatibility decisions, cache-key validation authority, proof-reuse
  validation, or cluster-db ownership;
- artifact serialization, manifest transaction authority, or publication-token
  issuance;
- LSP protocol conversion, source range conversion, document-version protocol
  handling, or editor command/code-action shaping;
- fake phase adapters, provisional artifact publication tokens, placeholder LSP
  bridges, or stub producer outputs.

## Exit Criteria

Hard gates:

- no blocking/high specification inconsistency remains;
- every public driver behavior is documented in paired EN/JA design files and
  traced to Rust tests or a classified deferred/external gap;
- no source behavior remains that is absent from design/spec/test intent;
- no `.miz` test, language spec, traceability metadata, or expectation sidecar
  is changed merely to match implementation behavior;
- driver boundaries are preserved: no phase semantics, proof authority, cache
  compatibility decision, artifact serialization/token issuance, or LSP
  protocol conversion moves into `mizar-driver`;
- required verification passes, or unrun commands are justified with residual
  risk;
- remaining risks are classified as `external_dependency_gap`, `deferred`,
  out-of-scope, or human-owned.

Verification commands:

```text
cargo test -p mizar-driver
cargo clippy -p mizar-driver --all-targets -- -D warnings
cargo fmt --check
cargo test -p mizar-build
cargo test -p mizar-ir
cargo test -p mizar-diagnostics
cargo test -p mizar-frontend
cargo clippy --all-targets --all-features -- -D warnings
cargo test
git diff --check
git diff --cached --check
```

Review expectations:

- use review-only sub-agents for implementation specification/documentation
  review, test sufficiency review, full implementation review, source/docs
  consistency review, and final quality review when available;
- treat blocking/high findings as actionable until fixed or explicitly
  reclassified;
- commit exactly one completed task per commit, after verification and reviews;
- stage only task-related paths with explicit `git add` arguments.
