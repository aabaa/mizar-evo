## 責務

所有する挙動:

- batch、watch、LSP 由来 build の build request lifecycle;
- 1 つの current source/dependency snapshot に対する `BuildSession` 作成;
- 置き換えられた watch/LSP 結果を拒否する current-snapshot publication guard;
- 決定的な phase-service 登録と lookup;
- pure phase service を包み、phase crate に query-engine 依存を漏らさない
  driver-owned query boundary;
- `mizar-build` による phase-0 bootstrap、`mizar-build` scheduler への task graph
  submission、`mizar-build` cancellation policy で表現される cancellation request;
- CLI progress、diagnostics readiness、phase completion、stale-result
  suppression、artifact-boundary handoff の protocol-agnostic build event;
- 同じ driver API 上の CLI batch entry point と watch-mode orchestration。

範囲外:

- phase semantics、type checking、name resolution、VC generation、proof
  acceptance、trusted status、kernel acceptance、proof-policy selection;
- cache compatibility decision、cache-key validation authority、proof-reuse
  validation、cluster-db ownership;
- artifact serialization、manifest transaction authority、publication-token
  issuance;
- LSP protocol conversion、source range conversion、document-version protocol
  handling、editor command / code-action shaping;
- fake phase adapter、provisional artifact publication token、placeholder LSP
  bridge、stub producer output。

観測された挙動:

- この task stream 前には `mizar-driver` は workspace に存在しない。
- `mizar-session` は `BuildRequestId`、`BuildSessionId`、`BuildSnapshotId`、
  `SnapshotRegistry::create_snapshot`、`SnapshotRegistry::is_current_for_request`
  を公開している。
- `mizar-build` は manifest planning、module indexing、task graph
  construction、scheduler run、scheduler event/result、cache scheduling
  decision、artifact-commit freshness guard、cancellation policy/state を公開している。
- `mizar-ir` は sealed phase-output storage、`PhaseOutputPublisher`、snapshot
  replacement、artifact projection draft を公開するが、real driver、
  diagnostics、producer payload、artifact-publication-token integration を
  external gap としている。
- `mizar-diagnostics` は stable diagnostic record/index と CLI rendering を公開する。
  message text は diagnostic identity ではない。
- `mizar-frontend` は real source-to-syntax orchestration を公開するが、
  driver-owned phase adapter は fake payload なしで real `mizar-ir` producer
  contract を通じて output を seal できる場合だけ消費できる。
- `mizar-artifact` は manifest transaction と opaque freshness check を公開しているが、
  full phase-15 producer emission と publication authority は driver が mint する API
  ではない。
- `mizar-lsp` は現状 source-range conversion utility のみを公開しており、build
  bridge は external owner seam のまま。

- blocking/high specification inconsistency が残らない;
- すべての public driver behavior が paired EN/JA design file に文書化され、Rust test
  または分類済み deferred/external gap に trace される;
- design/spec/test intent にない source behavior が残らない;
- implementation behavior に合わせるためだけに `.miz` test、language spec、
  traceability metadata、expectation sidecar を変更しない;
- driver boundary を守る: phase semantics、proof authority、cache compatibility
  decision、artifact serialization/token issuance、LSP protocol conversion を
  `mizar-driver` に移さない;
- required verification が通るか、未実行 command を residual risk とともに正当化する;
- 残る risk は `external_dependency_gap`、`deferred`、out-of-scope、human-owned
  として分類される。

Verification commands:

```text
cargo test -p mizar-driver
cargo clippy -p mizar-driver --all-targets -- -D warnings
cargo fmt --check
cargo test -p mizar-build
cargo test -p mizar-ir
cargo test -p mizar-diagnostics
cargo test -p mizar-frontend
cargo clippy --all-targets --all-features -- -D warnings
cargo test
git diff --check
git diff --cached --check
```

Review expectations:

- 利用可能な場合、implementation specification / documentation review、
  test sufficiency review、full implementation review、source/docs consistency
  review、final quality review に review-only sub-agent を使う;
- blocking/high finding は修正または明示 reclassification まで actionable とする;
- review と verification 後、完了した task を 1 commit ずつ commit する;
- task-related path だけを明示的な `git add` で stage する。
