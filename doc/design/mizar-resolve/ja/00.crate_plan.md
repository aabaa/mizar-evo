# crate plan: mizar-resolve

> 正本は英語です。英語版: [../en/00.crate_plan.md](../en/00.crate_plan.md)。

## 責務

所有する挙動:

- `SurfaceAst` と phase 0 の module-index 入力を消費し、`ResolvedAst` と
  `SymbolEnv` を生成する。
- module path、import、export、namespace、label、安定した symbol identity を
  解決する。
- type checking を行わずに declaration signature と依存側に見える module
  summary を収集する。
- 未解決または曖昧な参照を明示的に表現し、recoverability を保つ。
- output ordering、id、debug rendering、diagnostics を決定的に保つ。

対象外:

- syntax ownership、parser recovery、AST vocabulary の変更。
- type inference、overload winner selection、cluster firing、coercion insertion。
- proof validity、VC generation、ATP interaction、kernel acceptance、artifact emission。
- `mizar-build` と `mizar-session` が所有する package manifest parsing、
  lockfile resolution、source loading、module discovery。
- driver lifecycle、salsa/query integration、phase registry ownership。

## 開始可能性

`mizar-resolve` は [todo.md](./todo.md) の task 1〜7 から開始できる。
`mizar-build` wave A は完了済みであるため、resolver 側の module-index seam は
[module_index.md](../../mizar-build/ja/module_index.md) の build-side provider を
相手に仕様化できる。parser/syntax surface は `mizar-parser` task 45 まで最新であり、
task 46 は deferred なので resolver foundation をブロックしない。

完全な user-facing resolver diagnostics は、[architecture 03](../../architecture/ja/03.module_and_symbol_resolution.md)、
[spec chapter 22](../../../spec/ja/22.error_handling_and_diagnostics.md)、および
`mizar-diagnostics` に残る name/import/label diagnostic adoption gap が解決
または明示的に deferred されるまでは開始しない。共有 diagnostic registry は
`Resolution` family を reserve しているが、resolver-owned な具体的 semantic
name、numeric descriptor、migration behavior、consumer projection は後続の
integration work に残る。
