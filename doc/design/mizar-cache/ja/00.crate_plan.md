# Crate Plan: mizar-cache

> 正本は英語です。英語版:
> [../en/00.crate_plan.md](../en/00.crate_plan.md)。

## 責務

所有する挙動:

- 内部 build `CacheKey` の正準構築と domain-separated key hash。
- dependency slice、dependency fingerprint、compatibility diff、保守的な
  rebuild-trigger evaluation。
- `cache_dir` 配下の内部 cache record と content-addressed blob storage。
- `mizar-proof` が export する metadata 上の fail-closed proof-reuse validation。
- 受理済み registration / cluster contribution だけを扱う `cluster-db/`
  contribution index、origin tracking、import-scoped view。

範囲外:

- proof authority、proof search、kernel acceptance、ATP backend execution、
  deterministic winner selection、proof status projection、trusted
  `used_axioms` projection、artifact manifest publication、scheduler ownership。
- 未完成の `mizar-build` cache-aware scheduling、`mizar-ir` cache adapter、
  artifact committed-publication token との placeholder integration。
- cache record、externally attested evidence、backend diagnostic、backend log、
  timing metadata、arrival/write order を kernel-verified status や trusted
  proof fact へ昇格させること。

## Exit Criteria

Hard gates:

- unresolved blocking/high specification inconsistency がない。
- crate scope 内の source behavior が undocumented / untested のまま残っていない。
- cache record、externally attested evidence、diagnostic、log、timing、arrival/write
  order が proof authority にならない。
- reusable record ごとに complete dependency footprint または明示的な
  uncacheable/miss behavior がある。
- unknown schema/toolchain/policy compatibility は fail closed する。
- English/Japanese docs と source が同期している。
- downstream gap は stub ではなく分類されている。
- required verification command が通るか、未実行理由が明示されている。

Verification commands:

```sh
cargo test -p mizar-cache
cargo clippy -p mizar-cache --all-targets -- -D warnings
cargo fmt --check
cargo clippy --all-targets --all-features -- -D warnings
cargo test
```

Review expectations:

- 各 task は AGENTS.md の review phases を使う。
- read-only crate quality review は有効な 90/100 以上の score を付ける。
- task commit は 1 task 1 commit とし、明示 staging だけを使う。
