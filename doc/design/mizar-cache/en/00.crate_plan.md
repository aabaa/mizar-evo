# Crate Plan: mizar-cache

> Canonical language: English. Japanese companion:
> [../ja/00.crate_plan.md](../ja/00.crate_plan.md).

## Responsibility

Owned behavior:

- canonical internal build `CacheKey` construction and domain-separated key
  hashing;
- dependency slices, dependency fingerprints, compatibility diffs, and
  conservative rebuild-trigger evaluation;
- internal cache record and content-addressed blob storage under `cache_dir`;
- fail-closed proof-reuse validation over metadata exported by `mizar-proof`;
- `cluster-db/` contribution indexes, origin tracking, and import-scoped
  views over accepted registration/cluster contributions only.

Out of scope:

- proof authority, proof search, kernel acceptance, ATP backend execution,
  deterministic winner selection, proof status projection, trusted
  `used_axioms` projection, artifact manifest publication, and scheduler
  ownership;
- placeholder integration with unfinished `mizar-build` cache-aware
  scheduling, `mizar-ir` cache adapters, or artifact committed-publication
  tokens;
- promotion of cache records, externally attested evidence, backend
  diagnostics, backend logs, timing metadata, or arrival/write order into
  kernel-verified status or trusted proof facts.

## Exit Criteria

Hard gates:

- no unresolved blocking/high specification inconsistency;
- no source behavior remains undocumented or untested within the crate scope;
- cache records, externally attested evidence, diagnostics, logs, timing, and
  arrival/write order are never proof authority;
- complete dependency footprint or explicit uncacheable/miss behavior exists
  for every reusable record;
- unknown schema/toolchain/policy compatibility fails closed;
- source and documentation agree across English and Japanese docs;
- downstream gaps are classified rather than stubbed;
- required verification commands pass or unrun commands are justified.

Verification commands:

```sh
cargo test -p mizar-cache
cargo clippy -p mizar-cache --all-targets -- -D warnings
cargo fmt --check
cargo clippy --all-targets --all-features -- -D warnings
cargo test
```

Review expectations:

- every task uses the AGENTS.md review phases;
- read-only crate quality review assigns a valid score of at least 90/100;
- task commits remain one task per commit with explicit staging only.
