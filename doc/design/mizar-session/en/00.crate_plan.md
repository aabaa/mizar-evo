# Crate Plan: mizar-session

> Canonical language: English. Japanese companion: [../ja/00.crate_plan.md](../ja/00.crate_plan.md).

This plan records the completed `mizar-session` crate against the autonomous
crate-development protocol. It is a retrospective plan because the crate was
implemented before `doc/design/autonomous_crate_development.md` became the
standard crate-wide workflow.

## Responsibility

Owned behavior:

- source identity and source hashes for loaded package, open-buffer, and
  generated source text;
- content-derived build snapshot identity over source, dependency, lockfile,
  toolchain, and verifier configuration inputs;
- snapshot-scoped ids, source-version records, freshness checks, and snapshot
  leases;
- source coordinate maps between loaded source text, source-loading input,
  preprocessed lexical text, and generated spans;
- retention guards and collection eligibility for snapshots and retained source
  resources.

Out of scope:

- language syntax, static semantics, proof semantics, type behavior, name
  resolution, overload behavior, and parser recovery;
- phase scheduling, cache compatibility decisions, IR storage, diagnostic
  aggregation, artifact publication, and proof policy.

`mizar-session` is therefore not a language-behavior crate. It supports higher
language phases with identity and coordinate contracts, but it does not decide
whether a `.miz` program is syntactically or semantically valid.

## Relevant Tests

The executable contract for this infrastructure crate is Rust test coverage.
Test-first `.miz` additions are not applicable because the crate owns no syntax,
static semantics, proof semantics, diagnostics text, parser recovery, or
soundness-boundary language behavior.

For this crate, the autonomous protocol gate "no source behavior remains that is
absent from `doc/spec` and tests" is interpreted as follows:

- language behavior must still be represented in `doc/spec`, `.miz` tests, or
  traceability metadata before implementation;
- `mizar-session` infrastructure behavior must be represented in the
  `doc/design/mizar-session/en/` component specs and Rust tests listed below;
- if a future `mizar-session` change exposes language behavior, that change
  must add or cite the appropriate `doc/spec` / `.miz` authority before source
  changes.

| Test path | Intent | Spec/design refs |
|---|---|---|
| `crates/mizar-session/src/ids.rs` unit tests | Opaque id behavior, allocator behavior, and published snapshot-id serialization. | `ids.md` |
| `crates/mizar-session/src/source_map/tests.rs` | Line/column conversion, loading maps, preprocess maps, generated spans, retained service behavior, and error surfaces. | `source_map.md`, architecture source/frontend and diagnostics/LSP docs |
| `crates/mizar-session/src/snapshot/tests.rs` | Snapshot identity, canonical ordering, validation, freshness, lease accounting, and duplicate lease defense. | `snapshot.md`, architecture artifact/incremental docs |
| `crates/mizar-session/src/source/tests.rs` | Disk, open-buffer, and generated source loading, path normalization, source hashes, UTF-8/BOM/newline handling, and error fidelity. | `source.md`, architecture source/frontend docs |
| `crates/mizar-session/src/retention.rs` unit tests | Retain/release accounting, current marks, collection summaries, and inconsistent-state reporting. | `retention.md`, internal IR snapshot-handle docs |
| `crates/mizar-session/tests/determinism.rs` | Public deterministic behavior across insertion order, scheduling-like allocation order, and equivalent source-map retention order. | `ids.md`, `snapshot.md`, `source_map.md` |

Planned test-first additions:

- None. Future `.miz` tests belong to crates that own language behavior. Future
  `mizar-session` changes should add Rust tests unless they intentionally expose
  language behavior through another crate.

## Task Decomposition

### Task MS-001: Identifier and Hash Boundaries

Purpose: provide opaque session-local ids and content-derived snapshot ids.

Spec refs:

- [ids.md](./ids.md)
- [snapshot.md](./snapshot.md)

Tests:

- `crates/mizar-session/src/ids.rs`
- `crates/mizar-session/src/snapshot/tests.rs`
- `crates/mizar-session/tests/determinism.rs`

Affected design files:

- `doc/design/mizar-session/en/ids.md`
- `doc/design/mizar-session/en/snapshot.md`

Affected source files:

- `crates/mizar-session/src/identity.rs`
- `crates/mizar-session/src/ids.rs`
- `crates/mizar-session/src/snapshot.rs`

Completion condition:

- allocator-issued ids are opaque and non-persistable;
- `BuildSnapshotId` uses canonical published-schema serialization;
- snapshot hashing is deterministic and excludes session-local ids.

Forbidden behavior:

- using allocator-issued ids as artifact compatibility keys;
- embedding paths, timestamps, memory addresses, or task-local counters in
  content-derived snapshot ids.

Spec refs:

- [source.md](./source.md)

- `crates/mizar-session/src/source.rs`

Completion condition:

- disk and open-buffer sources validate paths, UTF-8, BOM, newline, and version

- snapshot creation validates source-version identity before hashing.

Forbidden behavior:

- lossy UTF-8 decoding;
- source-id allocation before rejection points that must fail first;
- package-name spelling decisions that belong to unresolved language/package
  specification work.

Spec refs:

- [source_map.md](./source_map.md)
- [source.md](./source.md)

Tests:

- `crates/mizar-session/src/source_map/tests.rs`

- `doc/design/mizar-session/en/source.md`

Affected source files:

- `crates/mizar-session/src/source_map.rs`

Completion condition:

- range validation is source-id aware;
- line/column conversion is checked and deterministic;

Forbidden behavior:

- reading files directly from `source_map`;
- implementing LSP UTF-16 conversion inside `mizar-session`;
- treating generated spans as primary user ranges without an origin reason.

## Exit Criteria

Hard gates:

- no blocking/high specification inconsistency in `mizar-session` scope remains;
- no `mizar-session` source behavior remains undocumented in the component
  design docs;
- language `.miz` coverage is explicitly not applicable to this crate scope;
- no test expectations were changed to match implementation behavior;
- English and Japanese design docs agree on public API/status commitments;
- crate boundaries exclude scheduling, IR storage, diagnostics aggregation,
  artifact publication, and proof policy;
- verification commands pass, or any unrun command is justified;
- remaining risks are classified as deferred, out of scope, or human-owned.

Verification commands:

```sh
cargo fmt --check
cargo clippy --all-targets --all-features -- -D warnings
cargo test
cargo run -p mizar-test -- plan --tests-root tests --manifest tests/coverage/spec_trace.toml
```

Review expectations:

- review the crate plan and exit report for protocol completeness;
- review English and Japanese document synchronization;
- review that no language-behavior claims were introduced without `doc/spec` or
  `.miz` test authority;
- review that `MS-GAP-001` remains deferred rather than silently repaired.
