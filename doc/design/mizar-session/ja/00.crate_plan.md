# Crate Plan: mizar-session

> 正本は英語です。英語版: [../en/00.crate_plan.md](../en/00.crate_plan.md)。

この計画は、完了済みの `mizar-session` crate を autonomous
crate-development protocol に照らして記録するものです。
`doc/design/autonomous_crate_development.md` が crate-wide 標準ワークフローに
なる前に実装が進んでいたため、これは後追いの crate plan です。

## 責務

所有する振る舞い:

- package source、open-buffer、generated source のソース同一性とソースハッシュ
- source、dependency、lockfile、toolchain、verifier configuration 入力から作る
  content-derived build snapshot identity
- snapshot-scoped id、source-version record、freshness check、snapshot lease
- loaded source text、source-loading input、preprocessed lexical text、generated span
  の間の source coordinate map
- snapshot と retained source resource の retention guard と collection eligibility

範囲外:

- 言語の syntax、static semantics、proof semantics、type behavior、name resolution、
  overload behavior、parser recovery
- phase scheduling、cache compatibility decision、IR storage、diagnostic aggregation、
  artifact publication、proof policy

したがって `mizar-session` は language-behavior crate ではありません。上位の
言語フェーズを identity と coordinate の契約で支えますが、`.miz` program が
構文的または意味的に妥当かは判断しません。

## 仕様項目

主要な正本である言語仕様は、`mizar-session` API を直接は規定しません。この
crate は、言語仕様を支える architecture と component design から派生します。

## 関連テスト

この infrastructure crate の executable contract は Rust テストで表されます。
この crate は syntax、static semantics、proof semantics、diagnostic text、
parser recovery、soundness-boundary language behavior を所有しないため、
test-first `.miz` 追加は適用外です。

この crate では、autonomous protocol の「source behavior が `doc/spec` と tests
に存在しないまま残っていない」という gate を次のように解釈します。

- language behavior は、実装前に `doc/spec`、`.miz` tests、または traceability
  metadata に表現されていなければならない。
- `mizar-session` の infrastructure behavior は、下記の
  `doc/design/mizar-session/ja/` component specs と Rust tests に表現されていれば
  よい。
- 将来 `mizar-session` の変更が language behavior を公開する場合は、source change
  の前に適切な `doc/spec` / `.miz` authority を追加または引用する。

| テストパス | 意図 | 仕様/設計参照 |
|---|---|---|
| `crates/mizar-session/src/ids.rs` unit tests | opaque id、allocator、published snapshot-id serialization。 | `ids.md` |
| `crates/mizar-session/src/source_map/tests.rs` | line/column conversion、loading map、preprocess map、generated span、retained service、error surface。 | `source_map.md`、architecture source/frontend と diagnostics/LSP docs |
| `crates/mizar-session/src/snapshot/tests.rs` | snapshot identity、canonical ordering、validation、freshness、lease accounting、duplicate lease defense。 | `snapshot.md`、architecture artifact/incremental docs |
| `crates/mizar-session/src/source/tests.rs` | disk/open-buffer/generated source loading、path normalization、source hash、UTF-8/BOM/newline handling、error fidelity。 | `source.md`、architecture source/frontend docs |
| `crates/mizar-session/src/retention.rs` unit tests | retain/release accounting、current mark、collection summary、inconsistent-state reporting。 | `retention.md`、internal IR snapshot-handle docs |
| `crates/mizar-session/tests/determinism.rs` | insertion order、scheduling-like allocation order、equivalent source-map retention order に対する public deterministic behavior。 | `ids.md`、`snapshot.md`、`source_map.md` |

Planned test-first additions:

- なし。今後の `.miz` テストは language behavior を所有する crate に属します。
  今後 `mizar-session` を変更する場合は、別 crate を通じて language behavior を
  意図的に公開しない限り Rust テストを追加します。

## Task Decomposition

### Task MS-001: Identifier and Hash Boundaries

Purpose: opaque な session-local id と content-derived snapshot id を提供する。

Spec refs:

- [ids.md](./ids.md)
- [snapshot.md](./snapshot.md)

Tests:

- `crates/mizar-session/src/ids.rs`
- `crates/mizar-session/src/snapshot/tests.rs`
- `crates/mizar-session/tests/determinism.rs`

Completion condition:

- allocator-issued id は opaque かつ non-persistable;
- `BuildSnapshotId` は canonical published-schema serialization を使う;
- snapshot hashing は deterministic で session-local id を除外する。

Forbidden behavior:

- allocator-issued id を artifact compatibility key として使うこと;
- path、timestamp、memory address、task-local counter を content-derived snapshot id に
  埋め込むこと。

### Task MS-002: Source Loading and Source Versions

Purpose: source text を immutable かつ hashable な source record として読み込む。

Spec refs:

- [source.md](./source.md)
- [snapshot.md](./snapshot.md)

Tests:

- `crates/mizar-session/src/source/tests.rs`
- `crates/mizar-session/src/snapshot/tests.rs`

Completion condition:

- disk/open-buffer source が path、UTF-8、BOM、newline、version boundary を検証する;
- generated source が文書化された byte-for-byte policy を保持する;
- snapshot creation が hashing 前に source-version identity を検証する。

Forbidden behavior:

- lossy UTF-8 decoding;
- 先に失敗すべき境界で拒否する前の source-id allocation;
- package-name spelling の強制。language/package specification が揃った後も、
  この責務は upstream build-plan layer に属する。

### Task MS-003: Source Coordinate Maps

Purpose: loaded source、original source-loading input、preprocessed lexical text、
generated anchor の間で range を変換する。

Spec refs:

- [source_map.md](./source_map.md)
- [source.md](./source.md)

Tests:

- `crates/mizar-session/src/source_map/tests.rs`
- `crates/mizar-session/tests/determinism.rs`

Completion condition:

- range validation が source-id aware である;
- line/column conversion が checked かつ deterministic である;
- loaded/original/preprocess/generated mapping が primary anchor と secondary
  anchor を保持する。

Forbidden behavior:

- `source_map` が file を直接読むこと;
- LSP UTF-16 conversion を `mizar-session` 内で実装すること;
- origin reason のない generated span を primary user range として扱うこと。

### Task MS-004: Snapshot Freshness, Leases, and Retention

Purpose: downstream consumer が参照している間、source/snapshot resource を保持する。

Spec refs:

- [snapshot.md](./snapshot.md)
- [retention.md](./retention.md)

Tests:

- `crates/mizar-session/src/snapshot/tests.rs`
- `crates/mizar-session/src/retention.rs`
- `crates/mizar-session/tests/determinism.rs`

Completion condition:

- stale snapshot は request に対して current ではない;
- live lease と current mark が collection eligibility を止める;
- collection summary は artifact を削除せず skipped/stale state を報告する。

Forbidden behavior:

- retention layer による artifact/cache deletion;
- retention state による semantic validity 判断;
- custom allocator の duplicate lease id による live lease accounting 破壊。

### Task MS-005: Protocol Evidence and Bilingual Synchronization

Purpose: 完了済み crate を autonomous crate-development evidence format へ移す。

Spec refs:

- `doc/design/autonomous_crate_development.md`
- `AGENTS.md` の repository documentation policy

Tests:

- documentation review;
- `cargo fmt --check`;
- behavior change がある場合、または exit report を更新する場合の broader cargo
  verification。

Completion condition:

- crate plan と exit report が存在する;
- hard gate、deferred item、human review surface、verification expectation が記録される;
- English/Japanese design index が同期している。

Forbidden behavior:

- Rust behavior の変更;
- current implementation behavior に合わせるための `doc/spec` 変更;
- `mizar-session` 内で package-name spelling を強制し、upstream build-plan layer
  の責務を移すこと。

## Exit Criteria

Hard gates:

- `mizar-session` scope に blocking/high specification inconsistency が残っていない;
- `mizar-session` source behavior が component design docs に文書化されている;
- language `.miz` coverage はこの crate scope では明示的に適用外;
- test expectation を implementation behavior に合わせて変更していない;
- English/Japanese design docs が public API/status commitment で一致している;
- crate boundary が scheduling、IR storage、diagnostic aggregation、artifact
  publication、proof policy を除外している;
- verification command が通るか、未実行 command の理由が明示される;
- 残リスクが deferred、out of scope、human-owned として分類されている。

Verification commands:

```sh
cargo fmt --check
cargo clippy --all-targets --all-features -- -D warnings
cargo test
cargo run -p mizar-test -- plan --tests-root tests --manifest tests/coverage/spec_trace.toml
```

Review expectations:

- crate plan と exit report の protocol completeness をレビューする;
- English/Japanese document synchronization をレビューする;
- `doc/spec` または `.miz` test の権威なしに language-behavior claim が導入されて
  いないことをレビューする;
- `MS-GAP-001` が英語正本と日本語 companion で解決済みであり、
  package-name spelling enforcement が `mizar-session` へ移っていないことをレビューする。
