# Crate Plan: mizar-frontend

> 正本は英語です。英語版: [../en/00.crate_plan.md](../en/00.crate_plan.md)。

この計画は、完了済みの `mizar-frontend` crate を autonomous crate-development
protocol に照らして記録する。`doc/design/autonomous_crate_development.md` が
crate 全体の標準ワークフローになる前に実装が完了していたため、これは後追いの
retrospective plan である。

## 責務

所有する挙動:

- フェーズ 1〜3 の frontend pipeline における source-to-syntax 統制。
- `mizar-session` の loaded source から frontend `SourceUnit` への射影。
- lexer byte range から session `SourceRange` への frontend 所有の span bridge。
- preprocessing の統制、写像済み comment/doc comment、lexical text map、写像済み
  shallow import stub。
- active lexical environment request、provider provenance validation、正準 import
  処理、conflict retry、依存 lexical summary 周辺の frontend diagnostic。
- parser-assisted lexing plan、tokenization 呼び出し、写像済み lexing diagnostic、
  `TokenStream` 出力。
- parser seam 呼び出し、parser input assembly、syntax diagnostic passthrough、
  任意の `SurfaceAst` handoff。
- `SourceUnit`、`PreprocessedSource`、`ActiveLexicalEnvironment`、`TokenStream`、
  `SurfaceAst` の層状 frontend content cache key。
- 決定的な frontend diagnostic merge と hard-error boundary。

範囲外:

- source identity、source hash、source snapshot、source retention
  （`mizar-session`）。
- raw scanning、comment stripping rule、reserved table、lexical environment assembly
  rule、scope-skeleton rule、token disambiguation rule（`mizar-lexer`）。
- `SurfaceAst` node definition と syntax-owned trivia structure（`mizar-syntax`）。
- grammar implementation、Pratt parsing、parser recovery decision、parser fuzz
  ownership（`mizar-parser`）。
- module graph construction、shallow stub を越える import legality、semantic name
  resolution、type checking、overload resolution、cluster registration、proof
  checking、VC generation、cache storage、artifact publication、LSP display
  rendering。

`mizar-frontend` は境界 crate である。source、lexical、parser output を言語向けに
統制するが、その統制を後続フェーズが所有する semantic fact に変えてはならない。

## 関連テスト

`mizar-frontend` の executable contract は主に Rust coverage である。この crate は、
すでに ownership が分かれた session、lexer、syntax、parser contract を統制する
ためである。complete-source `.miz` tests は、現在の parser seam が
frontend-reachable parser behavior を露出できる箇所で使う。

この crate では、autonomous protocol の「source behavior が `doc/spec` と tests
にないまま残らない」gate を次のように解釈する。

- language behavior は、frontend source 変更の前に `doc/spec`、`.miz` tests、
  または traceability metadata に表現されていなければならない。
- complete `.miz` language contract として独立に見えない frontend orchestration
  behavior は、`doc/design/mizar-frontend/ja/` と Rust tests に表現する。
- 将来の grammar growth は、frontend がその挙動を完了扱いする前に、owning parser
  task を通じて parser/frontend `.miz` または traceability coverage を追加する。
- 現在 producer を持たない non-exhaustive fallback variant は、owning upstream
  producer が concrete variant を公開するまで deferred とする。

  active lexical environment construction、parser-assisted lexing、parser seam
  invocation、cache-key computation、diagnostic merging を駆動する。

  non-exhaustive である。現在の fallback policy は、構築可能な範囲で文書化・

## タスク分解

### Task MF-001: Source Identity Projection And Span Bridging

Purpose: `mizar-session` の source identity/map を frontend source unit へ橋渡しし、
lexer byte span を session source range へ変換する。

Spec refs:

- [source.md](./source.md)
- [span_bridge.md](./span_bridge.md)
- `doc/design/architecture/ja/02.source_and_frontend.md`

Tests:

- `crates/mizar-frontend/src/source.rs`
- `crates/mizar-frontend/src/span_bridge.rs`

Affected design files:

- `doc/design/mizar-frontend/ja/source.md`
- `doc/design/mizar-frontend/ja/span_bridge.md`

Affected source files:

- `crates/mizar-frontend/src/source.rs`
- `crates/mizar-frontend/src/span_bridge.rs`

Completion condition:

- source loading metadata が再計算されずに射影される。
- loaded/lexical coordinate conversion が source-map boundary を保持する。
- invalid span と conflicting registration が決定的に報告される。

Forbidden behavior:

- source identity、hash、snapshot、retention の所有。
- coordinate space の暗黙変換。
- source-load failure に対する source range の捏造。

- 既知の frontend source behavior が design/spec/test evidence なしに残らない。

  reason を持つ。
- 現在の実装に合わせるためだけに test expectation を変更していない。
- 英語・日本語 design docs、README indexes、source correspondence evidence、source
  behavior が同期している。
- source/session、lexer、syntax、parser、resolver、cache-storage、diagnostic
  rendering boundary が守られている。
- 関連する verification command が成功するか、skip した command の理由が明示される。
- residual risk が deferred、out of scope、human-owned に分類されている。

cargo clippy --all-targets --all-features -- -D warnings

cargo run -p mizar-test -- plan --tests-root tests --manifest tests/coverage/spec_trace.toml
