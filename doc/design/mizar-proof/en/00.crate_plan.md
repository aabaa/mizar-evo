# Crate Plan: mizar-proof

> Canonical language: English. Japanese companion:
> [../ja/00.crate_plan.md](../ja/00.crate_plan.md).

## Responsibility

Owned behavior:

- verifier proof policy evaluation over untrusted ATP evidence, kernel check
  results, deterministic built-in discharge records, externally attested
  records, and open obligations;
- deterministic winner selection for proof evidence sets and artifact-facing
  proof selections;
- proof status projection for artifacts and diagnostics, including trusted
  `used_axioms` propagation only from `mizar-kernel::checker::KernelCheckResult`
  values whose status is `Accepted`;
- proof witness staging, manifest-gated publication references, and stable
  proof reuse metadata;
- policy queries used by `mizar-atp` for safe early-stop decisions.

Out of scope:

- kernel acceptance, SAT solving, ATP backend execution, proof search, premise
  selection, substitution invention, overload resolution, cluster search,
  implicit coercion insertion, cache lookup, or artifact manifest commit;
- promotion of backend proof methods, resolution traces, SMT proof objects,
  backend logs, externally attested records, cache records, or backend
  diagnostics into trusted proof status or trusted `used_axioms`;
- placeholder downstream integration. Missing consumers or producers are
  recorded as `external_dependency_gap` / `deferred`.

## Exit Criteria

Hard gates:

- no blocking/high specification inconsistency remains;
- no source behavior remains absent from module specs, tests, or explicit
  deferred records;
- crate-owned behavior has Rust coverage or documented deferred reasons;
- no `.miz` fixture or expectation sidecar is changed merely to match current
  implementation behavior;
- paired English/Japanese docs are synchronized;
- crate responsibility boundaries are not violated;
- required verification passes, or unrun commands are explicitly justified;
- remaining risks are classified as `deferred`, `out of scope`, or
  `external_dependency_gap`;
- final read-only quality review assigns a valid score of at least 90/100.

Verification commands:

```text
cargo test -p mizar-proof
cargo clippy -p mizar-proof --all-targets -- -D warnings
cargo fmt --check
cargo test -p mizar-kernel
cargo test -p mizar-vc
cargo test -p mizar-atp
cargo test -p mizar-artifact
cargo test -p mizar-checker
cargo clippy --all-targets --all-features -- -D warnings
cargo test
git diff --check
git diff --cached --check
```

Review expectations:

- Each task uses review-only agents for implementation-spec/documentation,
  test sufficiency, full implementation, and source/documentation consistency
  reviews as applicable to the task scope.
- The parent agent remains responsible for integration, verification, explicit
  staging, one-task-one-commit discipline, and final handoff.
