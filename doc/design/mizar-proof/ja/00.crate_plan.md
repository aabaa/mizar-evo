## 責務

所有する挙動:

- untrusted な ATP evidence、kernel check result、決定的 built-in discharge
  record、外部認証 record、open obligation に対する verifier proof policy 評価;
- proof evidence set と artifact 向け proof selection の決定的な勝者選択;
- artifact / diagnostics 向け proof status 射影。trusted `used_axioms` は
  `mizar-kernel::checker::KernelCheckResult` の status が `Accepted` の場合
  だけ伝播する;
- proof witness の staging、manifest gate 済み publication reference、安定した
  proof reuse metadata;
- `mizar-atp` が安全な early stop 判定に使う policy query。

範囲外:

- kernel acceptance、SAT solving、ATP backend execution、proof search、premise
  selection、substitution invention、overload resolution、cluster search、
  implicit coercion insertion、cache lookup、artifact manifest commit;
- backend proof method、resolution trace、SMT proof object、backend log、外部認証
  record、cache record、backend diagnostic を trusted proof status や trusted
  `used_axioms` に昇格させること;
- placeholder downstream integration。未完成 consumer / producer は
  `external_dependency_gap` / `deferred` として記録する。

## Exit Criteria

Hard gates:

- blocking/high specification inconsistency が残っていない;
- module specs、tests、または explicit deferred record にない source behavior が
  残っていない;
- crate-owned behavior に Rust coverage または documented deferred reason がある;
- 既存 `.miz` fixture / expectation sidecar を current implementation behavior に
  合わせるためだけに変更していない;
- paired English/Japanese docs が同期している;
- crate responsibility boundary に違反していない;
- required verification が pass するか、未実行 command が明示的に正当化されている;
- remaining risk は `deferred`、`out of scope`、または
  `external_dependency_gap` として分類されている;
- final read-only quality review が有効な 90/100 以上の score を付けている。

Verification commands:

```text
cargo test -p mizar-proof
cargo clippy -p mizar-proof --all-targets -- -D warnings
cargo fmt --check
cargo test -p mizar-kernel
cargo test -p mizar-vc
cargo test -p mizar-atp
cargo test -p mizar-artifact
cargo test -p mizar-checker
cargo clippy --all-targets --all-features -- -D warnings
cargo test
git diff --check
git diff --cached --check
```

Review expectations:

- 各 task は task scope に応じて implementation-spec/documentation、test
  sufficiency、full implementation、source/documentation consistency について
  review-only agent を使う。
- parent agent は integration、verification、明示 staging、one-task-one-commit
  discipline、final handoff の責任を持つ。
