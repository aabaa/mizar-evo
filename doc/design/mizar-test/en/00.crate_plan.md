# Crate Plan: mizar-test

> Canonical language: English. Japanese companion: [../ja/00.crate_plan.md](../ja/00.crate_plan.md).

## Responsibility

Owned behavior:

- deterministic discovery of corpus payloads and `.expect.toml` sidecars;
- parsing and validation of expectation metadata and traceability metadata;
- staged-model admission and coverage accounting;
- active runner gates for stage-owned compiler seams;
- snapshot comparison/update policy and deterministic report surfaces;
- fail/soundness bookkeeping for tests that must not silently pass, disappear,
  or weaken.

Out of scope:

- compiler language semantics;
- name resolution, type checking, proof search, ATP, certificate replay, or
  kernel proof authority outside explicit consumer-runner seams;
- changing `doc/spec`, existing `.miz` tests, or existing expectation files to
  match current implementation behavior.

## Task Decomposition

### Task 1: Lint-policy guard

Purpose: keep `mizar-test` opted into workspace lints and require documented
lint exceptions.

Completion: committed as `0b24d1a`.

### Task 2: Source/spec gap audit and status sync

Purpose: record the audit baseline, classify drift/gaps, and synchronize module
status.

Completion: this crate plan plus `todo.md` status/follow-up updates.

Forbidden behavior: repairing `doc/spec`, existing `.miz`, or existing
expectations in this audit commit.

### Task 3: Runner modes and CLI completion

Purpose: complete validation-mode CLI behavior over corpus and coverage
metadata, including exit-code fixtures and deterministic output.

Spec refs: `minimal_crate.md`, `harness.md`.

Completion: validation-mode CLI parsing, strict/permissive unknown-root policy,
profile metadata retention/filtering, plan CLI fixtures, expectation-schema
fixtures, and harness documentation sync.

### Task 4: Snapshot API and canonicalization

Purpose: introduce the general snapshot module and canonical record API.

Spec refs: `snapshot.md`.

Completion: general in-memory `SnapshotRecord` API, canonical hash input,
line-ending normalization, local absolute path validation, deterministic
profile metadata ordering, and comparison diffs.

### Task 5: Snapshot update policy and determinism checks

Purpose: implement explicit update flow and repeat-render determinism checks.

Spec refs: `snapshot.md`.

Completion: explicit `VerifyOnly` versus `Update { reason }` baseline helper,
safe `.snap` baseline path validation, create/update round-trips, no-write
verify failures, and repeat-render determinism mismatch reports.

### Task 6: Coverage and pass/fail-mix reporting

Purpose: compute/report trace coverage, stored status drift, and corpus
pass/fail mix from discovery and traceability data.

Spec refs: `traceability.md`, architecture test strategy.

Completion: per-requirement and per-stage coverage reports, mode-aware
coverage/status gates, manifest id-order validation, obsolete-reference
diagnostics, link-validator regression fixtures, and deterministic CLI
pass/fail mix output.

### Task 7: Stage-prerequisite validation

Purpose: prevent higher-stage coverage credit before prerequisites are covered,
declared built-in, or accepted through `depends_on`.

Spec refs: `staged_model.md`, `traceability.md`.

Completion: manifest `depends_on` validation, `built_in` declarations,
stage-mismatch diagnostics, unsatisfied-prerequisite diagnostics, and
coverage-credit withholding before mode-aware completeness checks.

### Task 8: Fail/soundness contract support

Purpose: implement fail/soundness required-domain bookkeeping, typed failure
identity checks, false-arithmetic coverage, and weakening/deletion regression
rules.

Spec refs: `fail_soundness.md`.

### Task 9: Corpus size and review-rule validation

Purpose: enforce corpus naming, size, class placement, generation metadata,
corpus-policy profile constraints, stress exclusion, and fuzz provenance rules.

Spec refs: `miz_corpus.md`.

### Task 10: Consumer-runner support

Purpose: keep discovery, expectations, stages, snapshots, active/planned gates,
and reporting synchronized with each consumer runner as it lands.

Spec refs: `harness.md`.

### Task 11: Determinism suite

Purpose: verify byte-stable discovery, plans, diagnostics, reports, snapshot
comparisons, and future parallel-equivalence surfaces.

Spec refs: `harness.md`.

### Task 12: Public-enum forward-compatibility policy

Purpose: decide and document public enum exhaustiveness/non-exhaustiveness for
the crate API.

Spec refs: all module specs.

### Task 13: Bilingual documentation sync audit

Purpose: synchronize every English canonical document with its Japanese
companion.

Spec refs: repository documentation policy.

### Task 14: Incremental/parallel verification regression matrix

Purpose: add metadata/reporting support for architecture-22 regression matrix
scenarios while keeping execution in consumer crates.

Spec refs: architecture test strategy and incremental verification contract.

### Task 15: Architecture-22 follow-up audit

Purpose: re-run source/spec and bilingual documentation audits after task 14
and record any remaining matrix gaps.

Spec refs: architecture test strategy, incremental verification contract, and
repository documentation policy.

## Exit Criteria

Hard gates:

- no blocking `spec_gap`, accepted `repo_metadata_conflict`, or language
  behavior change remains hidden;
- no source behavior remains undocumented at crate exit;
- all task-owned gaps are fixed or explicitly deferred with a reason;
- `doc/design/mizar-test/en/` and `doc/design/mizar-test/ja/` stay
  synchronized;

  future user request explicitly changes that intent.
