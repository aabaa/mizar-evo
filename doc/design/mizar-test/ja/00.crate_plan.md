# Crate Plan: mizar-test

> 正本は英語です。英語版: [../en/00.crate_plan.md](../en/00.crate_plan.md)。

## 責務

所有する振る舞い:

- corpus payload と `.expect.toml` sidecar の決定的 discovery
- expectation metadata と traceability metadata の parse / validation
- staged-model admission と coverage accounting
- stage-owned compiler seam の active runner gate
- snapshot comparison/update policy と決定的 report surface
- 黙って pass、削除、弱体化してはならない fail/soundness case の bookkeeping

対象外:

- compiler language semantics
- 明示的な consumer-runner seam 以外での name resolution、type checking、
  proof search、ATP、certificate replay、kernel proof authority
- current implementation に合わせる目的での `doc/spec`、既存 `.miz`、
  既存 expectation の変更

## タスク分解

### Task 1: Lint-policy guard

Purpose: `mizar-test` を workspace lint に opt-in させ、lint exception の
根拠を必須にする。

Completion: `0b24d1a` として commit 済み。

### Task 2: Source/spec gap audit and status sync

Purpose: audit baseline を記録し、drift/gap を分類し、module status を同期する。

Completion: この crate plan と `todo.md` の status/follow-up updates。

Forbidden behavior: この audit commit で `doc/spec`、既存 `.miz`、既存
expectation を修復すること。

### Task 3: Runner modes and CLI completion

Purpose: corpus と coverage metadata に対する validation-mode CLI behavior、
exit-code fixtures、deterministic output を完成させる。

Spec refs: `minimal_crate.md`, `harness.md`.

Completion: validation-mode CLI parsing、strict/permissive unknown-root
policy、profile metadata retention/filtering、plan CLI fixtures、
expectation-schema fixtures、harness documentation sync。

### Task 4: Snapshot API and canonicalization

Purpose: general snapshot module と canonical record API を導入する。

Spec refs: `snapshot.md`.

Completion: general in-memory `SnapshotRecord` API、canonical hash input、
line-ending normalization、local absolute path validation、deterministic
profile metadata ordering、comparison diffs。

### Task 5: Snapshot update policy and determinism checks

Purpose: explicit update flow と repeat-render determinism checks を実装する。

Spec refs: `snapshot.md`.

Completion: explicit `VerifyOnly` versus `Update { reason }` baseline helper、
safe `.snap` baseline path validation、create/update round-trips、no-write
verify failures、repeat-render determinism mismatch reports。

### Task 6: Coverage and pass/fail-mix reporting

Purpose: discovery と traceability data から trace coverage、stored status drift、
corpus pass/fail mix を compute/report する。

Spec refs: `traceability.md`, architecture test strategy.

Completion: per-requirement/per-stage coverage report、mode-aware
coverage/status gate、manifest id-order validation、obsolete-reference
diagnostics、link-validator regression fixtures、deterministic CLI pass/fail
mix output。

### Task 7: Stage-prerequisite validation

Purpose: prerequisites が covered、built-in 宣言済み、または `depends_on` により
accept されるまで higher-stage coverage credit を与えない。

Spec refs: `staged_model.md`, `traceability.md`.

### Task 8: Fail/soundness contract support

Purpose: fail/soundness required-domain bookkeeping、typed failure identity
checks、false-arithmetic coverage、weakening/deletion regression rules を実装する。

Spec refs: `fail_soundness.md`.

### Task 9: Corpus size and review-rule validation

Purpose: corpus naming、size、class placement、generation metadata、
corpus-policy profile constraints、stress exclusion、fuzz provenance rules を
強制する。

Spec refs: `miz_corpus.md`.

### Task 10: Consumer-runner support

Purpose: consumer runner の着地ごとに discovery、expectations、stages、
snapshots、active/planned gates、reporting を同期する。

Spec refs: `harness.md`.

### Task 11: Determinism suite

Purpose: discovery、plans、diagnostics、reports、snapshot comparisons、将来の
parallel-equivalence surface が byte-stable であることを検証する。

Spec refs: `harness.md`.

### Task 12: Public-enum forward-compatibility policy

Purpose: crate API の public enum exhaustiveness/non-exhaustiveness を決め、
文書化する。

Spec refs: all module specs.

### Task 13: Bilingual documentation sync audit

Purpose: すべての English canonical document と Japanese companion を同期する。

Spec refs: repository documentation policy.

### Task 14: Incremental/parallel verification regression matrix

Purpose: architecture-22 regression matrix scenarios の metadata/reporting
support を追加し、実行は consumer crate に置いたままにする。

Spec refs: architecture test strategy and incremental verification contract.

### Task 15: Architecture-22 follow-up audit

Purpose: task 14 後に source/spec と bilingual documentation audit を再実行し、
残る matrix gap を記録する。

Spec refs: architecture test strategy、incremental verification contract、
repository documentation policy.

## Exit Criteria

Hard gates:

- blocking な `spec_gap`、採用すべき `repo_metadata_conflict`、language
  behavior change が隠れたまま残らない
- crate exit 時点で source behavior が undocumented のまま残らない
- task-owned gap は修正済み、または理由付きで明示的に deferred
- `doc/design/mizar-test/en/` と `doc/design/mizar-test/ja/` が同期している
- 将来の user request が明示的に意図を変えない限り、`doc/spec`、既存
  `.miz`、既存 expectation は触らない

Verification commands:

- `cargo test -p mizar-test`
- `cargo clippy -p mizar-test --all-targets -- -D warnings`
- `cargo fmt --check`

Review expectations:

- task-specific spec/docs review
- test sufficiency review
- implementation review
- source/documentation consistency review
