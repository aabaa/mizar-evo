# Crate Plan: mizar-test

> 正本は英語です。英語版: [../en/00.crate_plan.md](../en/00.crate_plan.md)。

## 責務

所有する振る舞い:

- corpus payload と `.expect.toml` sidecar の決定的 discovery
- expectation metadata と traceability metadata の parse / validation
- staged-model admission と coverage accounting
- stage-owned compiler seam の active runner gate
- snapshot comparison/update policy と決定的 report surface
- 黙って pass、削除、弱体化してはならない fail/soundness case の bookkeeping

対象外:

- compiler language semantics
- 明示的な consumer-runner seam 以外での name resolution、type checking、
  proof search、ATP、certificate replay、kernel proof authority
- current implementation に合わせる目的での `doc/spec`、既存 `.miz`、
  既存 expectation の変更

  duplicate id checks、missing source/sidecar checks、deterministic plan ordering
  は実装・テスト済み
- active `parse-only`、`declaration-symbol`、`type-elaboration` runner は
  explicit active-tag gate と repository coverage を持つ
- active `type-elaboration` runner は対応済み source-derived reserve declaration
  slice について checker-owned `BindingEnv`、reserve `DeclarationInput`、
  binding 固有の builtin `set` / `object` `TypeExpressionInput`、
  `DeclarationChecker`、`TypedAst`、`ResolvedTypedAst` assembly まで到達し、
  その real payload を `mizar-core` の `ResolvedTypedAstSummary::from_ast` で読み、
  同じ real reserve binding から binder-only `CoreContext` を準備し、
  未対応の semantic / CoreIr / ControlFlowIr / VC / proof payload family は
  external-gap key のままにする
- architecture-22 matrix sidecar metadata は validate / report され、owning consumer が
  active clean/incremental/parallel/cache-race execution を提供するまで planned-only
  scenario coverage として扱われる
- transitional active parse-only `SurfaceAst` snapshot text baseline は
  validate / compare される
- general snapshot records、update mode、determinism mode、coverage reporting、
  pass/fail mix reporting、manifest ordering、obsolete-reference checks は実装済み
- stage prerequisite validation は実装済み
- fail/soundness metadata bookkeeping は certificate/kernel `rejection_reason`、
  認識済み required soundness case の shape/profile/phase、mode-aware な missing-case
  diagnostic を検証する。active proof/certificate/kernel execution は将来の
  consumer runner に律速される。
