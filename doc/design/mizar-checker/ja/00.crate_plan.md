## 責務

所有する挙動:

- pipeline phase 6-8、すなわち type checking、registration/cluster
  resolution、overload resolution を実装する。
- `mizar_resolve::resolved_ast::ResolvedAst` と
  `mizar_resolve::env::SymbolEnv` を消費し、`TypedAst`、
  `TypeFactTable`、`ResolutionTrace`、`ResolvedTypedAst` を生成する。
- soft type expression を決定的な predicate 風の正規形へ正規化する。
- resolver-owned symbol identity の上に checker-local な階層型 typing
  context を構築する。
- recoverable error state を保持しながら term と formula の型を推論する。
- type fact、coercion candidate、`InitialObligationId` を持つ initial
  obligation を記録し、`VcId` は割り当てない。
- pending / activated registration を索引し、再生可能な cluster closure を
  計算し、来歴付きで reduction registration を適用し、有界飽和失敗を検出する。
- overload candidate を展開・filter し、ordinary root を選択し、受理済み
  same-root refinement を join し、許可された `qua` view を挿入し、失敗 site を
  保持する。
- id、debug rendering、diagnostics、fact query、candidate order、trace order、
  artifact-facing projection を決定的に保つ。

対象外:

- lexical、syntax、parser、frontend、module-index、source-loading、package
  planning の所有。
- `mizar-resolve` が所有する name lookup、import/export validation、label
  resolution、signature collection。
- 任意の一階 reasoning、ATP search、proof certificate acceptance、theorem
  validity checking、kernel replay。
- `VcId` assignment、`ObligationAnchor` construction、VC generation、checker-local
  initial obligation の発行を超える proof policy decision。
- stable published artifact schema、cache storage、LSP rendering、build scheduling。
- 同じ source pass で先に現れたという理由だけで unverified local registration を
  活性化すること。

境界規則:

- checker fact はすべて declared type predicate、checker-owned inference fact、
  または trace provenance を持つ registration-derived fact として説明可能であること。
- source behavior は観測した source convenience ではなく `doc/spec/en/` と既存
  `.miz` test intent に従うこと。
- recovered または erroneous node には partial typed output を与えてよいが、
  invalid fact を verified metadata として export してはならない。
- automatic inference に使う active registration は、accepted dependency artifact
  または accepted local summary から来なければならない。暫定 policy は
  pending / unverified registration と generated obligation を記録してよいが、
  unverified registration を automatic fact に寄与させてはならない。
- proof search、hidden coercion insertion、未記録 cluster fact、return-type-based
  overload selection はどの task でも導入しない。

## crate completion and exit criteria

Hard gate:

- [todo.md](./todo.md) の non-deferred task がすべて 1 task = 1 commit で commit されている。
- blocking/high specification inconsistency が残っていない。
- `doc/spec/en` と tests に存在しない implemented source behavior が残っていない。
  まだ仕様化・テスト化できない behavior は milestone scope から除外するか、
  deferred design item として未実装のままにする。
- milestone-owned specification item には `.miz` coverage、task-local Rust test、
  または explicit deferred reason がある。
- 既存 `.miz` test と expectation は implementation behavior に合わせるためだけには変更されていない。
- design docs、英日 companion、source が同期している。
- checker crate boundary が侵害されていない。
- required verification command が通過しているか、未実行の明示理由がある。
- hard gate 通過後、read-only crate quality review が 90/100 以上を与えている。

Verification commands:

```text
cargo fmt --check
cargo clippy -p mizar-checker --all-targets -- -D warnings
cargo test -p mizar-checker
cargo clippy --all-targets --all-features -- -D warnings
cargo test
```

resolver boundary、corpus、shared harness に触れる task では [todo.md](./todo.md) に
従って追加 command を実行する。

Crate exit report:

- crate closeout で [crate_exit_report.md](./crate_exit_report.md) と英語正本を
  追加した。
- task commit、review result、verification、deferred item、external dependency
  gap、quality score、next-crate handoff を含む。
