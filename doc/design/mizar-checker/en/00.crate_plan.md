## Responsibility

Owned behavior:

- implement pipeline phases 6-8: type checking, registration/cluster
  resolution, and overload resolution;
- consume `mizar_resolve::resolved_ast::ResolvedAst` plus
  `mizar_resolve::env::SymbolEnv` and produce `TypedAst`,
  `TypeFactTable`, `ResolutionTrace`, and `ResolvedTypedAst`;
- normalize soft type expressions into deterministic predicate-like forms;
- build layered checker-local typing contexts over resolver-owned symbol
  identities;
- infer term and formula types while preserving recoverable error state;
- record type facts, coercion candidates, and `InitialObligationId`-backed
  initial obligations without assigning `VcId`s;
- index pending and activated registrations, compute replayable cluster
  closure, apply reduction registrations with provenance, and detect bounded
  saturation failures;
- expand and filter overload candidates, select ordinary roots, join accepted
  same-root refinements, insert allowed `qua` views, and preserve failed sites;
- keep ids, debug renderings, diagnostics, fact queries, candidate order, trace
  order, and artifact-facing projections deterministic.

Out of scope:

- lexical, syntax, parser, frontend, module-index, source-loading, and package
  planning ownership;
- name lookup, import/export validation, label resolution, and signature
  collection, which remain owned by `mizar-resolve`;
- arbitrary first-order reasoning, ATP search, proof certificate acceptance,
  theorem validity checking, and kernel replay;
- `VcId` assignment, `ObligationAnchor` construction, VC generation, and proof
  policy decisions beyond emitting checker-local initial obligations;
- stable published artifact schemas, cache storage, LSP rendering, and build
  scheduling;
- activating unverified local registrations merely because they occur earlier
  in the same source pass.

Boundary rules:

- every checker fact must be explainable as a declared type predicate,
  checker-owned inference fact, or registration-derived fact with trace
  provenance;
- source behavior must follow `doc/spec/en/` and existing `.miz` test intent,
  not observed source convenience;
- recovered or erroneous nodes may receive partial typed output, but invalid
  facts must not be exported as verified metadata;
- active registrations used for automatic inference must come from accepted
  dependency artifacts or accepted local summaries; an interim policy may
  record pending/unverified registrations and generated obligations, but it
  must not make unverified registrations contribute automatic facts;
- no task may introduce proof search, hidden coercion insertion, unrecorded
  cluster facts, or return-type-based overload selection.

  with one task per commit;
- no blocking/high specification inconsistency remains;

cargo test -p mizar-checker

harness tasks as noted in [todo.md](./todo.md).
