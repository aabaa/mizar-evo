# Crate Plan: mizar-lexer

> Canonical language: English. Japanese companion: [../ja/00.crate_plan.md](../ja/00.crate_plan.md).

This plan records the completed `mizar-lexer` crate against the autonomous
crate-development protocol. It is retrospective because the crate was
implemented before `doc/design/autonomous_crate_development.md` became the
standard crate-wide workflow.

## Responsibility

Owned behavior:

- source-text loading boundary helpers for UTF-8 validation, leading BOM
  stripping, CRLF-to-LF normalization, and minimal loading maps used by tests
  and early integration;
- lexical preprocessing for comment removal, comment trivia retention,
  lexical-text/source offset maps, and lexer-boundary malformed text
  diagnostics;
- LF-only raw scanning into source-span-preserving raw tokens;
- reserved-word, reserved-symbol, identifier, numeral, layout, and
  user-symbol-shape helpers;
- import-prelude shape extraction from raw tokens without resolving imports;
- active lexical environment construction from already resolved imports and
  module lexical summaries;
- conservative scope skeletons needed for lexical override decisions;
- context-sensitive final-token disambiguation from raw tokens, lexical
  environment, parser lexical context, and scope view.

Out of scope:

- filesystem traversal, package-root discovery, symlink/case policy, source
  snapshots, and production source identity;
- module resolution, import graph ownership, visibility checks, exported symbol
  discovery, and full module IR loading;
- authoritative parsing, syntax acceptance, name resolution, type checking,
  overload resolution, proof semantics, theorem validity, and VC generation;
- human-facing diagnostic rendering, tab expansion, one-based display columns,
  and LSP UTF-16 coordinate conversion.

## Relevant Tests

The executable contract for `mizar-lexer` is split between committed lexical
fixtures, expectation metadata, property/fuzz anchors, and crate-local Rust
tests.

The autonomous protocol prefers `.miz` tests for human-reviewed language
behavior. For this crate, the lexical fixture corpus under `tests/lexical`
serves as the human-reviewable executable intent because many lexer phases
operate before a complete `.miz` file is syntactically meaningful. `.miz`
coverage remains parser/frontend-owned and is deferred unless a future lexer
change exposes behavior that can only be reviewed through complete source
files.

| Test path | Intent | Spec refs |
|---|---|---|
| `tests/lexical/pass/*.src` with adjacent `.expect.toml` | Human-readable positive lexical fixtures for identifiers, layout, raw scanning, reserved tables, import pre-scan, scope skeletons, dot disambiguation, string contexts, user symbols, and final token spans. | `spec.en.02.lexical.*`, `spec.en.04.*`, `spec.en.12.*`, and `spec.en.16.*` entries referenced by lexical sidecars in `tests/coverage/spec_trace.toml`. |
| `tests/lexical/fail/*.src` with adjacent `.expect.toml` | Human-readable negative lexical fixtures for malformed raw input, malformed imports, rejected parser contexts, invalid string escapes, unsupported annotation markers, and recoverable scope skeleton diagnostics. | Lexical fail sidecars reference the same lexical, binder, import-prelude, and proof/block traceability families where applicable. |
| `tests/property/*.fixture.toml` and `.expect.toml` | Metadata anchors for property-style coverage of Phase 7, lexical environment, active-lexicon symbol management, and scope skeleton behavior. | Property, `spec.en.11.*`, and design-derived traceability entries in `tests/coverage/spec_trace.toml`. |
| `tests/fuzz/lexer_phase7_fuzz_handoff_001.*` | Metadata anchor for minimized fuzz regression handoff. | `spec.en.02.lexical.phase7.fuzz_handoff`, `spec.en.02.lexical.raw.span_preservation`. |
| `crates/mizar-lexer/src/tests/*.rs` | Crate-local unit coverage for raw scanning, source loading/preprocessing/maps, import pre-scan, lexical environments, disambiguation, scope skeletons, and Phase 7 invariants. | Module design docs and traceability entries. |
| `crates/mizar-lexer/tests/lexical_corpus.rs` | Integration runner that checks committed lexical fixture expectations. | `tests/lexical` expectation metadata and `tests/coverage/spec_trace.toml`. |
| `crates/mizar-lexer/benches/lexer_pipeline.rs` | Non-gating performance coverage for large `.miz`-like input and active-symbol lookup. | `test_and_implementation_plan.md` Phase 7 and `lexical_environment.md`. |

Planned test-first additions:

- None for the current retrospective migration. Future `.miz` additions should
  be made only when a complete-source lexer/frontend behavior is specified and
  cannot be reviewed adequately through lexical fixtures.

  environment, parser context, and scope view.

  diagnostic messages remain provisional.
- Public enums are non-exhaustive and public transfer-object fields remain
  visible for the `0.1` parser-integration milestone.

## Exit Criteria

Hard gates:

- no blocking/high specification inconsistency remains in current
  `mizar-lexer` scope;
- no lexer-owned source behavior is intentionally left absent from design docs,
  traceability metadata, or tests;
- lexer-owned specification items have lexical fixture/property/Rust coverage,
  and lack of `.miz` coverage is explicitly deferred for this pre-parser crate;
- test expectations are not changed merely to match implementation behavior;
- `doc/design/mizar-lexer` and `crates/mizar-lexer` responsibilities agree;
- parser, resolver, session/source, diagnostics, and LSP boundaries are not
  violated;
- verification commands pass or are explicitly justified;
- residual risks are classified as deferred, out of scope, or human-owned.

Verification commands:

```sh
cargo fmt --check
cargo clippy --all-targets --all-features -- -D warnings
cargo test
cargo run -p mizar-test -- plan --tests-root tests --manifest tests/coverage/spec_trace.toml
```

Review expectations:

- review the English canonical crate plan and exit report first;
- review the Japanese companions for synchronization;
- review `tests/coverage/spec_trace.toml` for the deferred partial selector
  handoff and design-derived trace ids;
- treat any future language behavior change as requiring `doc/spec`, `.miz`
  tests, or traceability metadata before source changes.
