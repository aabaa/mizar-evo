# Crate Plan: mizar-lexer

> 正本は英語です。英語版: [../en/00.crate_plan.md](../en/00.crate_plan.md)。

この計画は、完了済みの `mizar-lexer` crate を autonomous crate-development
protocol に照らして記録するものです。`doc/design/autonomous_crate_development.md`
が crate-wide 標準ワークフローになる前に実装が進んでいたため、これは後追いの
crate plan です。

## 責務

所有する振る舞い:

- テストと初期統合で使う、UTF-8 検証、先頭 BOM 除去、CRLF から LF への正規化、
  最小 loading map のための source-text loading boundary helper;
- コメント除去、comment trivia retention、lexical-text/source offset map、
  lexer-boundary malformed text diagnostic のための lexical preprocessing;
- LF-only 入力から source-span-preserving raw token への raw scanning;
- reserved word、reserved symbol、identifier、numeral、layout、user-symbol-shape
  helper;
- import を解決しない import-prelude shape extraction;
- 解決済み import と module lexical summary からの active lexical environment 構築;
- lexical override decision に必要な conservative scope skeleton;
- raw token、lexical environment、parser lexical context、scope view からの
  context-sensitive final-token disambiguation。

範囲外:

- filesystem traversal、package-root discovery、symlink/case policy、source
  snapshot、本番 source identity;
- module resolution、import graph ownership、visibility check、exported symbol
  discovery、full module IR loading;
- authoritative parsing、syntax acceptance、name resolution、type checking、
  overload resolution、proof semantics、theorem validity、VC generation;
- human-facing diagnostic rendering、tab expansion、one-based display columns、
  LSP UTF-16 coordinate conversion。

## 関連テスト

`mizar-lexer` の executable contract は、committed lexical fixtures、expectation
metadata、property/fuzz anchors、crate-local Rust tests に分かれています。

autonomous protocol は、人間レビュー済み language behavior には `.miz` tests を
優先します。この crate では、多くの lexer phase が complete `.miz` file として
構文的に意味を持つ前に動作するため、`tests/lexical` 以下の lexical fixture corpus
を human-reviewable executable intent として扱います。`.miz` coverage は
parser/frontend-owned とし、将来の lexer change が complete source file 経由でのみ
レビューできる振る舞いを公開するまで defer します。

| Test path | 意図 | Spec refs |

- 現在の retrospective migration ではなし。将来の `.miz` 追加は、complete-source
  lexer/frontend behavior が仕様化され、lexical fixtures だけでは十分にレビューできない
  場合に限ります。

## Exit Criteria

Hard gates:

- 現在の `mizar-lexer` scope に blocking/high specification inconsistency が残らない;
- lexer-owned source behavior が design docs、traceability metadata、tests の外に
  意図的に残らない;
- lexer-owned specification items は lexical fixture/property/Rust coverage を持ち、
  `.miz` coverage がない理由はこの pre-parser crate のため明示的に defer される;
- test expectations は implementation behavior に合わせるためだけには変更されない;
- `doc/design/mizar-lexer` と `crates/mizar-lexer` の responsibilities が一致する;
- parser、resolver、session/source、diagnostics、LSP boundaries を侵害しない;
- verification commands が pass するか、未実行理由を明示する;
- residual risks が deferred、out of scope、human-owned に分類される。

Verification commands:

```sh
cargo fmt --check
cargo clippy --all-targets --all-features -- -D warnings
cargo test
cargo run -p mizar-test -- plan --tests-root tests --manifest tests/coverage/spec_trace.toml
```

Review expectations:

- English canonical crate plan と exit report を最初に review する;
- 日本語 companion の同期を review する;
- `tests/coverage/spec_trace.toml` の deferred partial selector handoff と
  design-derived trace ids を review する;
- 将来の language behavior change は、source changes の前に `doc/spec`、`.miz`
  tests、または traceability metadata を要求する。
