# Crate Plan: mizar-syntax

> 正本は英語です。英語版:
> [../en/00.crate_plan.md](../en/00.crate_plan.md)。

状態: 自律 crate 開発を開始するための初期 kickoff plan。

この計画は、今後の `mizar-syntax` 実装作業を
[autonomous crate-development protocol](../../autonomous_crate_development.md)
に従って進める前に必要な crate scope の入口文書である。現在の境界、仕様と
テスト入力、既知の gap/drift、残りタスク、crate 完了条件を記録する。

## Responsibility

Owned behavior:

- `mizar-parser` が生成する source-shaped `SurfaceAst` 境界を定義する。
- rowan-backed syntax storage、決定的な syntax snapshot、typed 互換 view、
  syntax-kind 語彙を所有する。
- syntax-level diagnostic と recovery node 語彙を所有する。
- comment range、doc-comment attachment hint、skipped-token range、
  whitespace hint の syntax-facing trivia side table を所有する。
- 生の storage を semantic identity contract として露出せず、parser-facing
  builder/accessor API を提供する。

Out of scope:

- lexing、token disambiguation、parser-assisted lexing plan、raw comment
  extraction。これらは `mizar-frontend` / `mizar-lexer` の責務である。
- grammar parsing と synchronization producer。これは `mizar-parser` の責務である。
- name resolution、type checking、overload resolution、cluster fact、proof
  obligation、theorem validity、algorithm VC generation、package/build semantics。
- `salsa` query integration と cache orchestration。
- 安定した外部 artifact schema。

境界規則: `mizar-syntax` は surface syntax と recovery fact を表現してよいが、
resolved symbol id、inferred type、active environment が必要な semantic
category、proof/VC obligation をエンコードしてはならない。

## Task Decomposition

### Task S-009: Module, item, and shared path nodes

Purpose: shared qualified-symbol/module-path surface node、module file shape、
item list、import/export item、visibility syntax を表現する。

Spec refs: [12.modules_and_namespaces.md](../../../spec/en/12.modules_and_namespaces.md),
[appendix A](../../../spec/en/appendix_a.grammar_summary.md).

Tests: parser task 4 の path-helper unit coverage、parser task 5 の module skeleton
coverage、parser task 6 の import item active parse-only coverage、parser task 7 の
export/visibility active parse-only coverage は存在する。syntax snapshot は S-009 の
すべての node role を覆っている。

Affected design files: [ast.md](./ast.md)、doc-comment item attachment について
[trivia.md](./trivia.md)、item-level skipped/missing node について
[recovery.md](./recovery.md)。

Affected source files: `src/ast.rs`、必要に応じて `src/trivia.rs` と
`src/recovery.rs`。

Completion condition: parser tasks 4-7 について、shared path、module/item node
kind、child role、accessor、range、snapshot、corpus case、traceability entry が揃う。

Forbidden behavior: import resolution、visibility semantics、symbol identity、
item validity checking。

### Task S-010: Type expression nodes

Purpose: attribute chain、generic syntactic type head、type argument、`of`/`over`
argument list を表現する。

Spec refs: [03.type_system.md](../../../spec/en/03.type_system.md),
[appendix A](../../../spec/en/appendix_a.grammar_summary.md).

Tests: type-head ambiguity、malformed type argument、positive attribute-chain
case、`of` / `over` term argument list、bracket nested type-expression argument、
bracket `qua_arg` placeholder、tokenization が split を露出する場合の局所
parameter-prefix preservation を activate する。

Affected design files: [ast.md](./ast.md), [recovery.md](./recovery.md).

Affected source files: `src/ast.rs`、必要に応じて `src/recovery.rs`。

Completion condition: 明示的な仕様変更なしに semantic lookup を行わず、
parser task 8 の type expression が radix/mode ambiguity を preserve する。

Forbidden behavior: type head が mode、radix、structure、attribute-bearing
semantic category のどれかを決めること。

Result: paired parser task 8 により完了した。`ReserveItem` と `ReserveSegment` が
syntax-only `TypeExpression` node を host し、`TypeHead` は generic のままにする。
`TypeArguments` は `of` / `over` term argument list を保持し、task 9 以降はそこに
`TermExpression` child を入れる。bracket nested type と `qua_arg` placeholder も保持する。
token が split を露出する局所 `ParameterPrefix` は parser unit test で覆い、
active parse-only pass/fail coverage は `parser.type_fixtures` 経由で着地した。

### Task S-011: Term nodes

Purpose: S-009 の shared qualified-symbol/path vocabulary を消費し、primary
term、choice expression、application、unresolved dot chain、selector/update
surface、`qua`、operator expression、Fraenkel/set-builder form を表現する。

Spec refs: [13.term_expression.md](../../../spec/en/13.term_expression.md),
[appendix B](../../../spec/en/appendix_b.operator_precedence.md).

Tests: dot-chain、choice/update、`qua`、application delimiter、Fraenkel の行を
grammar position の実装に合わせて activate する。

Affected design files: [ast.md](./ast.md), [recovery.md](./recovery.md).

Affected source files: `src/ast.rs`、必要に応じて `src/recovery.rs`。

Completion condition: term node が source shape と operator fact を preserve し、
parser tasks 9-12 と 15 について name resolution を行わない。selector/update
node 固定前に unresolved dot-role shape が文書化される。

Forbidden behavior: selector-versus-namespace semantic resolution、overload
selection、inferred result type、sethood evidence。

Task 9 local classification: primary term が owning parser task になったため、
`of`/`over` と attribute argument list 内の task-8 `TermPlaceholder` child は
`source_drift` として `TermExpression` node に置き換える。
[13.term_expression.md](../../../spec/en/13.term_expression.md) に由来する
primary-term parse-only fixture は `test_gap` である。active user-symbol bracket
delimiter pair は、現在の `ParserInputs` が fixity entry は運ぶが bracket-pair metadata を
持たないため、parser task 12 の `source_drift` として残す。task 9 は reserved `[` / `]`
bracket functor form のみを扱う。`template_functor_application` は task 9 が template
argument surface を所有しないため、parser task 31 / S-016 の deferred `source_drift` として
残す。ambiguous zero-field `S()` constructor は、将来の semantic boundary が structure fact を
渡すまで generic application syntax のまま残す。

Task 9 result: primary-term 増分は S-011 の進捗として実装済みである。
`TermExpression` と現在の primary-term child は reserve-hosted term list から利用できる。
`of` / `over` と attribute argument position は task 8 の term placeholder を使わなくなった。
active parse-only primary-term pass/fail fixture により local `test_gap` は閉じた。
文書化済みの `source_drift` は parser task 11、12、31 / S-016 に明示的に残している。

## Exit Criteria

Hard gates:

- milestone-owned syntax behavior について blocking/high specification
  inconsistency が残っていない。
- `doc/spec/en/`、`.miz` tests、traceability metadata に支えられていない
  language-facing source behavior が残っていない。
- `doc/design/mizar-syntax`、Rust tests、明示的な deferred note に支えられて
  いない implementation-facing API behavior が残っていない。
- milestone-owned specification item に既存または test-first `.miz` coverage が
  あるか、明示的な deferred reason がある。
- 既存の `.miz` tests と `.expect.toml` files を current implementation behavior
  に合わせるためだけに rebaseline していない。
- `doc/design/mizar-syntax` と `crates/mizar-syntax` が target scope 内で一致している。
- parser、lexer、resolver、type、proof、VC の responsibility boundary を守っている。
- 必要な verification command が通るか、未実行 command がすべて明示的に justified されている。
- 残る risk が deferred、out of scope、human-owned として分類されている。
- hard gates 通過後、read-only review が valid crate quality score 90/100 以上を割り当てている。

Verification commands:

```text
cargo fmt --check
cargo clippy --all-targets --all-features -- -D warnings
cargo test
cargo run -p mizar-test -- plan --tests-root tests --manifest tests/coverage/spec_trace.toml
```

個別 increment では先に narrow command を実行してよい:

```text
cargo test -p mizar-syntax
cargo clippy -p mizar-syntax --all-targets --all-features -- -D warnings
cargo test -p mizar-parser
cargo test -p mizar-frontend
```

Review expectations:

- 各 increment の前に implementation specification を review する。
- この crate plan と owning spec refs に照らして test sufficiency を review する。
- full implementation を bug、boundary violation、missing edge case、drift の観点で review する。
- commit 前に source/documentation consistency を review する。
- crate-wide autonomous work 完了時に Crate Exit Report を作成する。
