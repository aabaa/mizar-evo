`SurfaceNodeKind`、stable snapshot spelling、raw-kind conversion、node-kind
classification、rowan projection、typed `as_operator_declaration` accessorをappendする。
以前のdiscriminantはrenumberしない。このincrementはsource syntaxだけを表し、activation、
fixity mutation、resolution、precedence semanticsは`mizar-syntax`外である。syntax unit
testはraw/surface/snapshot/rowan/accessor contractをcoverし、parser active pass/fail
pairはconstructionとrecoveryをcoverする。

状態: 自律 crate 開発計画。crate exit evidence は S-020 bilingual documentation

この計画は、今後の `mizar-syntax` 実装作業を
[autonomous crate-development protocol](../../autonomous_crate_development.md)
に従って進める前に必要な crate scope の入口文書である。現在の境界、仕様と
テスト入力、既知の gap/drift、残りタスク、crate 完了条件を記録する。

## Responsibility

Owned behavior:

- `mizar-parser` が生成する source-shaped `SurfaceAst` 境界を定義する。
- rowan-backed syntax storage、決定的な syntax snapshot、typed 互換 view、
  syntax-kind 語彙を所有する。
- syntax-level diagnostic と recovery node 語彙を所有する。
- comment range、doc-comment attachment hint、skipped-token range、
  whitespace hint の syntax-facing trivia side table を所有する。
- 生の storage を semantic identity contract として露出せず、parser-facing
  builder/accessor API を提供する。

Out of scope:

- lexing、token disambiguation、parser-assisted lexing plan、raw comment
  extraction。これらは `mizar-frontend` / `mizar-lexer` の責務である。
- grammar parsing と synchronization producer。これは `mizar-parser` の責務である。
- name resolution、type checking、overload resolution、cluster fact、proof
  obligation、theorem validity、algorithm VC generation、package/build semantics。
- `salsa` query integration と cache orchestration。
- 安定した外部 artifact schema。

境界規則: `mizar-syntax` は surface syntax と recovery fact を表現してよいが、
resolved symbol id、inferred type、active environment が必要な semantic
category、proof/VC obligation をエンコードしてはならない。

## Exit Criteria

Hard gates:

- milestone-owned syntax behavior について blocking/high specification
  inconsistency が残っていない。
- `doc/spec/en/`、`.miz` tests、traceability metadata に支えられていない
  language-facing source behavior が残っていない。
- `doc/design/mizar-syntax`、Rust tests、明示的な deferred note に支えられて
  いない implementation-facing API behavior が残っていない。
- milestone-owned specification item に既存または test-first `.miz` coverage が
  あるか、明示的な deferred reason がある。
- 既存の `.miz` tests と `.expect.toml` files を current implementation behavior
  に合わせるためだけに rebaseline していない。
- `doc/design/mizar-syntax` と `crates/mizar-syntax` が target scope 内で一致している。
- parser、lexer、resolver、type、proof、VC の responsibility boundary を守っている。
- 必要な verification command が通るか、未実行 command がすべて明示的に justified されている。
- 残る risk が deferred、out of scope、human-owned として分類されている。
- hard gates 通過後、read-only review が valid crate quality score 90/100 以上を割り当てている。

Verification commands:

```text
cargo fmt --check
cargo clippy --all-targets --all-features -- -D warnings
cargo test
cargo run -p mizar-test -- plan --tests-root tests --manifest tests/coverage/spec_trace.toml
```

個別 increment では先に narrow command を実行してよい:

```text
cargo test -p mizar-syntax
cargo clippy -p mizar-syntax --all-targets --all-features -- -D warnings
cargo test -p mizar-parser
cargo test -p mizar-frontend
```

Review expectations:

- 各 increment の前に implementation specification を review する。
- この crate plan と owning spec refs に照らして test sufficiency を review する。
- full implementation を bug、boundary violation、missing edge case、drift の観点で review する。
- commit 前に source/documentation consistency を review する。
- crate-wide autonomous work 完了時に Crate Exit Report を作成する。
