## 責務

所有する挙動:

- deterministic discharge 後の `VcStatus::NeedsAtp` VC obligation に対する phase 13 の
  ATP candidate production。
- backend-neutral な `AtpProblem` data、決定的 encoding、backend process execution、
  portfolio candidate collection。
- backend-visible な各 formula を VC/kernel evidence input に結び付ける provenance。
- 後続の kernel checking 向け untrusted formula/substitution evidence candidate。
- backend run の diagnostic と reproducibility metadata。

範囲外:

- trusted proof acceptance、kernel evidence checking、SAT checking、proof policy decision。
- winner selection、proof-status projection、witness publication、proof reuse policy。
  これらは downstream proof/cache/artifact integration に属する。
- premise invention、substitution invention、overload resolution、cluster search、
  implicit coercion insertion、fallback inference。
- backend proof method、resolution trace、SMT proof object、backend log、instantiated
  formula を trusted acceptance material として扱うこと。

## Exit Criteria

Hard gates:

- paired English/Japanese module spec のない semantic ATP source module が存在しない。
- crate-owned behavior は実装前に module spec、`doc/spec`、test、traceability metadata
  のいずれかに記録されている。
- generated evidence は `mizar-kernel` が検査するまで untrusted のままである。
- backend proof method、log、resolution trace、instantiated formula、SAT problem は
  trusted acceptance material にならない。
- `mizar-atp` は proof policy、winner selection、proof witness publication、cache
  promotion を所有しない。
- 完了した各 task が独立した commit を持つ。
- 残る gap は `deferred`、`external_dependency_gap`、または human-owned に分類されている。

Verification commands:

- Rust source task ごと:
  - `cargo test -p mizar-atp`
  - `cargo clippy -p mizar-atp --all-targets -- -D warnings`
  - `cargo fmt --check`
- VC/kernel boundary に触れる task:
  - `cargo test -p mizar-vc`
  - `cargo test -p mizar-kernel`
- 広い crate completion:
  - `cargo clippy --all-targets --all-features -- -D warnings`
  - `cargo test`

Review expectations:

- implementation specification と documentation の review-only agent。
- test sufficiency の review-only agent。
- full implementation の review-only agent。
- source/documentation consistency の review-only agent。
