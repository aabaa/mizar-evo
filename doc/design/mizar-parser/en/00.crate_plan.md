# Crate Plan: mizar-parser

> Canonical language: English. Japanese companion:
> [../ja/00.crate_plan.md](../ja/00.crate_plan.md).

## Responsibility

`mizar-parser` consumes frontend tokens and emits parser-owned syntax events,
recovery diagnostics, and a `mizar-syntax` surface tree. It owns concrete
grammar recognition, source-order preservation, and parser recovery at grammar
boundaries.

It does not own name resolution, type checking, proof acceptance, semantic
evidence, CoreIr, ControlFlowIr, VC generation, or public diagnostic-code
allocation outside the existing parser diagnostic vocabulary.

## Task Decomposition

### Task 47: `reconsider` justification parser alignment

Purpose: close only P-265-47A/B/C.

Authority slice: English Chapters 4 §4.4.2, 8 §8.2.2, 15 §§15.5.1/15.8.2/
15.12, Appendix A §§A.4/A.15; exact deferred trace rows; existing `.miz`
controls.

Consumer and visibility: the existing private `parse_reconsider_statement_at`
producer and existing parse-only runner. Reuse current public syntax node kinds
and typed accessors unchanged.

Required behavior:

- `reconsider ... as T;` produces a `ReconsiderStatement` with no
  justification child and no parser diagnostic.
- `reconsider ... as T by refs;` retains one `JustificationClause`.
- `reconsider ... as T proof ... end;` retains one `ProofBlock`, including its
  existing missing-`end` recovery behavior. `ProofBlock` owns `proof`, its
  reasoning body, and `end`; `ReconsiderStatement` owns the final semicolon.
- bare/equated item parsing, target type ownership, source order, ranges,
  `then` wrapping, explicit-simple behavior, and malformed item/type recovery
  remain unchanged.
- `consider` remains a mandatory-simple-justification host.

Tests:

- add the planned test-first pass fixture and sidecar;
- add exact unit coverage for all three tail forms, AST child ownership,
  omission without recovery, proof-block missing-`end` recovery, and unchanged
  `consider`/malformed item behavior;
- remove only `reconsider` from any existing unit-test list of hosts that reject
  proof blocks, leaving every genuinely simple-only host guarded;
- update the existing mixed recovery expectation and unit counts only for the
  obsolete omitted-tail error;
- activate exactly the two Task-47 trace rows and add the new pass case to the
  general Chapter-15 statement row.
