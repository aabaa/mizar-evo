# 内部設計: IR ストレージとスナップショットハンドル

> 正本は英語です。英語版: [../en/06.ir_storage_and_snapshot_handles.md](../en/06.ir_storage_and_snapshot_handles.md)。

## 目的

この文書は、内部 IR 出力を、ビルドスナップショットをまたいでどのように保存・識別・共有・無効化・キャッシュ・射影するかを定義する。

ここでは、アーキテクチャの IR 階層化規則を、実装に近いハンドルとライフタイム契約へと詳細化する。IR 値は、コンパイラ内部の不変なスナップショットである。フェーズタスクは、可変な IR オブジェクトではなく、それらのスナップショットへのハンドルをやり取りする。公開アーティファクトは安定した外部射影であり続け、生の IR ダンプではない。

## 責務

### IR ストレージサービス

`IrStorageService` は、ワークスペースのビルドマネージャにおける、コンパイラ内部のフェーズ出力ストレージを管理する。

責務:

- フェーズ出力のために不変のストレージスロットを割り当てる。
- フェーズ出力が封印された後、型付きの `PhaseOutputRef<T>` ハンドルを返す。
- 大きな出力を、メモリ上、またはコンテンツアドレス方式の内部 blob に保存する。
- スナップショットの所有関係とライフタイム規則を強制する。
- ウォッチ／LSP スナップショット、診断、説明要求が参照している間、ビルドセッションをまたいで出力を保持する。
- 依存タスク、LSP スナップショット、診断、キャッシュ書き込み手が不要になった、参照されない出力を破棄する。

公開アーティファクトのシリアライズや、キャッシュ互換性の判断は担わない。

### スナップショットハンドルレジストリ

`SnapshotHandleRegistry` は、ソース・ビルド・フェーズ出力の同一性を、安定したハンドルへ対応付ける。

責務:

- 厳密なソース／依存／ツールチェイン／設定の状態に `BuildSnapshotId` 値を割り当てる。
- 1 つのスナップショット内で `SourceId`、`ModuleId`、`ItemId`、`ExprId`、`VcId` などの同一性を割り当てる。
- フェーズ出力間の親／派生の関係を記録する。
- キャッシュ検証なしに、互換性のないスナップショット間でハンドルが再利用されることを防ぐ。

### フェーズ出力パブリッシャー

`PhaseOutputPublisher` は、フェーズサービスが出力値を封印するための、限定された API である。

責務:

- 出力が、タスクのスナップショットと作業単位に属することを検証する。
- キャッシュのために、正準的な内部エンコーディングでコンテンツハッシュを計算する。
- ソースマップと診断のサイドテーブルを付与する。
- 不変のハンドルを依存タスクへ公開する。

構築途中の IR 値を、他のタスクへ見せてはならない。

### IR キャッシュアダプタ

`IrCacheAdapter` は、内部のフェーズ出力とキャッシュレコードとの相互変換を担う。

責務:

- どのフェーズ出力がキャッシュ可能かを決める。
- スキーマバージョンと依存サマリを付けてキャッシュレコードをシリアライズする。
- `PhaseOutputRef<T>` ハンドルを再構築する前に、キャッシュヒットを検証する。
- キャッシュヒットを証明の根拠として扱うことを拒否する。

キャッシュレコードは、生の内部 IR エンコーディングを含んでよい。公開アーティファクトは含んではならない。

### アーティファクト射影の境界

`ArtifactProjectionService` は、封印されたフェーズ出力を読み、フェーズ 15 用の `VerifiedArtifactDraft` 値を作る。

内部 IR から、安定した外部メタデータを射影する責務:

- エクスポートされたシンボル、ラベル、ソース範囲
- レンダリングまたは正規化されたシグネチャ
- 証明ステータスとウィットネス参照
- 診断と説明参照
- 下流ツールが必要とするドキュメントおよび LSP メタデータ

生の `SurfaceAst`、`TypedAst`、`CoreIr`、`ControlFlowIr`、`VcIr`、`AtpProblem`、カーネル内部の状態を公開してはならない。

## データモデル

### スナップショットの同一性

```rust
struct BuildSnapshot {
    id: BuildSnapshotId,
    workspace_root: PathBuf,
    source_versions: Vec<SourceVersion>,
    dependency_artifacts: Vec<DependencyArtifactRef>,
    lockfile_hash: Hash,
    toolchain: ToolchainInfo,
    verifier_config_hash: Hash,
}

struct SourceVersion {
    source_id: SourceId,
    package_id: PackageId,
    module_path: ModulePath,
    normalized_path: PathBuf,
    source_hash: Hash,
    edition: Edition,
}
```

`BuildSnapshotId` は、1 つのファイルのソーステキストだけでなく、ビルド入力の状態全体を識別する。あるスナップショットから派生したハンドルは、キャッシュアダプタが等価性を検証しない限り、別のスナップショットの現在の結果として使うことはできない。

### フェーズ出力参照

```rust
struct PhaseOutputRef<T> {
    id: PhaseOutputId,
    phase: PipelinePhase,
    unit: WorkUnit,
    snapshot: BuildSnapshotId,
    content_hash: Hash,
    side_table_hash: Hash,
    storage: OutputStorageRef<T>,
    side_tables: SideTableRef,
    lineage: PhaseOutputLineage,
}

struct PhaseOutputLineage {
    producer_task: TaskId,
    input_hashes: Vec<NamedHash>,
    dependency_outputs: Vec<PhaseOutputId>,
}
```

`PhaseOutputRef<T>` は、公開後に不変となる。型パラメータは、期待される出力種別をコンパイル時に示す文書であり、動的なストレージもスキーマバージョンと実行時の種別タグを記録する。`side_tables` は、その出力に関連するソースマップ、診断、説明参照、ドキュメント付与メタデータへの、安定したハンドルである。

### ストレージ参照

```rust
enum OutputStorageRef<T> {
    InMemory {
        arena: ArenaId,
        slot: OutputSlotId,
        marker: PhantomData<T>,
    },
    Blob {
        blob: ContentBlobRef,
        schema_version: SchemaVersion,
        marker: PhantomData<T>,
    },
    CacheRecord {
        record: CacheRecordRef,
        output_hash: Hash,
        marker: PhantomData<T>,
    },
}
```

ストレージ参照は公開パスではない。プロセスメモリ、一時ファイル、またはソースレベルの意味論を変えずに削除できる内部キャッシュ blob を指してよい。

### IR 同一性テーブル

```rust
struct IrIdentityTables {
    source_ids: IdTable<SourceId, SourceIdentity>,
    module_ids: IdTable<ModuleId, ModuleIdentity>,
    item_ids: IdTable<ItemId, ItemIdentity>,
    expr_ids: IdTable<ExprId, ExprIdentity>,
    vc_ids: IdTable<VcId, VcIdentity>,
}
```

同一性テーブルは `BuildSnapshotId` にスコープされる。安定した公開同一性はアーティファクトスキーマを通じて射影される。内部 ID は互換性の約束ではない。

### ソースマップとサイドテーブル

```rust
struct IrSourceMap {
    source_id: SourceId,
    ranges: Vec<IrRangeEntry>,
}

struct IrSideTables {
    source_maps: Vec<IrSourceMap>,
    diagnostics: Vec<DiagnosticDraft>,
    explanation_refs: Vec<ExplanationDraftRef>,
    doc_attachments: Vec<DocAttachmentId>,
}
```

ソースマップとサイドテーブルは出力値とともに保存され、出力ハンドルから参照される。これにより、診断、アーティファクト、LSP、ドキュメントは、IR 値を変更することなくソース範囲について合意できる。

## 制御フロー

### フェーズ出力の公開

1. フェーズタスクが、スナップショットがタスクのスナップショットと一致する入力ハンドルを受け取る。
2. フェーズサービスが、タスクローカルな可変ストレージ内で出力値とサイドテーブルを構築する。
3. `PhaseOutputPublisher` が、作業単位の同一性、ソースマップの網羅性、宣言された出力種別を検証する。
4. パブリッシャーが、正準的な内部エンコーディング、名前付きの入力ハッシュ、依存出力のコンテンツハッシュから、コンテンツハッシュを計算する。
5. 出力が `IrStorageService` へ封印される。
6. スケジューラが `PhaseOutputRef<T>` を受け取り、依存タスクへ渡す。

依存タスクは、構築途中の IR を観測することはない。

### キャッシュヒットの再水和

1. スケジューラが、タスクのキャッシュキーで `CacheStore` にレコードを問い合わせる。
2. `IrCacheAdapter` が、スキーマバージョン、ツールチェインの互換性、依存ハッシュ、ソースハッシュ、ポリシーフィンガープリントを検証する。
3. アダプタが、現在の `BuildSnapshotId` 内に出力ハンドルを再構築する。
4. 再構築されたハンドルは、`OutputStorageRef::CacheRecord` の系譜と、キャッシュレコードから再水和されたサイドテーブル参照を記録する。
5. 下流タスクは、通常の完了済み出力と同じように消費する。

証明に関わる出力では、キャッシュ検証は、受理された証明ウィットネスまたは決定的な義務解消のレコードも検証しなければならない。キャッシュヒット自体は証明の根拠ではない。

### ウォッチと LSP のためのスナップショット入れ替え

1. ソースの編集が、新しい `BuildSnapshotId` を作る。
2. 古いスナップショットのハンドルは、参照されている間、古い診断や説明要求のために読み取り可能なまま残してよい。
3. 新しいタスクは、キャッシュ検証または明示的な「ソース未変更」チェックを通してのみ、古いデータを再利用してよい。
4. LSP ブリッジは、診断集約の後に、現在の意味スナップショットをアトミックに切り替える。
5. 参照されない古いハンドルは、回収の対象となる。

開いているバッファはソーススナップショットであり、公開アーティファクトではない。LSP 機能は、データが現在のスナップショット、古いスナップショット、構文のみのフォールバックのいずれから来たかを報告しなければならない。

### アーティファクトへの射影

1. `ArtifactProjectionService` が、1 つのモジュールの封印されたフェーズ出力を読む。
2. 安定した公開メタデータを `VerifiedArtifactDraft` へ射影する。
3. 内部 ID を、アーティファクト ID、FQN、ラベル、ソース範囲へ対応付ける。
4. 生の IR ストレージ参照は、ドラフトから取り除く。
5. `ArtifactService` が、フェーズ 15 のマニフェストトランザクションを通じて、アーティファクトをシリアライズして公開する。

アーティファクトの公開後、下流のパッケージ、ドキュメント、LSP のアーティファクト読み取りは、内部 IR ストレージではなくアーティファクトを消費する。

## API 概要

### IR ストレージサービス

```rust
trait IrStorageService {
    fn seal<T>(&self, output: SealOutputInput<T>) -> Result<PhaseOutputRef<T>, IrStorageError>;
    fn get<T>(&self, handle: &PhaseOutputRef<T>) -> Result<Arc<T>, IrStorageError>;
    fn side_tables(&self, handle: SideTableRef) -> Result<Arc<IrSideTables>, IrStorageError>;
    fn retain(&self, handle: PhaseOutputId, owner: RetainOwner) -> RetainGuard;
    fn collect(&self, snapshot: BuildSnapshotId) -> CollectionSummary;
}
```

### スナップショットハンドルレジストリ

```rust
trait SnapshotHandleRegistry {
    fn create_snapshot(&self, input: SnapshotInput) -> BuildSnapshot;
    fn source_identity(&self, snapshot: BuildSnapshotId, source: SourcePath) -> Option<SourceVersion>;
    fn derive_output_id(&self, input: OutputIdentityInput) -> PhaseOutputId;
    fn is_current(&self, snapshot: BuildSnapshotId) -> bool;
}
```

### フェーズ出力パブリッシャー

```rust
trait PhaseOutputPublisher {
    fn publish<T>(&self, task: TaskId, output: PhaseOutputDraft<T>) -> Result<PhaseOutputRef<T>, PublishError>;
}

struct PhaseOutputDraft<T> {
    phase: PipelinePhase,
    unit: WorkUnit,
    snapshot: BuildSnapshotId,
    value: T,
    side_tables: IrSideTables,
    dependencies: Vec<AnyPhaseOutputRef>,
}
```

パブリッシャーは、意味的な `content_hash` を、`value`、名前付きの入力ハッシュ、依存の `content_hash` 値から導出する。`side_table_hash` は、診断、説明プレビュー、ソースマップのサイドテーブル、ドキュメント付与メタデータから別途導出する。診断の文言、説明プレビュー、開発専用のサイドテーブルの変更は、そのフェーズがこれらのサイドテーブルを意味的な入力だと明示しない限り、意味的なキャッシュエントリを無効化してはならない。パブリッシャーは、`PhaseOutputId`、`TaskId`、メモリアドレス、セッションローカルなストレージ ID を、いずれのハッシュにも含めてはならない。

## エラー処理

### ストレージエラー

| 条件 | 処理 |
|---|---|
| ストレージスロットの欠落 | 由来に応じて、内部エラーまたはキャッシュミスとして扱う |
| スキーマの不一致 | キャッシュ再利用を拒否し、可能であればタスクを再実行する |
| 内部 blob の破損 | キャッシュレコードを破棄し、設定されていればキャッシュ整合性の診断を報告する |
| 要求された型の不一致 | 内部 API エラー。バイト列を別の IR 種別として再解釈してはならない |

### スナップショットエラー

| 条件 | 処理 |
|---|---|
| 古いスナップショットのハンドルを現在として使用 | 拒否し、現在スナップショットのタスクまたはキャッシュ検証を要求する |
| 依存アーティファクトのハッシュ不一致 | 依存ハンドルを無効化する |
| ソースハッシュの不一致 | ソース由来のハンドルを無効化する |
| ソースマップの欠落 | 診断やアーティファクト射影が不要とする場合のみ計算を許す。そうでなければタスクを失敗させる |

### 射影エラー

射影の失敗は、当該モジュールのアーティファクトコミットをブロックするが、封印された IR 出力を変更することはない。フェーズ 15 が有効な置き換えをコミットするまで、以前のアーティファクトマニフェストが権威であり続ける。

## 制約と前提

- 内部 IR 値は、それを生成したフェーズが公開した後、不変である。
- 内部 ID は、アーティファクトの同一性へ明示的に射影されない限り、`BuildSnapshotId` にスコープされる。
- フェーズタスクは、可変な IR オブジェクトではなくハンドルをやり取りする。
- フェーズ出力のコンテンツハッシュは、スナップショットローカルな ID ではなく、内容と名前付きの入力に基づく。
- キャッシュレコードは最適化用のデータであり、証明の根拠ではない。
- 公開アーティファクトは安定した射影であり、生のコンパイラ内部 IR ストレージ参照を含んではならない。
- 古いスナップショットは、LSP や診断から参照されている間は読み取り可能でよいが、現在のものとして報告してはならない。
- ストレージの回収は、公開アーティファクトやソースレベルの意味論を変更してはならない。
