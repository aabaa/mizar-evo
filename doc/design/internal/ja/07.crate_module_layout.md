# Internal Design: Crate and Module Layout

> Canonical language: English. English canonical version: [../en/07.crate_module_layout.md](../en/07.crate_module_layout.md).

## 目的

この文書は、architecture / internal design documents を実装するための想定 crate / module layout をまとめる。

これは Cargo workspace manifest ではない。module-level design work を evo2 architecture guardrails、つまり small kernel verification、immutable IR snapshots、deterministic builds、local verification artifacts、dependency fingerprints、stable failure semantics、fail-heavy testing に揃えるための ownership map である。

## Constraints and Assumptions

- crate boundaries は変わってよいが、ownership boundaries は明示し続けなければならない。
- module specs は、自分が詳細化する architecture documents へ link しなければならない。
- proof acceptance に影響する module は、それが trusted kernel code、untrusted evidence production、policy projection のどれかを明記しなければならない。
- hash を計算する module は canonical ordering と excluded nondeterministic inputs を明記しなければならない。
- failure を report する module は stable categories と、必要に応じて structured rejection reasons を emit しなければならない。
