## 責務

所有する挙動:

- pipeline phase 14: 信頼された proof-certificate checking を実装する;
- normalized certificate を構文解析し、構造検証する。ただし構文解析だけで
  意味論的信頼は与えない;
- SAT clause を正準表現し、重複 literal を除去し、不正または未対応の形を
  拒否し、決定的な rendering/hash を生成する;
- explicit parent clause と pivot から MiniSAT 互換 resolution trace を
  決定的に replay する;
- substitution、alpha-conversion、freshness、free-variable side condition を
  `mizar-core` binder contract に対して独立に検査する;
- immutable kernel context input に対し、stable id と content fingerprint で
  imported fact を検査する;
- upstream cluster-search output が明示的に serialized されている場合に限り、
  explicit cluster/reduction trace を replay する;
- policy-independent な kernel acceptance または stable rejection record を、
  決定的順序で返す。

範囲外:

- SAT solving、ATP backend execution、proof search、premise selection、
  heuristic search、overload resolution、cluster search、registration
  activation、implicit coercion insertion、fallback inference、global mutable
  compiler state;
- source loading、parsing、name resolution、type checking、elaboration、VC
  generation、ATP encoding、portfolio winner selection、proof-policy
  projection、artifact publication、cache lookup、cache reuse validation;
- 現在の実装挙動に合わせるための `doc/spec`、既存 `.miz` test、expectation
  sidecar の変更;
- 未完成の `mizar-atp`、`mizar-proof`、`mizar-cache`、`mizar-artifact` seam に
  対する placeholder integration。

依存方針:

- production dependency は `mizar-session` と `mizar-core` に限定する;
- `mizar-atp`、`mizar-proof`、`mizar-cache`、`mizar-artifact` は
  `mizar-kernel` の production dependency にしない;
- 不足している producer/consumer integration は kernel 内で mock せず、
  `external_dependency_gap` または `deferred` として分類する。

kernel 禁止事項:

- proof search 禁止;
- heuristic premise selection 禁止;
- overload resolution 禁止;
- cluster search 禁止;
- ATP search または backend invocation 禁止;
- implicit coercion insertion 禁止;
- fallback inference 禁止;
- backend-reported success だけによる acceptance 禁止;
- mutable compiler-global state、wall-clock time、random state、filesystem
  cache、unordered iteration の hidden read 禁止。

task 1 で予定する trusted-baseline lint policy:

- crate root は unsafe code を forbid し、workspace warning/clippy denial に
  opt in する;
- crate-local `tests/lint_policy.rs` guard は production dependency の完全一致、
  dev/build/target dependency escape hatch がないこと、workspace membership、
  `[lints] workspace = true`、crate-root trust statement、paired spec が存在する
  まで public semantic surface を expose しないこと、ATP/proof/cache/artifact
  coupling の既知 forbidden import/token がないことを検査する;
- public enum は、後続の compatibility audit が正当化された例外を記録しない
  限り `#[non_exhaustive]` とする予定である;
- 対応する source module を expose する前に、module spec は trusted-kernel 文と
  kernel 禁止事項を再掲しなければならない。

## Exit criteria

Hard gates:

- unresolved blocking/high specification inconsistency がない;
- source behavior が module specs、tests、traceability metadata、explicit human
  approval、または explicit deferred row なしに残っていない;
- milestone-owned certificate、clause、substitution、resolution、imported-fact、
  cluster-replay、checker-service behavior が Rust coverage または具体的な
  deferred reason を持つ;
- 既存 `.miz` fixture と expectation sidecar は実装に合わせる目的で変更されていない;
- 英語正本と日本語 companion が同期している;
- crate dependency と module exposure policy guard が通る;
- kernel が solver、ATP、proof-policy、cache、artifact、overload、cluster-search、
  coercion-insertion、fallback-inference、global-state coupling を含まない;
- required verification command が通るか、未実行 command が明示的に正当化されている;
- remaining risk は `external_dependency_gap`、`deferred`、out of scope、human-owned
  として分類されている。

Rust source task ごとの verification:

```text
cargo fmt --check
cargo test -p mizar-kernel
cargo clippy -p mizar-kernel --all-targets --all-features -- -D warnings
git diff --check
git diff --cached --check
```

binder contract や checker/trace boundary に触れる場合は、task が実行不能な
具体的理由を記録しない限り、以下も required check とする:

```text
cargo test -p mizar-core
cargo test -p mizar-checker
```

docs-only task は少なくとも以下を実行する:

```text
git diff --check
git diff --cached --check
```

crate closeout では以下を実行する:

```text
cargo fmt --check
cargo clippy --all-targets --all-features -- -D warnings
cargo test
git diff --check
git diff --cached --check
```

Review expectations:

- すべての task は implementation specification / documentation review、test
  sufficiency review、full implementation review、source/documentation consistency
  review に review-only agent を使う;
- blocking/high finding は commit 前に修正し再 review する;
- medium finding は修正するか理由付きで explicit deferred とする;
- crate completion には hard gates pass と read-only quality score 90/100 以上が必要。
