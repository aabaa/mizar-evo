## Responsibility

Owned behavior:

- implement pipeline phase 14: trusted proof-certificate checking;
- parse and structurally validate normalized certificates without granting
  semantic trust during parsing;
- represent SAT clauses canonically, remove duplicate literals, reject malformed
  or unsupported shapes, and produce deterministic renderings and hashes;
- replay MiniSAT-compatible resolution traces deterministically from explicit
  parent clauses and pivots;
- verify substitution, alpha-conversion, freshness, and free-variable side
  conditions independently against the `mizar-core` binder contract;
- check imported facts by stable id and content fingerprint against immutable
  kernel context inputs;
- replay explicit cluster/reduction traces only when their evidence is present
  and upstream cluster-search output is already serialized;
- return policy-independent kernel acceptance or stable rejection records with
  deterministic ordering.

Out of scope:

- SAT solving, ATP backend execution, proof search, premise selection,
  heuristic search, overload resolution, cluster search, registration
  activation, implicit coercion insertion, fallback inference, and global
  mutable compiler state;
- source loading, parsing, name resolution, type checking, elaboration, VC
  generation, ATP encoding, portfolio winner selection, proof-policy
  projection, artifact publication, cache lookup, or cache reuse validation;
- changing `doc/spec`, existing `.miz` tests, or expectation sidecars merely to
  match current implementation behavior;
- placeholder integration with unfinished `mizar-atp`, `mizar-proof`,
  `mizar-cache`, or `mizar-artifact` seams.

Dependency policy:

- production dependencies are limited to `mizar-session` and `mizar-core`;
- `mizar-atp`, `mizar-proof`, `mizar-cache`, and `mizar-artifact` must not be
  production dependencies of `mizar-kernel`;
- any missing consumer or producer integration is classified as
  `external_dependency_gap` or `deferred`, not mocked inside the kernel.

Kernel prohibitions:

- no proof search;
- no heuristic premise selection;
- no overload resolution;
- no cluster search;
- no ATP search or backend invocation;
- no implicit coercion insertion;
- no fallback inference;
- no acceptance from backend-reported success alone;
- no hidden reads of mutable compiler-global state, wall-clock time, random
  state, filesystem caches, or unordered iteration.

Trusted-baseline lint policy planned for task 1:

- the crate root forbids unsafe code and opts into workspace warnings/clippy
  denial;
- a crate-local `tests/lint_policy.rs` guard checks exact production
  dependencies with no dev/build/target dependency escape hatches, workspace
  membership, `[lints] workspace = true`, crate-root trust statement, no public
  semantic surface before paired specs exist, and the absence of known
  forbidden imports/tokens for ATP/proof/cache/artifact coupling;
- public enums are planned to be `#[non_exhaustive]` unless a later
  compatibility audit records a justified exception;
- module specs must include a trusted-kernel statement and restate the kernel
  prohibitions before their corresponding source modules are exposed.

## Exit Criteria

Hard gates:

- no unresolved blocking/high specification inconsistency remains;
- no source behavior remains absent from module specs, tests, traceability
  metadata, explicit human approval, or explicit deferred rows;
- milestone-owned certificate, clause, substitution, resolution, imported-fact,
  cluster-replay, and checker-service behavior has Rust coverage or a concrete
  deferred reason;
- existing `.miz` fixtures and expectation sidecars are not changed merely to
  match implementation behavior;
- English canonical docs and Japanese companions are synchronized;
- crate dependency and module exposure policy guards pass;
- the kernel contains no solver, ATP, proof-policy, cache, artifact, overload,
  cluster-search, coercion-insertion, fallback-inference, or global-state
  coupling;
- required verification commands pass, or any unrun command is explicitly
  justified;
- remaining risks are classified as `external_dependency_gap`, `deferred`, out
  of scope, or human-owned.

- crate completion requires hard gates to pass and a read-only quality score of
  at least 90/100.
