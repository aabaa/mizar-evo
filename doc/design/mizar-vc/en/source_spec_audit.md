# Source/Spec Correspondence Audit: mizar-vc

> Canonical language: English. Japanese companion:
> [../ja/source_spec_audit.md](../ja/source_spec_audit.md).

Task 18 audits the current `mizar-vc` public surface and promised behavior after
the public-enum policy gate. It changes no source behavior, `.miz` fixtures,
expectations, public APIs, or language specifications. Remaining unavailable
behavior is recorded as explicit `external_dependency_gap`, `test_gap`, or
`deferred` work instead of treating current implementation limits as normative.

## Scope And Method

The inventory covers the public modules exported by
`crates/mizar-vc/src/lib.rs`, top-level public items in each module, and public
methods exposed by those public data types. Public newtypes generated by the
crate-local `dense_id!` and `string_key!` macros are grouped with `vc_ir`
because the module spec owns their constructors and accessors.

Module specifications audited:

- [vc_ir.md](./vc_ir.md)
- [generator.md](./generator.md)
- [discharge.md](./discharge.md)
- [dependency_slice.md](./dependency_slice.md)

Result: no unclassified `source_drift`, `design_drift`,
`source_undocumented_behavior`, `test_expectation_drift`,
`boundary_violation`, or `repo_metadata_conflict` is observed for the current
explicit-payload implementation. Remaining gaps are the known deferred or
external seams listed below.
