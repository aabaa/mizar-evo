## 責務

所有する挙動:

- pipeline phase 11-12、つまり VC 生成と決定的 pre-ATP discharge を実装する;
- `mizar_core::core_ir::CoreIr`、
  `mizar_core::control_flow::ControlFlowIr`、core obligation-seed handoff を
  消費する;
- prover 非依存の `VcIr`、決定的な `VcId` 割り当て、
  `ObligationAnchor` 構築、canonical VC fingerprint、local-context
  fingerprint を所有する;
- explicit な core/control-flow 入力から、定理、proof step、定義 correctness、
  registration / redefinition / reduction correctness、checker initial、
  algorithm contract、assertion、不変条件、ghost erasure、termination の VC を
  生成する;
- symbolic premise reference、proof hint、source/core provenance、局所 proof /
  program path、policy status、安定した diagnostic ordering を保持する;
- replayable evidence と安定した computation limit を持つ決定的 pre-ATP
  discharge を行う;
- cache miss / reuse 判断に供給できる保守的な VC ごとの dependency slice と
  fingerprint を計算し、不完全な依存情報を隠さない;
- task 25 が builder を実装した後、既存 `VcIr` の formula、substitution、
  provenance、target-binding data から prover-independent kernel evidence handoff
  package を生産する。ただしその package を acceptance として扱わない。

範囲外:

- source loading、lexing、parsing、name resolution、type checking、
  registration activation、overload resolution、elaboration、control-flow
  construction;
- 現在の実装挙動に合わせるためだけの `doc/spec`、既存 `.miz` fixture、
  expectation sidecar の変更;
- ATP encoding、backend process execution、portfolio scheduling、
  certificate validation、kernel proof acceptance、SAT checking、artifact
  publication、cache storage、proof-reuse validation;
- diagnostics registry が方針を所有する前の user-facing public diagnostic code
  割り当て;
- deterministic discharge、ATP success、policy-open、assumed、
  externally attested 状態を kernel-verified proof status に昇格すること。

境界規則:

- `mizar-vc` は現在の build snapshot 内でのみ `VcId` を割り当てる。`VcId` は
  編集横断 proof-reuse identity ではない;
- `ObligationAnchor` は reuse-candidate key であり proof evidence ではない。
  reuse にはさらに canonical VC、context、dependency-slice、policy、
  witness/discharge hash が必要である;
- ATP 行きの VC は full local context と `NeedsAtp` status を持ったまま残す;
- 未対応または不完全な依存情報は、下流 cache consumer が miss を強制できる
  よう保守的に表現する;
- この crate は untrusted な deterministic evidence を生産できるが、trusted
  acceptance は後段の proof/kernel policy が所有する。

## exit criteria

Hard gates:

- 未解決の blocking/high specification inconsistency が残らない;
- milestone scope 内で、`doc/spec`、tests、traceability metadata、明示的な人間の
  承認、または明示 deferred row にない language/proof source behavior が残らない;
- milestone-owned behavior は Rust coverage、traceability metadata、または具体的な
  deferred/external-dependency reason を持つ;
- 実装挙動に合わせるためだけの test expectation 変更を行わない;
- 英語正本 docs と日本語 companion を同期する;
- `mizar-vc` は ATP、kernel、proof policy、cache storage、artifact publication、
  upstream extraction の ownership に踏み込まない;
- 各 commit 前に必要な verification command が成功する;
- 残る risk は deferred、out of scope、または human-owned と分類する。

Verification commands:

```text
docs-only/spec-only:
  git diff --check
  git diff --cached --check

Rust source tasks:
  cargo test -p mizar-vc
  cargo clippy -p mizar-vc --all-targets -- -D warnings
  cargo fmt --check

Conditional task-local verification:
  cargo test -p mizar-core      # core/control-flow boundary に触れる場合
  cargo test -p mizar-test      # corpus runner/metadata behavior に触れる場合
  cargo test -p mizar-cache     # crate が存在し、task scope が使う場合のみ
  cargo test -p mizar-proof     # crate が存在し、task scope が使う場合のみ

Closeout / broad verification:
  cargo fmt --check
  cargo clippy --all-targets --all-features -- -D warnings
  cargo test
```

Review expectations:

- 各 task は implementation specification / documentation review、test
  sufficiency review、full implementation review、source/documentation
  consistency review に review-only agent を使う;
- blocking/high finding は commit 前に修正して再 review する;
- medium finding は修正するか、具体的理由つきで明示 deferred にする;
- closeout は hard gates passing と 90/100 以上の read-only crate quality
  review を含める。
