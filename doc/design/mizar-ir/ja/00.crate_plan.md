## 責務

所有する挙動:

- コンパイラ内部の不変 IR 出力ストレージと型付き
  `PhaseOutputRef<T>` ハンドル。
- snapshot スコープの IR 識別テーブルと、phase 出力の親/派生 lineage。
- 完成した出力だけを seal し、未 seal・部分・obsolete な出力を current
  publication から隠す phase-output publication。
- content-addressed な内部 blob と、retain された出力のライフタイム回収。
- 検証済み `mizar-cache` record だけを消費し、fail-closed 検証の後でのみ
  handle を再水和する IR cache adapter 境界。
- seal 済み内部出力から `mizar-artifact` の安定した外部 schema へ射影する
  artifact projection。

対象外:

- proof acceptance、trusted status、verifier policy selection、決定的な proof
  winner selection、kernel acceptance、`used_axioms` の信頼。
- `mizar-cache` の `CacheKey`、dependency fingerprint、dependency slice、
  proof-reuse validation の所有。
- published artifact における生の `SurfaceAst`、`TypedAst`、`CoreIr`、
  `ControlFlowIr`、`VcIr`、`AtpProblem`、kernel 内部状態、storage reference。
- 所有 crate が実 seam を公開するまでの `mizar-driver` build session、
  `mizar-diagnostics` integration、real producer projection token、
  downstream artifact publication token。

## Exit Criteria

Hard gates:

- blocking/high specification inconsistency が残らない。
- public/private を問わず source behavior が module specs/tests にないまま
  残らない。private canonicalization、retention、fail-closed cache、
  projection-filtering behavior を含む。
- implementation behavior に合わせるために language specification、`.miz` test、
  traceability metadata、expectation sidecar を変更しない。
- `mizar-ir` milestone について docs と source が一致する。
- EN/JA mizar-ir docs が同期している。
- `mizar-ir` は `mizar-driver` dependency を持たず、driver、diagnostics、
  producer-token、publication-token placeholder を作らない。
- cache rehydration は fail-closed で、validation 前に handle を作らない。
- published projection values は stable external schema のみを含む。
- cache hit と rehydrated handle は proof/trusted status を昇格しない。
- required verification commands が pass するか、未実行 command が明示的に
  justify されている。
- 残る risk がすべて deferred、out of scope、human-owned、または許可された
  protocol class と risk tag `external_dependency_gap` の組として分類されている。
- read-only quality review が 90/100 以上の valid score を付ける。

Verification commands:

```text
cargo test -p mizar-ir
cargo clippy -p mizar-ir --all-targets -- -D warnings
cargo fmt --check
cargo test -p mizar-cache
cargo test -p mizar-artifact
cargo test -p mizar-build
cargo clippy --all-targets --all-features -- -D warnings
cargo test
git diff --check
git diff --cached --check
```

この checkout で real driver crate が存在するまでは
`cargo test -p mizar-driver` は実行不能であり、placeholder crate で埋めず
risk tag `external_dependency_gap` として扱う。

Review expectations:

- spec/docs、test sufficiency、implementation、source/documentation consistency、
  final quality について review-only Agent review を使う。
- medium finding は修正するか理由付きで明示 defer する。
- blocking/high finding は該当 task commit の前に修正する。
