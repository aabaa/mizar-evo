# Crate Plan: mizar-ir

> Canonical language: English. Japanese companion:
> [../ja/00.crate_plan.md](../ja/00.crate_plan.md).

## Responsibility

Owned behavior:

- compiler-internal immutable IR output storage and typed
  `PhaseOutputRef<T>` handles;
- snapshot-scoped IR identity tables and parent/derived phase-output lineage;
- phase-output publication that seals complete outputs and hides unsealed,
  partial, or obsolete outputs from current publication;
- content-addressed internal blobs and lifetime collection for retained
  outputs;
- an IR cache adapter boundary that consumes validated `mizar-cache` records
  and rehydrates handles only after fail-closed validation;
- phase-dispatch input identity bundles and sealed parent output handles for
  scheduler-selected real phase dispatch;
- artifact projection from sealed internal outputs into
  `mizar-artifact` stable external schemas.

Out of scope:

- proof acceptance, trusted status, verifier-policy selection, deterministic
  proof winner selection, kernel acceptance, or `used_axioms` trust;
- `mizar-cache` `CacheKey`, dependency fingerprint, dependency slice, or
  proof-reuse validation ownership;
- raw `SurfaceAst`, `TypedAst`, `CoreIr`, `ControlFlowIr`, `VcIr`,
  `AtpProblem`, kernel-internal state, or storage references in published
  artifacts;
- `mizar-driver` scheduling/front-door orchestration,
  `mizar-diagnostics` rendering integration, real producer projection tokens,
  and downstream artifact publication tokens until those owning crates expose
  real seams.

## Exit Criteria

Hard gates:

- no blocking/high specification inconsistency remains;
- no source behavior remains absent from module specs/tests, including private
  canonicalization, retention, fail-closed cache, and projection-filtering
  behavior;
- no language specification, `.miz` test, traceability metadata, or expectation
  sidecar is changed to match implementation behavior;
- docs and source agree for the `mizar-ir` milestone;
- EN/JA mizar-ir documentation is synchronized;
- `mizar-ir` has no `mizar-driver` dependency and does not create driver,
  diagnostics, producer-token, or publication-token placeholders;
- cache rehydration is fail-closed and cannot create a handle before
  validation;
- published projection values contain stable external schema only;
- scheduler-selected dispatch input identities and parent handles are owned by
  `mizar-ir` and consumed by the driver/front door;
- cache hits and rehydrated handles never upgrade proof/trusted status;
- required verification commands pass or unrun commands are explicitly
  justified;
- all remaining risks are classified as deferred, out of scope, human-owned,
  or have an allowed protocol class plus risk tag `external_dependency_gap`;
- read-only quality review assigns a valid score of at least 90/100.

cargo test -p mizar-ir

and consumes the IR-owned dispatch input bundle. Missing real producer,
diagnostics, artifact-token, semantic/proof adapter, cache-compatibility, and
LSP seams remain risk-tagged `external_dependency_gap` or `deferred`; they are
not patched with placeholder implementations.

Review expectations:

- review-only Agent reviews for spec/docs, test sufficiency, implementation,
  source/documentation consistency, and final quality;
- medium findings are fixed or explicitly deferred with a reason;
- blocking/high findings are fixed before the relevant task commit.
