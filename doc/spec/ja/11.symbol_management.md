## 11.1 同義語と反意語

Mizar では、述語、functor、モード、属性の **同義語** と **反意語** を定義できます。
これらは、意味上の同等性を維持しながら、代替表記を提供します。predicate と
functor の alias は、直接の `pred` / `func` 定義と同じ任意の user notation
symbol を使用できます。mode と attribute の alias は constructor name を使用し、
任意の operator-like mode / attribute name のための特別な抜け道にはなりません。

**構文 (EBNF)**:

```ebnf
synonym_def  ::= "synonym" alt_pattern "for" original_pattern ";" ;
antonym_def  ::= "antonym" alt_pattern "for" original_pattern ";" ;
alt_pattern      ::= pred_pattern | func_pattern | alt_mode_pattern | alt_attr_pattern ;
original_pattern ::= pred_pattern | func_pattern | original_mode_pattern | original_attr_pattern ;
alt_mode_pattern      ::= mode_def_name [ type_params ] ;
original_mode_pattern ::= mode_ref_name [ type_params ] ;
alt_attr_pattern      ::= [ param_prefix ] attribute_def_name ;
original_attr_pattern ::= [ param_prefix ] attribute_ref_name ;
```

### 11.1.1 述語の同義語と反意語

```mizar
definition
  let a, b be Real;
  pred LessThanDef: a < b means ...;
  synonym b > a for a < b;
  antonym a >= b for a < b;
end;
```

### 11.1.2 functorの同義語

```mizar
definition
  let f be Function;
  func InverseDef: f" -> Function means ...;
  synonym inv f for f";
end;
```

### 11.1.3 モードの同義語

```mizar
definition
  let G be set;
  mode FinSequenceDef: FinSequence of G is ...;
  synonym FinSeq of G for FinSequence of G;
end;
```

### 11.1.4 属性の同義語と反意語

```mizar
definition
  let R be Ring;
  attr CommutativeDef: R is commutative means ...;
  synonym abelian for commutative;
end;

definition
  let S be set;
  attr FiniteDef: S is finite means ...;
  antonym infinite for finite;
end;
```

### 11.1.5 利点

|メリット |説明 |
| --- | --- |
| **別の表記** |さまざまな数学的規則をサポート |
| **自然否定** |明示的な `non` または `not` を使用せずに明示的な否定 |
| **意味上の同等性** |可読性を向上させながら論理的同一性を維持 |
| **互換性** |さまざまな数学的伝統の規則を一致させる |

## 11.2 範囲と可視性

### 11.2.1 スコープの階層

シンボルは次の階層に従って表示されます。

|レベル |説明 |例 |
| --- | --- | --- |
| **ローカル** |現在の証明ブロック内で定義 | `deffunc`、`defpred` |
| **module** |moduleレベルで定義 | `pred`、`func`、`mode`、`attr`、`struct`、`synonym`、`antonym`、operator declaration |
| **輸入品** | `import` 経由で入手可能 | `import mml.algebra.group` |

lexing では、imported summary が module body 開始時点の environment を初期化します。
current module の module-level declaration は、その declaration item が完了した後に
だけ environment を拡張します。後続宣言への forward reference は許されません。
inline `deffunc` と `defpred` は identifier-only の local proof abbreviation であり、
lexer user-symbol dictionary entry を導入しません。

### 11.2.2 moduleレベルの定義

```mizar
definition
  let x, y be Integer;
  pred DivisibilityDef: x divides y means ex k be Integer st y = x * k;
end;
```

### 11.2.3 インポートされたシンボル

```mizar
import mml.algebra.group;
import mml.topology.space as top;
```

### 11.2.4 優先規則

1. **Lexing は overload candidate を保持します**。同じ spelling の local symbol と
   imported symbol があっても、imported candidate を lexer が削除して shadow する
   ことはありません。
2. **Semantic lookup** は resolver の scope / overload rule を適用します。
   local proof/block binding、definition parameter、current-module declaration、
   import、re-export、built-in prelude がこの順で考慮されます。
3. **Import conflicts** は、修飾または alias で解決されない限り compile-time error
   になります。
4. **Redefinitions** には `redefine` keyword が必要で、新しい lexical spelling は
   導入しません。

## 11.3 インポート動作と競合解決

**構文 (EBNF)**:

```ebnf
import_stmt        ::= "import" module_alias_decl
                       { "," module_alias_decl } ";" ;
module_alias_decl  ::= module_path [ "as" module_identifier ]
                     | module_branch_import ;
module_branch_import ::= module_path ".{" module_identifier
                       { "," module_identifier } "}" ;
module_path        ::= [ relative_prefix ] module_identifier
                       { "." module_identifier } ;
relative_prefix    ::= "." | ".." ;
module_identifier  ::= identifier ;
qualified_name     ::= [ module_path "." ] symbol_name ;
```

### 11.3.1 インポート構文

Import declarations は `.miz` file 先頭の module import prelude でのみ許可されます。配置ルールについては Chapter 12, §12.3 を参照してください。

```mizar
import mml.algebra.group;                    :: direct import
import mml.topology.space as top;            :: aliased import
import mml.algebra.{group, ring};            :: branch import
import ..foundation.sets;                    :: relative import
```

### 11.3.2 紛争解決戦略

**1.namespaceの資格**

```mizar
import mml.algebra.group;
import mml.topology.topological_group as top_grp;

let G be mml.algebra.group.Group;
let T be top_grp.Group;
```

**2.moduleの優先順位**

```mizar
import mml.set_theory.cardinal;

definition
  let A, B be set;
  :: Local definition takes precedence over imported cardinal.card_leq
  pred LocalCardLeq: A card_leq B means card A c= card B;
end;
```

**3.最長一致ルール**

字句解析中に、最も長く一致するtokenが選択されます。

```mizar
:: Suppose + and ++ are both defined symbols
a ++ b   :: Parsed as (a) (++) (b), not (a) (+) (+) (b)
```

## 11.4 パブリックシンボルとプライベートシンボル

**構文 (EBNF)**:

```ebnf
visible_declaration ::= [ visibility ] definition_body ;
visibility          ::= "private" | "public" ;
definition_body     ::= pred_def | func_def | mode_def | attr_def
                      | struct_def | theorem_item
                      | synonym_def | antonym_def ;
```

### 11.4.1 可視性のデフォルト

|シンボルの種類 |デフォルトの可視性 |
| --- | --- |
| `pred`、`func`、`mode`、`attr`、`struct` |パブリック |
| `theorem`、`lemma` |パブリック |
| `synonym`、`antonym` |パブリック |

### 11.4.2 プライベート宣言

```mizar
definition
  let x, y be Real;
  private pred HelperPred: helper(x, y) means x * x + y * y < 1;
end;

definition
  let x be Real;
  private func IntermediateFunc: intermediate(x) -> Real equals x * x + 1;
end;
```

### 11.4.3 プライベートシンボルの動作

* パブリックmoduleインターフェースから除外
* 他のmoduleからのインポートではアクセスできません
* 依存moduleによって再エクスポートされない
* 内部ヘルパー定義に役立ちます
* 定義 module 内では、宣言位置より後の lexing / parsing には active です

## 11.5 論理エンコーディングの詳細

このセクションでは、シンボル管理構造が一階論理にどのようにマップされるかについて説明します。

### 11.5.1 同義語のエンコーディング

同義語は論理的等価としてエンコードされます。

|Mizar source | FOL エンコーディング |
| --- | --- |
| `synonym b > a for a < b` | `∀a,b. (b > a) ↔ (a < b)` |
| `synonym inv f for f"` | `∀f. inv(f) = inverse(f)` |
| `synonym FinSeq of G for FinSequence of G` | `∀G,x. is_FinSeq(x,G) ↔ is_FinSequence(x,G)` |

### 11.5.2 反意語のエンコーディング

反意語は論理否定としてエンコードされます。

|Mizar source | FOL エンコーディング |
| --- | --- |
| `antonym a >= b for a < b` | `∀a,b. (a >= b) ↔ ¬(a < b)` |
| `antonym infinite for finite` | `∀S. is_infinite(S) ↔ ¬is_finite(S)` |

### 11.5.3 可視性エンコーディング

プライベート シンボルは論理インターフェイスにエクスポートされません。

```
private pred helper(x, y) means φ
⇒  (internal only, not in exported theory)
```

パブリック シンボルはエクスポートされた公理を生成します。

```
pred P(x, y) means φ
⇒  forall x, y (P(x, y) ↔ φ)  [exported]
```

### 11.5.4 インポートエンコーディング

インポートにより、**オンデマンド**で外部公理を利用できるようになります。インポートmoduleで実際に参照されるシンボルのみが論理理論に含まれます。

```
import mml.algebra.group
⇒  For each symbol S from mml.algebra.group used in this module:
     Include the axioms defining S
```

この遅延読み込みアプローチは次のとおりです。
- 発生する証明義務の規模を削減します
- 検証パフォーマンスの向上
- 論理的健全性を維持します (推移的な依存関係は自動的に含まれます)。

## 11.6 完全な構文 (EBNF)

シンボル管理構文の概要。

```ebnf
(* Synonyms and Antonyms *)
synonym_def    ::= "synonym" alt_pattern "for" original_pattern ";" ;
antonym_def    ::= "antonym" alt_pattern "for" original_pattern ";" ;
alt_pattern      ::= pred_pattern | func_pattern | alt_mode_pattern | alt_attr_pattern ;
original_pattern ::= pred_pattern | func_pattern | original_mode_pattern | original_attr_pattern ;

(* Import Statements *)
import_stmt        ::= "import" module_alias_decl
                       { "," module_alias_decl } ";" ;
module_alias_decl  ::= module_path [ "as" module_identifier ]
                     | module_branch_import ;
module_branch_import ::= module_path ".{" module_identifier
                       { "," module_identifier } "}" ;
module_path        ::= [ relative_prefix ] module_identifier
                       { "." module_identifier } ;
relative_prefix    ::= "." | ".." ;
module_identifier  ::= identifier ;
qualified_name     ::= [ module_path "." ] symbol_name ;

(* Visibility *)
visible_declaration ::= [ visibility ] definition_body ;
visibility          ::= "private" | "public" ;
definition_body     ::= pred_def | func_def | mode_def | attr_def
                      | struct_def | theorem_item
                      | synonym_def | antonym_def ;

(* Symbol Patterns *)
pred_pattern   ::= [ loci ] def_predicate_symbol [ template_loci ] [ loci ] ;
func_pattern   ::= [ loci ] functor_symbol [ template_loci ] [ loci ] ;
loci           ::= locus_list | "(" locus_list ")" ;
locus_list     ::= locus { "," locus } ;
locus          ::= identifier ;
template_loci  ::= "[" locus_list "]" ;
alt_mode_pattern      ::= mode_def_name [ type_params ] ;
original_mode_pattern ::= mode_ref_name [ type_params ] ;
alt_attr_pattern      ::= [ param_prefix ] attribute_def_name ;
original_attr_pattern ::= [ param_prefix ] attribute_ref_name ;
mode_def_name         ::= constructor_name ;
mode_ref_name         ::= qualified_constructor_name ;
attribute_def_name    ::= constructor_name ;
attribute_ref_name    ::= qualified_constructor_name ;
qualified_symbol            ::= { namespace_segment "." } user_symbol ;
qualified_constructor_name  ::= { namespace_segment "." } constructor_name ;
namespace_segment     ::= identifier ;
```
