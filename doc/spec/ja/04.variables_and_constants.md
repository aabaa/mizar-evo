## 4.1 変数と論理

Mizar では、変数はコンテキストに応じて 3 つの異なる役割を果たします。

1. **自由変数**: `let` によって導入されました。これらは、証明ブロック内で固定された任意の定数 (Skolem 定数) として機能します。
2. **境界変数**: 量指定子 (`for`、`ex`) によって導入されます。範囲は式または用語に限定されます。
3. **ローカル定数**: `set`, named `reconsider`, named `take`, `deffunc`, `defpred` によって導入されます。これらは、特定の値、用語、または local schema (略語) に関連付けられた名前です。

**ソフトタイプシステム**:
Mizar 変数は型付きです。論理的には、宣言 `let x be T` は、単項述語 `T(x)` によって制約された、FOL の型なし変数として `x` を扱うことに対応します。

## 4.2 変数宣言 (`let`)

`let` ステートメントは、現在の環境に **自由変数** を導入します。

**構文 (EBNF)**:

```ebnf
let_decl        ::= "let" qualified_vars [ "such" conditions ] [ "by" references ] ";" ;
qualified_vars  ::= explicit_qualified_vars [ "," implicit_qualified_vars ]
                  | implicit_qualified_vars ;
explicit_qualified_vars ::= qualified_segment { "," qualified_segment } ;
qualified_segment ::= identifier_list ( "being" | "be" ) type_expression ;
implicit_qualified_vars ::= identifier_list ;
identifier_list ::= identifier { "," identifier } ;
conditions      ::= "that" proposition { "and" proposition } ;
proposition     ::= [ label_identifier ":" ] formula ;
```

* **意味論**: 新しい任意の要素を導入します。
* **タイプ仮定**: `be T` が指定されている場合、論理コンテキストは仮定 `x is T` で拡張されます。
* **Such That 節**: オプションの `such that` 節は、宣言された変数に対して 1 つ以上の仮定 (仮説) を追加します。各条件には、後で参照できるように独自のラベルが付いている場合があります。
* **制約の放免**: 末尾の `by references` 句は、use site で constrained mode が導入する well-formedness constraint を放免できます。これは仮定を追加するための一般的な justification ではありません。statement-level の規則については [第15章 (文), §15.2.1](./15.statements.md#1521-一般化ステートメント-let) を参照してください。

> [！注記]
> `let` (一般化) の完全なステートメント レベルの構文については、「[第15章 (文), §15.2.1](./15.statements.md#1521-一般化ステートメント-let)」を参照してください。

**例**:

```mizar
let x be Real;                      :: x is an arbitrary Real
let n, m be Nat;                    :: n and m are Nats
let G be Group;
let a be positive Real;             :: Attributes can be included
let B be Basis[V] by FinDim;        :: FinDim discharges the Basis[V] constraint
let x be Real such that x > 0;      :: x is Real and assumed to be > 0
let p be Prime such that p > 100;   :: p is Prime and assumed > 100
```

## 4.3 予約変数 (`reserve`)

`reserve` ディレクティブは、特定の識別子の **デフォルト タイプ**を定義します。これは変数宣言そのものではなく**、parserに対するマクロのようなヒントです。

`reserve` 宣言は top-level の module declaration としてのみ許可されます。`definition`、`registration`、`proof`、`now`、`algorithm`、その他の nested block の内部では許可されません。block-local な `reserve` 形の statement は構文エラーです。block-local な変数は、`let`、quantified binder、または後述の local constant 形式で導入します。

**構文 (EBNF)**:

```ebnf
reserve_decl    ::= "reserve" reserve_segment ";" ;
reserve_segment ::= identifier_list "for" type_expression ;
```

**意味論**:
`x` が `Real` 用に予約されている場合:

1. `let x;` は `let x be Real;` に拡張されます。
2. `for x holds P(x)` は `for x being Real holds P(x)` に拡張されます。
3. reserved identifier が theorem declaration (`theorem`、`lemma`、将来導入される
   theorem-role alias、`definition` block または `claim` block 内の theorem item を含む)
   や scheme declaration に local binding なしで自由出現し、shadowing もされていない場合、
   その theorem または scheme proposition は reserved type 付きでその identifier に
   暗黙に全称閉包されます。
4. 明示的な型は予約をオーバーライドします。`let x be Nat;` は有効です (予約をシャドウします)。

暗黙の全称閉包は、compact statement、`assume`、`case`、`suppose` item のような
proof-local proposition には適用されません。これらのコンテキストで自由な reserved
identifier が現れる場合、囲む binder または local declaration によって既に導入されて
いなければなりません。そうでなければ unbound-variable error です。

`claim` block では、この reserved identifier の閉包を claim theorem の内部で適用してから、
その theorem を named algorithm のすべての valid execution にわたるものとして解釈します。

複数の reserved identifier が暗黙に閉包される場合、量化順序は reserved type の
dependency order です。`X` の reserved type が reserved identifier `n` を参照するなら、
`n` は `X` より前に量化されます。独立した reserved identifier は proposition 内での
最初の source occurrence 順に並べます。reserved type 間の cyclic dependency は
compile-time error です。

**例**：

```mizar
reserve i, j for Nat;
reserve x for Real;

let i;      :: Equivalent to: let i be Nat;
let x;      :: Equivalent to: let x be Real;
let i be Integer; :: Valid override.

```

## 4.4 ローカル定数 (`set`, `reconsider`, `take`, `deffunc`, `defpred`)

ローカル定数は、特定の用語に名前を割り当てます。

### 4.4.1 Set ステートメント (`set`)

用語式に名前を割り当てます。これは純粋に構文上の略語 (マクロ) または等価公理です。

**構文 (EBNF)**:

```ebnf
set_decl      ::= "set" equating_list ";" ;
equating_list ::= equating { "," equating } ;
equating      ::= identifier "=" term_expression ;

```

**例**：

```mizar
set zero = 0;
set X = {1, 2, 3};
set f = id REAL;
set a = f.x, b = f.y;

```

* **論理**: 各 identifier を導入し、対応する公理 `identifier = term_expression` を追加します。

### 4.4.2 ステートメントの再検討 (`reconsider`)

既存の用語から新しい定数を定義しますが、**新しい型** になります。これにより、証明義務（正確性条件）が発生します。

**構文 (EBNF)**:

```ebnf
reconsider_decl ::= "reconsider" type_change_list "as" type_expression
                    [ simple_justification ] ";" ;
type_change_list ::= reconsider_item { "," reconsider_item } ;
reconsider_item  ::= identifier
                   | identifier "=" term_expression ;
```

次の 2 つの形式があります。

1. **新しい定数形式**: `reconsider P = R as T;` — `T` 型の `R` に等しい新しい定数 `P` を作成します。
2. **型キャスト形式**: `reconsider x as T;` — 新しい名前を作成せずに、既存の変数 `x` を型 `T` にキャストします。

**例**:

```mizar
set R = { x where x is Real : x > 0 };
reconsider P = R as non empty set;  :: New constant P, must prove R is non empty

let n be Nat;
reconsider n as Integer;            :: Type cast: n is now treated as Integer
reconsider x, y = f.x as Element of A;
```

* **意味**: どちらの形式も、用語が指定された型を持つという証明義務 (正確性条件) を生成します。

### 4.4.3 Named Witness and Local Definition Forms

次の form は local name を導入し、その lexical scope が active な間は imported user symbol を shadow します:

```ebnf
take_decl       ::= "take" example_list ";" ;
example_list    ::= example { "," example } ;
example         ::= term_expression | identifier "=" term_expression ;
deffunc_decl    ::= "deffunc" identifier "(" [ typed_params ] ")"
                    "->" type_expression "equals" term_expression ";" ;
defpred_decl    ::= "defpred" identifier "(" [ typed_params ] ")"
                    "means" formula ";" ;
typed_params    ::= typed_param { "," typed_param } ;
typed_param     ::= identifier ( "being" | "be" ) type_expression ;
```

Unnamed witness form `take term_expression;` は local name を導入しません。
同様に、`reconsider identifier as type_expression;` は既存の local name の
type view を変更し、新しい local name は導入しません。

## 4.5 定量化された変数

普遍的 (`for`) または存在的 (`ex`) 量指定子によってバインドされた変数。

**構文 (EBNF)**:

```ebnf
quantified_segment ::= identifier_list [ ( "being" | "be" ) type_expression ] ;

```

* **注**: `being T` を省略した場合は、`reserve` タイプが使用されます。予約が存在しない場合、それは構文エラーです (コンテキストで型なしが許可されていない場合、たとえば、集合論)。

**例**:

```mizar
for x being Real holds x^2 >= 0;
ex n being Nat st n > 0;
```

## 4.6 スコープとシャドウイング

Mizar はブロック構造で **字句スコープ** を採用しています。

### 4.6.1 スコープレベル

1. **module レベル**: top-level の `reserve` ディレクティブは、directive の位置以後の file全体に適用されます (シャドウイングされていない限り)。nested block の内部では宣言できません。
2. **定理/定義ブロック**: `let` 変数と local constants は、ブロックの `end` (または `thesis` 完了) まで有効です。
3. **ローカル ブロック (`now ... end`, proof `case` / `suppose`, algorithm `do`)**: 内部で宣言された変数はブロックに対してローカルです。
4. **Statement Level**: quantifier binders (`for`, `ex`) と `given` binders は、それを導入した statement/formula range に限定されます。

### 4.6.2 シャドーイングルール

* **許可**: 内部スコープの変数は、外部スコープの変数と同じ名前を持つことができます。内側の宣言は外側の宣言をシャドウします。
* **禁止**: **same** スコープ内で同じ変数名を再宣言すると、構文エラーになります。

```mizar
let x be Real;
now
  let x be Nat;  :: Valid shadowing. Inside this block, x is Nat.
end;
:: Here x is Real again.

let x be Integer; :: Error: Redeclaration in same scope.

```

## 4.7 論理エンコーディングの詳細

このセクションでは、変数が ATP 入力 (TPTP 形式など) にどのようにマップされるかについて説明します。

### 4.7.1 型ガード (相対化)

一階論理は通常、型指定されていないため、Mizar 型は **Type Guard Predicates** としてエンコードされます。

* **出典**: `let x be Element of G;`
* **エンコーディング**：
* 変数 `x` は汎用変数として扱われます。
* 仮説 `is_Element_of(x, G)` が Antecedent に追加されます。



### 4.7.2 述語としての属性

形容詞（属性）は接続詞に平坦化されます。

* **出典**: `let x be positive Real;`
* **エンコーディング**：
  * 仮説: `x is Real ∧ x is positive`

### 4.7.3 定数とスコーレム化

* **グローバル定数** (引数 0 の `func`): アリティ 0 の関数シンボルとしてエンコードされます。
* **ローカル定数** (`set x = t`): ローカル スコープで等価公理 `x = t` としてエンコードされます。

## 4.8 完全な構文 (EBNF)

変数関連の構文の概要。

```ebnf
variable_directive ::= reserve_decl
                     | let_decl
                     | set_decl
                     | reconsider_decl ;

(* Reservation *)
reserve_decl       ::= "reserve" reserve_segment ";" ;
reserve_segment    ::= identifier_list "for" type_expression ;

(* Free Variable Declaration *)
let_decl           ::= "let" qualified_vars [ "such" conditions ] [ "by" references ] ";" ;
qualified_vars     ::= explicit_qualified_vars [ "," implicit_qualified_vars ]
                     | implicit_qualified_vars ;
explicit_qualified_vars ::= qualified_segment { "," qualified_segment } ;
qualified_segment  ::= identifier_list ( "being" | "be" ) type_expression ;
implicit_qualified_vars ::= identifier_list ;

(* Local Constant Declaration *)
set_decl           ::= "set" equating_list ";" ;
equating_list      ::= equating { "," equating } ;
equating           ::= identifier "=" term_expression ;
reconsider_decl    ::= "reconsider" type_change_list "as" type_expression
                       [ simple_justification ] ";" ;
type_change_list   ::= reconsider_item { "," reconsider_item } ;
reconsider_item    ::= identifier
                     | identifier "=" term_expression ;

(* Quantifier Binding (used in formulas) *)
quantified_segment ::= identifier_list [ ( "being" | "be" ) type_expression ] ;

identifier_list    ::= identifier { "," identifier } ;
```
