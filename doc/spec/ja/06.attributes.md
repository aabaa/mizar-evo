## 6.1 定義と目的

属性は、追加の論理プロパティを指定して型を絞り込むために使用される述語 (おそらくパラメーター化されたもの) です。これらは次の目的に役立ちます。

1. **サブ型の詳細化**: 論理条件に基づいて既存の型のサブ型を定義します。
2. **プロパティ アノテーション**: module式で再利用可能な方法でプロパティを型に添付します。
3. **型推論**: 自動型導出のためのクラスタリング メカニズムに参加します。

## 6.2 構文

**構文 (EBNF)**:

```ebnf
attr_def       ::= "attr" label ":" subject "is" attr_pattern "means" formula ";" ;
label          ::= identifier ;
subject        ::= identifier ;
attr_pattern   ::= [ param_prefix ] attribute_name ;
param_prefix   ::= parameter "-"
                 | "(" parameter_list ")" "-" ;
parameter_list ::= parameter { "," parameter } ;
parameter      ::= identifier | numeral ;
attribute_name ::= identifier ;
```

* `label`: この定義を参照するための必須の識別子
* `subject`: テストされる変数 (例: `T`、`n`、`V`)
* `param_prefix`: パラメータ化された属性のオプションのプレフィックス (例: `n-`、`(m,n)-`)

**例 (単純)**:

```mizar
definition
  let i be Integer;
  attr EvenDef: i is even means 2 divides i;
end;
```

**例 (パラメータ化)**:

```mizar
definition
  let n be Nat;
  let V be VectorSpace;
  attr NDimDef: V is n-dimensional means
    ex B being Basis of V st card B = n;
end;
```

**例 (複数のパラメータ)**:

```mizar
definition
  let m, n be Nat;
  let A be Matrix;
  attr MNSizeDef: A is (m,n)-size means
    rows A = m & cols A = n;
end;
```

## 6.3 使用法

### 6.3.1 型の認定

属性は変数宣言内の型を絞り込みます。

```mizar
let G be associative unital Magma;
let n be positive even Integer;
let V be n-dimensional VectorSpace;  :: Parameterized attribute
let A be (n,n)-size Matrix;          :: Multiple parameters
```

### 6.3.2 プロパティのテスト

属性は式でテストできます。

```mizar
assume M is commutative;
if n is odd then ...;
```

### 6.3.3 否定と組み合わせ

`non` キーワードは属性を無効にします。

```mizar
let S be non empty finite set;
```

### 6.3.4 型修飾された属性

同じ属性名が複数の構造に存在する場合に曖昧さを解消するために使用されます。

```mizar
R is AddLoopStr.associative;
R is MulLoopStr.commutative;
```

あるいは、`qua` 演算子を使用します。

```mizar
(R qua AddLoopStr) is associative;
(R qua MulLoopStr) is commutative;
```

## 6.4 クラスタリング（概要）

クラスタリングにより、属性の自動伝播が可能になります。

|クラスターの種類 |目的 |
| --- | --- |
| **実存的** |属性を持つ型指定された値の存在を証明します。
| **条件付き** |属性間の含意を宣言します。
| **関手** |関手の結果に属性を割り当てます。

**例 (存在)**:

```mizar
registration
  cluster empty finite set;
  existence
  proof ... end;
end;
```

**例 (条件付き)**:

```mizar
registration
  cluster empty -> finite set;
  coherence
  proof ... end;
end;
```

> [！注記]
> 詳細なクラスター構文と意味論は、**第 17 章 (クラスターと登録)** で定義されています。

## 6.5 属性の継承

親構造に対して定義された属性は、派生構造に適用されます。

```mizar
definition
  inherit LoopStr extends Magma;
end;
```

`LoopStr` は、`Magma` に定義されたすべての属性を継承します。

## 6.6 曖昧さ回避

複数の継承パスで同じ属性名を定義する場合は、修飾構文を使用します。

```mizar
R is AddLoopStr.associative;
R is MulLoopStr.associative;
```

または、明示的に明確な属性を定義します。

```mizar
definition
  let R be DoubleLoopStr;
  attr AddAssocDef: R is add_associative means R is AddLoopStr.associative;
end;
```

## 6.7 再定義

属性を別の定義に再定義するには:

* 元の定義
```mizar
definition
  let n be Integer;
  attr integer_even: n is even means
  ex k be Integer st n = 2*k;
end;
```

* 再定義
```mizar
definition
  let n be Nat;
  redefine attr nat_even: n is even means ex k be Nat st n = 2*k;
  coherence with integer_even
  proof
    :: for Nat, divisibility and multiplication-based definitions coincide
    ...;
  end;
end;
```

**要件**：
* 元の定義との論理的等価性
* 再定義を正当化する一貫性の証明
* (オプション) 特定の親定義を参照するためのラベル

## 6.8 論理エンコーディングの詳細

このセクションでは、属性が一階論理にどのようにマップされるかについて説明します。

### 6.8.1 述語としての属性

型 `T` に適用される各属性 `A` は、述語としてエンコードされます。

|Mizar source | FOL エンコーディング |
| --- | --- |
| `x is even` | `is_even(x)` |
| `x is positive` | `is_positive(x)` |
| `V is n-dimensional` | `is_n_dimensional(V, n)` |

### 6.8.2 否定符号化

`non` キーワードは否定された述語を生成します。

|Mizar source | FOL エンコーディング |
| --- | --- |
| `x is non empty` | `¬is_empty(x)` |
| `V is non n-dimensional` | `¬is_n_dimensional(V, n)` |

### 6.8.3 接続詞としての属性チェーン

複数の属性は接続詞としてエンコードされます。

|Mizar source | FOL エンコーディング |
| --- | --- |
| `x is positive even Integer` | `is_Integer(x) ∧ is_positive(x) ∧ is_even(x)` |

### 6.8.4 公理としてのクラスタリング

クラスター登録により FOL 公理が生成されます。

**条件付きクラスター**:
```
cluster empty -> finite set
⇔  forall x (is_set(x) & is_empty(x) implies is_finite(x))
```

## 6.9 完全な構文 (EBNF)

属性関連の構文の概要。

```ebnf
(* Attribute Definition *)
attr_def       ::= "attr" label ":" subject "is" attr_pattern "means" formula ";" ;
label          ::= identifier ;
subject        ::= identifier ;
attr_pattern   ::= [ param_prefix ] attribute_name ;
param_prefix   ::= parameter "-"
                 | "(" parameter_list ")" "-" ;
parameter_list ::= parameter { "," parameter } ;
parameter      ::= identifier | numeral ;
attribute_name ::= identifier ;

(* Attribute Redefinition *)
redefine_attr  ::= "redefine" "attr" label ":" subject "is" attr_pattern "means" formula ";"
                   "coherence" [ "with" label ] ";"
                   [ proof_block ] ;

(* Attribute Usage in Types *)
attribute_chain ::= { [ "non" ] attribute_ref } ;
attribute_ref   ::= [ param_prefix ] [ struct_name "." ] attribute_name [ "(" argument_list ")" ];
```
