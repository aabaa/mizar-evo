## 9.1 概要と構文

述語はプロパティと関係を表します。各述語には、名前、型付きパラメータ、および論理定義があります。述語は常にブール値 (true または false) を生成します。

**構文 (EBNF)**:

```ebnf
pred_def         ::= "pred" label ":" pred_pattern "means" formula ";" ;
label            ::= identifier ;
pred_pattern     ::= [ loci ] predicate_symbol [ loci ] ;
loci             ::= locus | "(" locus_list ")" ;
locus_list       ::= locus { "," locus } ;
locus            ::= identifier ;
predicate_symbol ::= identifier | symbolic_pred ;
symbolic_pred    ::= symbol_char+ ;
```

**例**：

```mizar
definition
  let X, Y be set;
  pred SubsetDef: X c= Y means
    for x being object st x in X holds x in Y;
end;
```

述語には次のものがあります。

* **内蔵**: 例: `x in X`、`x = y`、`x <> y`
* **ユーザー定義**: `pred` および `means` で宣言
* **overload**: パラメーターの型によって曖昧さが解消されます。

> [！注記]
> 証明 (`defpred`) 内の **インライン述語定義** については、[第15章 (文), §15.2.4](./15.statements.md#1524-inline-predicate-definition-defpred) を参照してください。

## 9.2 述語と属性

述語と属性は両方とも論理アサーションですが、異なる目的を果たします。

|側面 |述語 (`pred`) |属性 (`attr`) |
| --- | --- | --- |
| **目的** |用語間の一般的な関係を表現する |プロパティを追加して型を絞り込む |
| **アリティ** |任意 (単項、二項、n 項) |単項 (単一の主題に適用) |
| **構文** | `x c= y`、`a divides b` | `x is even`、`G is associative` |
| **タイプ システムの役割** |数式と証明で使用 |型式とクラスタリングで使用される |
| **クラスタリング** |クラスタリング可能ではありません |クラスター登録に参加します |

**述語を使用する場合**:
- 複数のオブジェクト間の関係を表現する: `x c= y`
- 数学的関係の定義: `a divides b`
- 証明における一般的な論理的主張

**属性を使用する場合**:
- 精錬タイプ：`let n be positive even Integer`
- クラスターによる型推論の有効化
- サブタイプによって継承する必要があるプロパティ

**比較例**:

```mizar
:: As an attribute (for type refinement)
definition
  let n be Integer;
  attr EvenAttr: n is even means 2 divides n;
end;

:: As a predicate (for general use)
definition
  let a, b be Integer;
  pred DividesDef: a divides b means
    ex k be Integer st b = a * k;
end;

:: Attributes participate in type expressions
let n be positive even Integer;

:: Predicates are used in formulas
assume 2 divides n;
```

> [！注記]
> 詳細な属性構文と意味論は [第6章 (属性)](./06.attributes.md) で定義されています。

## 9.3 定義スタイル: 記号とフレーズ

Mizar は、次の 2 つのスタイルの述語パターンをサポートしています。

### 9.3.1 記号述語

数学的表記には記号演算子を使用します。

```mizar
definition
  let x, y be Real;
  pred LessOrEqualDef: x <= y means
    x < y or x = y;
end;
```

### 9.3.2 フレーズ述部

読みやすくするために単語ベースのパターンを使用します。

```mizar
definition
  let n be Nat;
  pred EvenDef: n is_even means ex k be Nat st n = 2 * k;
end;

definition
  let G be Group, a, b be Element of G;
  pred CommuteDef: a, b are_commute means
    G.binop(a, b) = G.binop(b, a);
end;
```

### 9.3.3 複数引数の述語

述語は、シンボルの片側または両側で複数の引数を取ることができます。

```mizar
definition
  let X, Y, Z be set;
  pred PartitionDef: X, Y partition Z means
    X \/ Y = Z & X /\ Y = {};
end;
```

> [！注記]
> `pred_pattern` では述語記号が 1 つしか許可されないため、`a < b < c` (記号が 2 回出現する) のようなパターンは**サポートされません**。代わりに、定義式で接続詞を使用してください: `pred BetweenDef: (a, b, c) are_between means a < b & b < c;`

## 9.4 依存パラメータと型付きパラメータ

パラメータの型は、以前のパラメータに依存する場合があります。

```mizar
definition
  let G be Group;
  let H be Subgroup of G;
  let a, b be Element of G;
  pred CongruenceDef: a, b are_congruent_mod H means
    a * b" in H;
end;
```

属性を使用して型を制限することもできます。

```mizar
definition
  let G be non empty associative Magma;
  let a, b be Element of G;
  pred CommuteDef: a, b are_commute means
    G.binop(a, b) = G.binop(b, a);
end;
```

`assume` によるドメイン制限:

```mizar
definition
  let n, m be Integer;
  assume m <> 0;
  pred DivisibleDef: m divides n means
    ex k be Integer st n = m * k;
end;
```

## 9.5 正確性の条件

関手とは異なり、`means` で定義された述語はブール条件を直接指定するため、存在証明や一意性証明を**必要としません**。

ただし、次の条件が適用されます。

|状態 |説明 |
| --- | --- |
| **整形式** |定義式は構文的に有効である必要があります。
| **型の正確性** |式内のすべての項は型が正しい必要があります。
| **定義** | `assume` が使用される場合、述語は想定されるドメインの外では未定義です。

> [！注記]
> 述語は単純に true または false を評価するため、関手のような一意性の要件はありません。

### 9.5.1 述語のプロパティ

述語は、検証器が自動推論に使用する代数プロパティを宣言できます。各性質には証明義務が必要です。

**構文 (EBNF):**

```ebnf
pred_property ::= ( "symmetry" | "asymmetry" | "connectedness"
                 | "reflexivity" | "irreflexivity" ) justification ";" ;
```

**財産とFOLの義務:**

型 T の引数を持つバイナリ述語 `P(a, b)` の場合:

|プロパティ | FOLの義務 |検証器の効果 |
|---|---|---|
| `symmetry` | `∀a,b:T. P(a,b) → P(b,a)` | | と一致するときに引数を自動交換します。
| `asymmetry` | `∀a,b:T. P(a,b) → ¬P(b,a)` |引数が交換されたときに否定を導出します。
| `connectedness` | `∀a,b:T. a ≠ b → (P(a,b) ∨ P(b,a))` |個別の要素の論理和を導出する |
| `reflexivity` | `∀a:T. P(a,a)` | `P(x,x)` 目標を自動放電 |
| `irreflexivity` | `∀a:T. ¬P(a,a)` | `P(x,x)` から矛盾を自動導出 |

**例：**

```mizar
definition
  let x, y be Real;
  pred LessThanDef: x < y means ...;
  asymmetry
  proof
    let a, b be Real;
    assume a < b;
    thus not b < a by mml.number.real.Lt_asymmetric;
  end;
  irreflexivity
  proof
    let a be Real;
    thus not a < a by mml.number.real.Lt_irreflexive;
  end;
  connectedness
  proof
    let a, b be Real;
    assume a <> b;
    thus a < b or b < a by mml.number.real.Lt_connected;
  end;
end;

definition
  let X, Y be set;
  pred SubsetDef: X c= Y means
    for x being object st x in X holds x in Y;
  reflexivity
  proof
    let A be set;
    thus A c= A;
  end;
end;
```

> [！注記]
> プロパティは、定義式および正当性条件の後に、`definition ... end;` ブロック内に配置されます。

## 9.6 述語の再定義

既存の述語は、より具体的なタイプに合わせて再定義できます。これにより、明確さが向上し、冗長な条件が削除されます。

**構文**：

```ebnf
redefine_pred  ::= "redefine" "pred" pred_pattern "means" formula ";"
                   "coherence" [ "with" label ] proof_or_refs ";" ;
proof_or_refs  ::= proof_block | "by" references ";" ;
```

**例**：

```mizar
definition
  let x be object, f be Function;
  pred fixpoint_general: x is_a_fixpoint_of f means
    x in dom f & x = f.x;
end;

definition
  let A be non empty set, a be Element of A, f be Function of A, A;
  redefine pred a is_a_fixpoint_of f means
    a = f.a;
  coherence with fixpoint_general
  proof
    :: dom f = A, so membership is implied
    ...;
  end;
end;
```

再定義要件:

* `redefine pred`を使用する
* ラベルを使用してオリジナルを参照します (例: `fixpoint_general`)
* 新しいパラメータ タイプでの論理等価性を示す `coherence` 証明を提供します。

## 9.7 表記スタイル

サポートされている表記法:

|表記法 |パターン |例 |
| --- | --- | --- |
| **中置** | `x symbol y` | `x c= y`、`a divides b` |
| **プレフィックス** | `symbol x` | `is_prime n` |
| **後置** | `x symbol` | `n is_even` |
| **複数中置** | `x, y symbol z` | `a, b partition c` |

**象徴的な例**:

```mizar
pred SubsetDef: X c= Y means ...;       :: infix symbolic
pred ElementDef: x in X means ...;      :: infix symbolic
pred EqualsDef: x = y means ...;        :: infix symbolic
pred LessThanDef: x < y means ...;      :: infix symbolic
```

**フレーズの例**:

```mizar
pred DividesDef: m divides n means ...;      :: infix phrase
pred PrimeDef: is_prime n means ...;         :: prefix phrase
pred CommuteDef: a, b are_commute means ...; :: multi-infix phrase
```

> [！注記]
> 関手とは異なり、**述語には演算子の優先順位や結合性**がありません。述語アプリケーションはアトミックな式として解析され、独自の固定優先順位を持つ論理接続子 (`&`、`or`、`implies` など) を使用して結合されます。論理演算子の優先順位については、[第14章 (量化と論理結合子)](./14.formulas.md) を参照してください。

## 9.8 シンボルの解決とインポート

ユーザー定義の述語は、定義moduleをインポートすることで利用可能になります。名前間の競合は次のように解決されます。

* **レクスト中の最長一致ルール**
* **必要に応じて修飾名**:

```mizar
import algebra.group as grp;
let G be set;
assume grp.is_subgroup(H, G);
```

あいまいな述語記号は、修飾名を使用して解決することもできます。

```mizar
let R be DoubleLoopStr;
let a, b be Element of R.carrier;
assume a, b algebra.additive.are_commute;  :: addition commutativity
assume a, b algebra.multiplicative.are_commute;  :: multiplication commutativity
```

> [！注記]
> `qua` 演算子を使用した属性の曖昧さの解消については、「[第6章 (属性), §6.6](./06.attributes.md#66-disambiguation)」を参照してください。

> [！注記]
> 述語記号は、実際に使用される場合にのみ明確でなければなりません。述語の**同義語**と**反意語**については、[第 11 章 (シンボル管理)、§11.1](./11.symbol_management.md#111-synonyms-and-antonyms)を参照してください。

## 9.9 論理エンコーディングの詳細

このセクションでは、述語が一階論理にどのようにマップされるかについて説明します。

### 9.9.1 FOL述語としての述語

各ユーザー定義の述語は、FOL 述語シンボルとしてエンコードされます。

|Mizar source | FOL エンコーディング |
| --- | --- |
| `x c= y` | `subset(x, y)` |
| `m divides n` | `divides(m, n)` |
| `a, b are_commute` | `are_commute(a, b)` |

### 9.9.2 組み込み述語

組み込み述語には標準の FOL エンコーディングが含まれています。

|Mizar source | FOL エンコーディング |説明 |
| --- | --- | --- |
| `x in X` | `x ∈ X` |メンバーシップを設定する |
| `x = y` | `x = y` |等価性 (標準公理を使用) |
| `x <> y` | `x ≠ y` (つまり、`¬(x = y)`) |不平等 |

これらの組み込み述語はプリミティブであり、`means` 定義がありません。

### 9.9.3 二重条件としての述語定義

述語定義は、普遍的に定量化された二条件公理を生成します。

```
pred X c= Y means for x st x in X holds x in Y
⇒  forall X, Y (subset(X, Y) ↔ forall x (x ∈ X → x ∈ Y))
```

より一般的には:

```
pred P(x1, ..., xn) means φ
⇒  forall x1, ..., xn (P(x1, ..., xn) ↔ φ)
```

### 9.9.4 ガードとしての型付きパラメータ

パラメータの型制約は先行詞としてエンコードされます。

|Mizar source | FOL エンコーディング |
| --- | --- |
| `let x, y be Real; pred x < y means ...` | `forall x, y (is_Real(x) ∧ is_Real(y) → (less(x, y) ↔ ...))` |
| `let G be Group; let a be Element of G; pred ...` | `forall G, a (is_Group(G) ∧ is_Element(a, G) → ...)` |

### 9.9.5 ドメイン制限 (`assume`)

述語で `assume` を使用する場合、条件は追加の先行詞になります。

```
let n, m be Integer;
assume m <> 0;
pred m divides n means ...

⇒  forall m, n (is_Integer(m) ∧ is_Integer(n) ∧ m ≠ 0 → (divides(m, n) ↔ ...))
```

### 9.9.6 否定エンコーディング

述語の否定は論理否定としてエンコードされます。

|Mizar source | FOL エンコーディング |
| --- | --- |
| `x does not divide y` | `¬divides(x, y)` |
| `a, b do not are_commute` | `¬are_commute(a, b)` |

### 9.9.7 エンコーディングの再定義

再定義により、制限された型の下で等価公理が生成されます。

```
redefine pred a is_a_fixpoint_of f means a = f.a;
⇒  forall A, a, f (
     ¬is_empty(A) ∧ is_set(A) ∧ is_Element(a, A) ∧ is_Function_of(f, A, A)
     → (fixpoint(a, f) ↔ a = f(a))
   )
```

## 9.10 完全な構文 (EBNF)

述語関連の構文の概要。

```ebnf
(* Predicate Definition *)
pred_def         ::= "pred" label ":" pred_pattern "means" formula ";" ;
label            ::= identifier ;
pred_pattern     ::= [ loci ] predicate_symbol [ loci ] ;
loci             ::= locus | "(" locus_list ")" ;
locus_list       ::= locus { "," locus } ;
locus            ::= identifier ;
predicate_symbol ::= identifier | symbolic_pred ;
symbolic_pred    ::= symbol_char+ ;

(* Predicate Redefinition *)
redefine_pred    ::= "redefine" "pred" pred_pattern "means" formula ";"
                     "coherence" [ "with" label ] proof_or_refs ";" ;
proof_or_refs    ::= proof_block | "by" references ";" ;

(* Predicate Properties *)
pred_property    ::= ( "symmetry" | "asymmetry" | "connectedness"
                   | "reflexivity" | "irreflexivity" ) justification ";" ;

(* Predicate Application - see Chapter 14 *)
predicate_application ::= [ term_list ] [ negation ] predicate_symbol [ term_list ] ;
negation         ::= "does" "not" | "do" "not" ;
term_list        ::= term_expression { "," term_expression } ;

(* Built-in Predicates *)
builtin_pred     ::= "in" | "=" | "<>" ;
```
