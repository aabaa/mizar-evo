## 1.1 Mizar 再設計の背景

Mizar の再設計は、その方向性を定義する**3 つの中核的な設計原則**、すなわち**可読性 (readability)**、**AI 親和性 (AI-readiness)**、**スケーラビリティ (scalability)** によって何よりもまず導かれています。これら三本の柱は、本仕様におけるすべての設計判断を貫く指針です。

- **可読性**: 形式的なテキストは、現場の数学者が用いる自然言語にできる限り近い形で読めるべきです。安定的で均一、かつ検索しやすい構文は、証明を人間にとって読みやすく保ち、非形式的な推論とその形式的対応物との間の認知的距離を最小化します。
- **AI 親和性**: 本システムは、自動推論tool および大規模言語モデル (LLM) と協働するよう、基礎から設計されています。標準化されたインターフェイス (モデル コンテキスト protocol など) と、最小限で機械検証可能な信頼 kernel により、AI が大規模に証明を提案する一方で、論理的健全性は独立して保証されます。とりわけ、人間に資するのと同じ可読性が、Mizar のテキストを LLM の学習・プロンプトのための良質な素材にします。
- **スケーラビリティ**: module システム、namespace、template、より豊かな型システム、パッケージ管理、増分検証といった最新のソフトウェア エンジニアリングの実践により、保守性や再現性を犠牲にすることなく、library とそのコミュニティが成長できます。

これら三本の柱は、数学の形式化と対話型定理証明 (ITP) の進化する状況における、以下のいくつかの具体的な要素を通じて実現されます。

1. **ソフトウェア プラクティスの最新化**:
    50 年にわたる伝統を持つ Mizar は、時代遅れのソフトウェア慣行によって制約を受けてきました。この再設計では、template、namespace、module システム、オーバーロード解決などの最新のソフトウェア パラダイムが導入され、形式化された証明開発の効率と表現力が向上します。

2. **一階論理ベースのシステムの活性化**:
    定理証明の傾向は高階論理と型理論に移行していますが、私たちは一階論理ベースのシステムを進歩させることに大きな価値があると信じています。これは以下によってサポートされています。
   a) 一階論理内の自動定理証明 (ATP) における最近の大きな進歩。
   b) 数学者の間で共通の論理枠組みとして、一階論理に基づく集合論の普及。
   c) 従来の数学的思考と形式的検証の間のギャップを埋めるシステムの必要性。
   d) Mizar のソフトタイピング規律による集合論的基盤を維持することで、基本的な表現の選択とlibrary アーキテクチャの間の過剰な結合を回避したいという要望。
   e) 明示的な属性、モード、および登録を備えた一階の集合論的コアを優先することで、形式的な開発における組み合わせの増加を制御する必要性。

3. **機能の拡張**:
    アルゴリズムの記述と検証を導入することで、Mizar の適用可能性を純粋な数学を超えて、ソフトウェア検証、暗号protocol分析、量子コンピューティングなどの分野に広げることを目指しています。

4. **AI および大規模言語モデルとの相乗効果**:
    大規模言語モデル (LLM) と自動推​​論の急速な進歩により、私たちは相乗的なアプローチの可能性を認識しています。新しい Mizar は、[Vampire](https://vprover.github.io/) や [E](https://wwwlehre.dhbw-stuttgart.de/~sschulz/E/E.html) などの最先端の ATP を堅牢な推論エンジンとして統合するとともに、**モデル コンテキスト protocol (MCP)** などの標準を採用して AI エージェントとのシームレスな対話を促進します。このアーキテクチャにより、Mizar は、**最小限の信頼された kernel を通じて論理的健全性を厳密に維持しながら、高レベルの証明の検索と提案に LLM を活用することができます。** この設計は、AI の直観的な力と数学的証明の厳格な確実性を組み合わせています。

5. **数学的直観を維持する**:
    Mizar は、数学者によって数学者のために設計された言語を備えた唯一の ITP として際立っています。このユニークなアプローチは、50 年にわたって形式化コミュニティからの支持を集めており、直感的な数学的思考と表現との整合性を実証しています。

6. **構文上の柔軟性よりも規律ある表現力**:
    最新のシステムの中には拡張可能な構文やメタプログラミング機能を好むものもありますが、Mizar はより規律あるアプローチを採用しています。安定した均一な構文を強制することにより、システムは人間の可読性、長期的な保守性、および従来の数学的解説との整合性を促進します。この設計の選択は、認知的オーバーヘッドを最小限に抑え、ユーザーが数学的推論の本質により完全に集中できるようにすることを目的としています。

7. **SAT ベースの小型kernelによる基本的な信頼性**:
    絶対的な信頼性に対する需要の高まりに応えて、新しい Mizar は de Bruijn 基準に準拠した「スモール kernel」アーキテクチャを採用しています。このシステムは、モノリシック検証機能に依存する代わりに、証明探索と証明チェックを切り離します。一階推論の妥当性は、特定の置換インスタンスを介して命題論理に還元することによって検証されます。これにより、信頼された kernel を、インスタンス化を実行し、SAT ソルバーを使用して充足不能性 (UNSAT) をチェックする単純なコンポーネントに最小化することができ、論理的健全性が高レベルの自動化の複雑さから独立していることが保証されます。

Mizar のこの再設計は、既存のtoolの単なる更新ではなく、一階論理ベースの定理証明を形式数学と自動推論の最前線に位置づけるための戦略的な動きです。これは、真に数学者の言語を話すシステムであるという中心的な哲学を維持しながら、伝統的な数学的論理の長所を最新のソフトウェア エンジニアリングの実践および最先端の自動推論技術と組み合わせることを目的としています。何よりも、これらの要素はいずれも、再設計を支える三本の柱——可読性、AI 親和性、スケーラビリティ——に奉仕するものです。

## 1.2 この文書の目的

この文書は、最新の機能と改善点を含む、再設計された Mizar 言語の公式仕様として機能します。この仕様の主な目的は次のとおりです。

1. Mizar 言語の包括的かつ信頼できるリファレンスを提供し、その構文、意味論、およびコア機能を詳しく説明します。
2. 初心者と経験豊富なユーザーの両方が、形式的な数学的証明のために Mizar システムを理解し効果的に利用するためのガイドとして機能します。
3. Mizar 言語の最近の機能強化と変更を文書化し、ユーザーが最新の機能とベスト プラクティスを確実に認識できるようにします。
4. Mizar ベースのtool、library、教育教材の開発の標準を確立する。
5. 明確なインターフェイス仕様を提供することで、Mizar と他の数学ソフトウェアおよび証明アシスタントとの統合を容易にする。
6. Mizar の新機能が、一階論理および集合論に基づく数学における従来の強みとどのように統合されるかを説明します。

この仕様は、次のようなさまざまな読者を対象としています。

- 自分の研究を形式化しようとしている数学者
- 形式的手法と定理の証明に興味のあるコンピューター科学者
- 形式論理と数学的推論を教える教育者
- Mizar 用のtoolを作成または Mizar と統合するソフトウェア開発者
- 数学と自動推論の基礎を研究する研究者
- ITP との統合に関心のある ATP の開発者

このドキュメントは Mizar 言語を網羅的にカバーすることを目的としていますが、チュートリアルやユーザー マニュアルとして意図されたものではありません。読者は、Mizar を効果的に使用するための実践的なガイダンスとして、チュートリアル、ユーザー ガイド、サンプル リポジトリなどの補足資料を参照することをお勧めします。

Mizar 言語はより強力で効率的な形式化機能を目指して進化し続けるため、この仕様は新機能、改善、ベスト プラクティスを反映するために定期的に更新されます。ユーザーは、伝統的な数学と新しい応用分野の両方における Mizar の拡張機能に関する最新情報を確実に入手するために、このドキュメントの最新バージョンを参照することをお勧めします。

## 1.3 Mizar の簡単な歴史

最古の ITP の 1 つである Mizar には、1970 年代初頭にまで遡る豊かな歴史があります。その開発と進化は、重要なマイルストーンと成果によって特徴づけられてきました。主なイベントには次のようなものがあります。

- 1973: アンジェイ・トリビュレツのリーダーシップの下、Mizar プロジェクトが開始。
- 1975: Mizar-PC の最初のバージョン。
- 1982: Mizar-MSE の最初のバージョン。
- 1989: 形式化された数学の包括的なリポジトリである [Mizar Mathematical Library (MML)](https://mizar.uwb.edu.pl/library/) の開始。また、先駆的な機械検証された数学出版物である [Journal of Formalized Mathematics](https://fm.mizar.org/) の発売。
- 2001: ジョルダン曲線定理 [^ART2005] [^ART2007] の形式化。
- 1995-2003: 「連続格子大要」[^BAN2002] の形式化。

2026 年 1 月の時点で、Mizar プロジェクトは Freek Wiedijk の「[100 の定理を形式化する](https://www.cs.ru.nl/F.Wiedijk/100/)」リストにある 71 の定理の形式化を完了しました。より詳細な歴史と形式化の成果については、Matuszewski および Rudnicki (2005)[^ROM2005]、Bancerek et al. を参照してください。 (2015)[^BAN2015] および Bancerek et al。 (2018)[^BAN2018]。

Mizar 言語の論理構造は、Jaśkowski によって提案された古典的な 一階論理自然演繹システムの変形に基づいています。 Tarski-Grothendieck 集合理論は、アクセス不可能な基数を含む ZFC 集合理論の拡張であり、MML の基礎として適用されます。現在の Mizar システムの詳細な仕様については、Grabowski et al.(2010) [^GRA2010] を参照してください。

Mizar は、その進化を通じて、自然の数学言語と推論を厳密に反映するという中核となる哲学を維持してきました。このアプローチは、複雑な数学的証明や理論を形式化するために MML 開発者の間で人気があります。このシステムの長寿命と一貫した開発により、基本的な集合論から高度な代数構造まで幅広い分野をカバーする、形式化された数学の最大のlibrary の 1 つが生まれました。

Mizar の影響は直接の応用を超えて広がり、他の証明アシスタントの側面に影響を与え、形式化された数学の分野に大きく貢献しました。ジョルダン曲線定理の形式化と格子理論に関する広範な研究は、複雑な数学的概念を処理するMizar の能力を実証しています。さらに、Wiedijk の「100 の定理」プロジェクトへの貢献は、定理証明器の広範な分野における Mizar の役割を浮き彫りにしています。

この再設計に着手するにあたり、私たちはこの豊かなレガシーを基礎にして、形式的検証と自動推論の変化する状況に適応しながら、Mizar の強みを維持することを目指しています。

## 1.4 新機能の概要

Mizar の再設計では、いくつかの重要な強化と新機能を導入します。本概要では、これらを [§1.1](#11-background-for-redesigning-mizar) で導入した三本の設計の柱——**可読性**、**AI 親和性**、**スケーラビリティ**——の下にまとめて示します。

### 可読性

1. **数学に忠実な安定構文**:
    - 拡張可能な構文やメタプログラミングよりも規律ある表現力を意図的に優先し、伝統的な数学的記法を反映する均一な構文。
    - 認知的オーバーヘッドを最小化し、ユーザーが数学的推論の本質に集中できるようにします。

2. **source エンコーディングと構文ポリシー**:
    - **UTF-8 サポート**: source file は UTF-8 で完全にエンコードされており、コメントや注釈で Unicode 文字を使用できるため、ユーザーは母国語でドキュメントや説明を作成できます。
    - **ASCII ベースのコア構文**: コア論理構文は ASCII ベースのままとし、正規の表現と library 全体での効率的な検索性を保証します。これにより表記の断片化を防ぎ、Mizar コードの「検索しやすい」性質を維持します。

3. **自動ドキュメント生成**:
    - コードを知識として認識し、Mizar source file と注釈から直接包括的な HTML ドキュメントを生成することで、形式的な証明をアクセス可能で理解しやすい状態に保ちます。

4. **改善されたエラー処理と診断**:
    - より詳しいエラー メッセージと警告。
    - 複雑な証明と定義のためのデバッグ機能の強化。

### AI 親和性

5. **置換ベースの検証kernel**:
    - **証明書ベースの検証**: 推論ステップごとに、システムは特定の変数置換 (インスタンス) で構成される証明証明書を生成します。
    - **SAT への還元**: 検証は、否定された式にこれらの置換を適用し、その充足不能性 (UNSAT) を確認することによって実行されます。
    - **最小限の TCB**: 信頼された kernel はインスタンス化メカニズムと基本的な SAT ソルバーに厳密に制限され、信頼された計算基盤が大幅に削減されることで、AI が提案する証明を安全に受け入れられます。

6. **高度な自動化**:
    - 検証済みの `algorithm` 定義と `by computation` 証明ステップを通じて計算的な自動化を提供します（[§20. algorithm と verification](./20.algorithm_and_verification.md) を参照）。一方で、証明スクリプト自体は宣言的な自然演繹のままです。
    - 外部定理証明器 (Vampire、E など) との統合により証明機能が強化されます。
    - 証明の開発を支援する機械学習技術の組み込み。

7. **包括的なアノテーション システム**:
    - ドキュメントとメタ情報の改善に加え、自動推論のための構造化された機械可読な文脈としても機能する多用途の注釈システム。
    - 自動化された推論をガイドするための推論エンジンのヒントのサポート。
    - 検証に使用される ATP バージョンを記録する機能。
    - 形式的な証明を Journal of Formalized Mathematics の記事形式に組み立てるためのメタデータ。
    - library の再編成中に定理の起源を追跡することにより、既存の Mizar libraryとの互換性情報を維持するためのアノテーション。

8. **インタラクティブな開発アーキテクチャ**:
    - **増分解析と検証**: 従来のバッチ処理アプローチとは異なり、新しいアーキテクチャは増分検証をサポートしています。これにより、文書の部分的なチェックが可能になり、ユーザーが入力したときにエラーと証明の有効性に関する即時フィードバックが可能になります。大規模な開発全体の再検証を回避することで、増分検証はスケーラビリティにも寄与します。
    - **エラー耐性のある解析**: parserは回復力があるように設計されており、構文エラーが存在する場合でも部分的な構文ツリーを構築できるため、編集中の継続的なtoolのサポート (オートコンプリートなど) が容易になります。

9. **標準化されたtool インターフェイス (LSP および MCP)**:
    - **言語サーバー protocol (LSP) サポート**: LSP の組み込みサポートにより、構文の強調表示、コード補完、すぐに使える定義への移動などの機能を備えた Mizar を最新の IDE (VS Code、Emacs、Neovim など) に簡単に統合できます。
    - **モデル コンテキスト protocol (MCP) の統合**: 大規模言語モデル (LLM) との相乗効果を促進するために、Mizar はモデル コンテキスト protocolを実装しています。これにより、library の構造、定理検索、検証ステータスが AI エージェントの標準化されたコンテキストとして公開され、LLM が Mizar システムと確実に対話できるようになります。

### スケーラビリティ

10. **拡張module システムとnamespace**:
    - コード構成を改善するための、より堅牢なmodule システムの導入。
    - namespaceの実装により、名前の競合を防ぎ、コードの再利用性が向上します。

11. **template**:
    - template システムによる汎用プログラミングのサポート。
    - より柔軟で再利用可能な数学的構造と定理を作成できるようになります。

12. **改良された型システム**:
    - 強化された型推論機能。
    - より表現力豊かな型階層と自動変換。
    - オーバーロードされたエンティティの解決ルールが改善されました。

13. **アルゴリズムの検証とプログラムの生成**:
    - アルゴリズム仕様と形式的検証のための新しい構文と意味論により、Mizar の適用範囲を純粋数学を超えて拡大します。
    - 古典的なアルゴリズムを検証するための標準 Hoare Logic の実装。
    - 暗号protocolには確率論的 Hoare 論理、量子アルゴリズムには量子 Hoare 論理を採用。
    - 事前条件、事後条件、ループ不変条件、および終了証明のサポート。
    - アルゴリズムの正確性を証明し、プログラムを生成するための機能が強化されました。

14. **統合されたパッケージ管理とビルドシステム**:
    - 依存関係、バージョン管理、再現可能なビルドを処理するための専用のパッケージ マネージャーの導入。これにより、モノリシックなリポジトリからmodule理論の分散型エコシステムへの移行が容易になります。

15. **最新の標準library**:
    - より広範囲の数学的概念と構造をカバーするための標準library の拡張と再編成。

16. **互換性レイヤー**:
    - Mizar の古いバージョンからの移行を容易にする互換性レイヤーの組み込み。
