## 5.1 概念と目的

構造体は、名前付きフィールドとプロパティを持つ複合型を定義するために使用される基数型です。これらは次の 3 つの目的を果たします。

1. **データのカプセル化**: 論理的に関連するコンポーネントをグループ化する
2. **型形成**: 構造定義による新しい型の作成
3. **数学モデリング**: 代数的および解析的構造の表現

## 5.2 構文

**構文 (EBNF)**:

```ebnf
struct_def    ::= "struct" struct_name [ type_params ] "where"
                  struct_member { struct_member } "end" ";" ;
struct_member ::= field_decl | property_decl ;
field_decl    ::= "field" identifier "->" type_expression ";" ;
property_decl ::= "property" identifier "->" type_expression ";" ;
type_params   ::= ( "of" | "over" ) parameter_list ;
parameter_list ::= identifier { "," identifier } ;
```

* `field`: 組み込みコンポーネント (保存されたデータ) を表します。
* `property`: 一意に決定された派生値を表します。それらの定義条件は、モード宣言を介して外部で確立されます (第 7 章を​​参照)

**例**：

```mizar
definition
  struct UnitalMagmaStr where
    field carrier -> set;
    field binop -> BinOp of carrier;
    property unit -> Element of carrier;
  end;
end;
```

## 5.3 継承

構造は、`inherit` を介して他の構造から継承できます。継承する場合、フィールドとプロパティをより特定の型に特化 (絞り込む) できます。

### 5.3.1 構文 (EBNF)

```ebnf
inherit_def     ::= "inherit" struct_name "extends" parent_type
                    [ "where" inherit_member { inherit_member }
                      [ coherence_block ]
                      "end" ] ";" ;

parent_type     ::= struct_name | "set" ;

inherit_member  ::= field_redef | property_redef ;

(* name may differ from parent (renaming); type is optional if unchanged from parent *)
field_redef     ::= "field" identifier [ "->" type_expression ] "from" (identifier | "it") ";" ;
property_redef  ::= "property" identifier [ "->" type_expression ] "from" identifier ";" ;

coherence_block ::= "coherence" justification ";" ;
```

* **短縮形式** (`inherit X extends Y;`): `Y` のすべてのフィールドとプロパティは、同じ型の名前によって自動的にマップされます。 `where` ブロックは必要ありません。
* **明示的な形式** (`inherit X extends Y where ... end;`): フィールドまたはプロパティが **名前変更** (子の名前が親の名前と異なる)、**タイプ特化** (絞り込み)、または **一貫性** 証明が必要な場合に必要です。
* すべての基本フィールド/プロパティはいずれかの形式でカバーされる必要があります。

> [！注記]
> 各 `inherit` ステートメントは、**1 つの親** を正確に指定します。多重継承の場合は、複数の `inherit` 定義を使用します。これにより、単一ステートメント内でのフィールド/プロパティ名の競合が回避されます。

**例** — 名前の変更:

```mizar
definition
  inherit Derived extends Base where
    field derived_field from base_field;      :: renamed
    property derived_prop from base_prop;     :: renamed
  end;
end;
```

**セットの継承** が許可されます:

```mizar
definition
  inherit OneSorted extends set where
    field carrier from it;
  end;
end;
```

### 5.3.2 コヒーレンス (型の特殊化)

**フィールドまたはプロパティ**がより狭い型で継承される場合、値が実際に新しい型に属していることを検証するために `coherence` 証明が必要です。

**例**：
`PolyStr` は、`Nat` を返すプロパティ `degree` を定義します。 `NonConstPolyStr` はそれを `non zero Nat` に特化し、一貫性の証明を必要とします。

```mizar
definition
  struct PolyStr where
    field coeffs -> FinSequence of Int;
    property degree -> Nat;
  end;
end;

definition
  struct NonConstPolyStr where
    property degree -> non zero Nat;
  end;

  inherit NonConstPolyStr extends PolyStr where
    field coeffs from coeffs;

    :: Property narrowing: proof required
    property degree -> non zero Nat from degree;
    coherence
    proof
      :: non zero Nat ⊆ Nat (upcast is valid)
      thus for x be non zero Nat holds x is Nat;
    end;
  end;
end;
```

## 5.4 多重継承とダイヤモンド継承

構造名が異なる場合、多重継承が許可されます。ダイヤモンド継承がサポートされています。複数の親が **同じ名前と型**のフィールドを共有する場合、検証器は `from` チェーンをトレースすることによって自動的に一貫性を確認します。明示的な証明は必要ありません。継承された型が異なる場合、(安全なアップキャストを保証するために) 型の包含を検証するために `coherence` 証明が必要です。

**例** — 同じタイプ、証明は必要ありません:

```mizar
definition
  struct ZeroStr where
    field carrier -> set;
    property zero -> Element of carrier;
  end;

  struct OneStr where
    field carrier -> set;
    property one -> Element of carrier;
  end;

  struct ZeroOneStr where
    field carrier -> set;
    property zero -> Element of carrier;
    property one -> Element of carrier;
  end;

  :: shorthand: all fields/properties map by name with identical types
  inherit ZeroOneStr extends ZeroStr;
  inherit ZeroOneStr extends OneStr;
end;
```

**例** — さまざまなタイプ、`coherence` が必要です:

リングは、加法構造 (`AddStr`、`carrier -> non empty set`、ゼロ要素は空でないことを保証します) と乗法構造 (`MulStr`、`carrier -> set`) を組み合わせています。 `RingStr` が両方から継承する場合、キャリア タイプが異なるため、コヒーレンス証明が必要になります。

```mizar
definition
  struct AddStr where
    field carrier -> non empty set;
    field add -> BinOp of carrier;
    property zero -> Element of carrier;
  end;

  struct MulStr where
    field carrier -> set;
    field mul -> BinOp of carrier;
  end;

  struct RingStr where
    field carrier -> non empty set;
    field add -> BinOp of carrier;
    field mul -> BinOp of carrier;
    property zero -> Element of carrier;
  end;

  :: shorthand: all fields/properties (carrier, add, zero) map by name
  inherit RingStr extends AddStr;

  :: RingStr.carrier : non empty set, MulStr.carrier : set
  :: non empty set ⊆ set: coherence required
  inherit RingStr extends MulStr where
    field carrier from carrier;
    field mul from mul;
    coherence
    proof
      :: non empty set is a subtype of set (upcast is valid)
      thus for x being non empty set holds x is set;
    end;
  end;
end;
```

## 5.5 コンストラクター

構造体は、デフォルトのコンストラクターまたは変換コンストラクターを使用してインスタンス化できます。

### 5.5.1 デフォルトのコンストラクター

**構文 (EBNF)**:

```ebnf
constructor_call ::= struct_name "(" [ named_arg { "," named_arg } ] ")" ;
named_arg        ::= identifier ":" term_expression ;
```

名前付き引数を任意の順序で指定します。

```mizar
let x = OneStr(carrier: A, one: b);
```

### 5.5.2 コンストラクターの変換

互換性のある構造体型間の暗黙的な型強制を許可します。

```mizar
let G be Group;
reconsider x = G as OneStr;  :: view G as OneStr (carrier and one come from G)
```

## 5.6 依存構造

構造は、`over` または `of` を使用してパラメータ化できます。

**例**：

```mizar
definition
  let R be Ring;
  struct Polynomial over R where
    field coeffs -> FinSequence of R.carrier;
    property degree -> Nat;
  end;
end;
```

* 複数のパラメータ: `struct Name over T, U`
* 使用法: `let p be Polynomial over REAL;`

## 5.7 フィールドアクセス (セレクター)

フィールドとプロパティにはドット表記を使用してアクセスします。

**構文 (EBNF)**:

```ebnf
field_access ::= term_expression "." identifier ;
```

**例**:

```mizar
let G be Group;
let c = G.carrier;     :: Access field
let u = G.unit;        :: Access property
let f = G.binop;       :: Access field

:: Chained access (nested structures)
let S be Module over R;
let base_carrier = S.ring.carrier;  :: Access parent structure's field
```

## 5.8 論理エンコーディングの詳細

このセクションでは、構造が一階論理にどのようにマップされるかについて説明します。エンコードは、構造コンポーネントをグローバル述語と関数に平坦化することに依存します。

### 5.8.1 述語としての構造

各構造型 `S` は、単項述語 `is_S(x)` としてエンコードされます。

|Mizar source | FOL エンコーディング |
| --- | --- |
| `x is UnitalMagmaStr` | `is_UnitalMagmaStr(x)` |

### 5.8.2 関数およびタイプガードとしてのフィールド

各 `field` は、グローバル関数シンボルとしてエンコードされます。さらに、**型ガード公理** により、有効な構造体に適用された場合にセレクターが正しい型の値を返すことが保証されます。

* **記号**: `carrier(x)`
* **公理**：
```
forall x (is_UnitalMagmaStr(x) implies is_set(carrier(x)))
```

> [！注記]
> プロパティは、同じタイプ ガード パターンを使用して関数シンボルとしてもエンコードされます。

### 5.8.3 含意としての継承

継承関係は、フィールド セレクターの等価性とともに普遍的な意味 (包含) としてエンコードされます。

```
inherit Derived extends Base where field d_field from b_field
⇔  forall x (is_Derived(x) implies is_Base(x))
   forall x (is_Derived(x) implies d_field(x) = b_field(x))
```

### 5.8.4 コンストラクターの公理

デフォルトのコンストラクターは、**Projection** (フィールド値) と **Typing** (結果の型) の 2 種類の公理を生成する集計関数としてエンコードされます。

`let x = OneStr(carrier: A, one: b)` が用語 `Agg_OneStr(A, b)` としてエンコードされた場合:

> アグリゲータの引数は、構造体定義のフィールド宣言の順序に従います。

1. **射影公理**:
```
carrier(Agg_OneStr(A, b)) = A
one(Agg_OneStr(A, b)) = b
```

2. **入力公理**:
```
is_OneStr(Agg_OneStr(A, b))
```

### 5.8.5 拡張性

構造の「多数のソートされた集合」の性質を捉えるために、拡張性公理は、同一性がフィールドによってのみ決定されることを指示します。

フィールド `f1`、...、`fn` を持つ構造体 `S` の場合:

```
forall x, y (
  (is_S(x) & is_S(y))
  implies
  ((f1(x) = f1(y) & ... & fn(x) = fn(y)) iff x = y)
)
```

## 5.9 完全な構文 (EBNF)

構造関連の構文の概要。

```ebnf
(* Structure Definition *)
struct_def     ::= "struct" struct_name [ type_params ] "where"
                   struct_member { struct_member } "end" ";" ;
struct_member  ::= field_decl | property_decl ;
field_decl     ::= "field" identifier "->" type_expression ";" ;
property_decl  ::= "property" identifier "->" type_expression ";" ;
type_params    ::= ( "of" | "over" ) parameter_list ;

(* Inheritance *)
inherit_def    ::= "inherit" struct_name "extends" parent_type
                   [ "where" inherit_member { inherit_member }
                     [ coherence_block ]
                     "end" ] ";" ;
parent_type    ::= struct_name | "set" ;

inherit_member ::= field_redef | property_redef ;
field_redef    ::= "field" identifier [ "->" type_expression ] "from" (identifier | "it") ";" ;
property_redef ::= "property" identifier [ "->" type_expression ] "from" identifier ";" ;

(* Coherence for Type Specialization *)
coherence_block ::= "coherence" justification ";" ;

(* Constructor Call *)
constructor_call ::= struct_name "(" [ named_arg { "," named_arg } ] ")" ;
named_arg        ::= identifier ":" term_expression ;

(* Field Access *)
field_access   ::= term_expression "." identifier ;

(* Common *)
struct_name    ::= identifier ;
parameter_list ::= identifier { "," identifier } ;
```
