## 14.1 概要

数式は、論理命題の構文表現です。数学的オブジェクトを示す用語式 ([第13章](./13.term_expression.md)) とは異なり、式は定理、証明、および制御構造内で使用されるブール値のステートメントを示します。

数式は 3 つの層から構成されます。

1. **アトミック式** — 述語の適用、型アサーション、属性アサーション
2. **複合数式** — 論理接続子による組み合わせ (`&`、`or`、`implies`、`iff`、`not`)
3. **定量化された式** — 普遍的 (`for ... holds`) と存在的 (`ex ... st`)

**構文 (EBNF)**:

```ebnf
formula         ::= quantified_formula | iff_formula ;
iff_formula     ::= implies_formula [ "iff" ( implies_formula | quantified_formula ) ] ;
implies_formula ::= or_formula [ "implies" ( implies_formula | quantified_formula ) ] ;
or_formula      ::= and_formula
                    { "or" ( and_formula | quantified_formula )
                    | "or" "..." "or" ( and_formula | quantified_formula ) } ;
and_formula     ::= not_formula
                    { "&" ( not_formula | quantified_formula )
                    | "&" "..." "&" ( not_formula | quantified_formula ) } ;
not_formula     ::= "not" ( not_formula | quantified_formula )
                  | atomic_formula
                  | "(" formula ")"
                  | "contradiction"
                  | "thesis" ;
```

## 14.2 原子式

アトミック式は、述語の適用と `is` assertion によって形成される最も単純な式です。
parser-facing な文法では generic な `is_assertion` を保持し、name / type
resolution が後で type assertion または attribute assertion に分類します。

**構文 (EBNF)**:

```ebnf
atomic_formula ::= predicate_application
                 | inline_predicate_application
                 | is_assertion ;

predicate_application        ::= user_predicate_application
                               | builtin_predicate_application ;
user_predicate_application   ::= predicate_segment { predicate_chain_segment } ;
predicate_segment            ::= [ term_list ] [ negation ] predicate_head [ term_list ] ;
predicate_chain_segment      ::= [ negation ] predicate_head term_list ;
predicate_head               ::= predicate_symbol [ template_args ] ;
builtin_predicate_application ::= term_expression builtin_pred term_expression ;
inline_predicate_application ::= inline_pred_name "(" [ term_list ] ")" ;
is_assertion                 ::= term_expression "is" [ "not" ] is_assertion_body ;
is_assertion_body            ::= type_expression | attribute_test_chain ;

attribute_test_chain ::= [ "non" ] attribute_ref { [ "non" ] attribute_ref } ;
attribute_ref        ::= [ param_prefix ] [ struct_name "." ] attribute_name
                         [ "(" argument_list ")" ] ;
negation         ::= "does" "not" | "do" "not" ;
term_list        ::= term_expression { "," term_expression } ;
argument_list    ::= term_expression { "," term_expression } ;
builtin_pred     ::= "in" | "=" | "<>" ;
```

述語適用では user-defined predicate symbol の連鎖を許します。overload
resolution 後、`a < b < c` のような連鎖は隣接する述語適用の連言として、
概念的には `a < b & b < c` と解釈されます。隣接適用として解決できない
連鎖は、name/type resolution で ill-formed predicate application として
拒否されます。

template predicate application では、`a, b are_congruent[H]` のように
template argument を predicate symbol の直後に置きます。`template_args`
production は theorem reference と template functor application と共有されます。

chain の分割は決定的です。最初の segment 以後では、暗黙の左境界は直前に解決された
segment の最後の項そのものです。その segment に書かれた `term_list` は残りの引数を
与えます。それ以外の再グループ化は試みません。segment level の `does not` / `do not`
修飾子は、chain conjunction を形成する前に、その隣接述語適用だけを否定します。

組み込み述語 (`=`、`<>`、`in`) は単独の atomic formula であり、user predicate
chain には参加しません。`a < b = c` のような混在 chain は、`&`、`or`、または
別の formula connective で完全な formula 同士を結合する形に書き直さない限り
構文エラーです。

### 14.2.1 述語の適用

述語は、[第9章 (述語)](./09.predicates.md) で定義されている記号表記または句表記を使用して 1 つ以上の引数に適用できます。 `does not` / `do not` 修飾子は、構文レベルで述語を否定します。

**例**：

```mizar
:: Symbolic predicates
x in X;
x = y;
x c= Y;

:: Phrase predicates
a divides b;
a, b are_commute;

:: Negated predicate applications
x does not divide y;
x, y do not are_commute;
```

> [！注記]
> 述語定義構文 (`pred`) については、「[第9章 (述語)](./09.predicates.md)」を参照してください。
> 証明内のインライン述語定義 (`defpred`) については、「[第15章 (文), §15.2.4](./15.statements.md#1524-インライン述語定義-defpred)」を参照してください。

### 14.2.2 インライン述語アプリケーション

インライン述語 (証明ブロックで `defpred` で定義) には常に括弧が必要です。

**例**：

```mizar
proof
  defpred IsSmall(n be Nat) means n < 10;

  A1: IsSmall(3);       :: true
  A2: not IsSmall(100); :: true
  ...;
end;
```

### 14.2.3 型アサーション

resolution 後、型アサーションは項が特定の型に属しているかどうかを確認します。
`is` キーワードを使用します。

subject 項は、assert される型または属性 chain に対して静的に適用可能でなければなりません。
型 assertion では、assert される型が subject の推論型から通常の型検査、widening、
または `qua` のような明示 view によって到達可能でなければなりません。属性 assertion では、
subject が属性の宣言 radix と parameter に適用可能でなければなりません。そうでない場合、
その formula は型エラーです。検証器は、属性の guard された domain の外に無制約な
attribute atom を作りません。

**例**：

```mizar
x is Nat;
x is Element of A;
f is Function of A, B;
```

### 14.2.4 属性アサーション

resolution 後、属性アサーションは項に 1 つ以上の属性があるかどうかを確認します。
複数の形容詞は、**結合的に**解釈される属性チェーンを形成します。`non`
修飾子は個々の属性を否定します。`not` はアサーション全体を否定します。

**例**：

```mizar
:: Single attribute
n is even;

:: Multiple attributes (conjunctive)
y is positive even Integer;
:: equivalent to: y is positive & y is even & y is Integer

:: Negated attributes
G is non empty;         :: attribute-level negation: G has the property "non empty"
x is not finite;        :: formula-level negation: equivalent to not (x is finite)
```

> [！重要]
> `non` と `not` には異なる意味論があります。
> - `x is non empty` — `non empty` は単一の形容詞です (属性 `empty` の否定)
> - `x is not finite` — 式 `x is finite` は否定されます (`not (x is finite)` と同等)

## 14.3 論理接続詞

論理接続子は、単純な式を結合して複合式を作成します。

**構文 (EBNF)**:

```ebnf
iff_formula     ::= implies_formula [ "iff" ( implies_formula | quantified_formula ) ] ;
implies_formula ::= or_formula [ "implies" ( implies_formula | quantified_formula ) ] ;
or_formula      ::= and_formula
                    { "or" ( and_formula | quantified_formula )
                    | "or" "..." "or" ( and_formula | quantified_formula ) } ;
and_formula     ::= not_formula
                    { "&" ( not_formula | quantified_formula )
                    | "&" "..." "&" ( not_formula | quantified_formula ) } ;
not_formula     ::= "not" ( not_formula | quantified_formula )
                  | atomic_formula
                  | "(" formula ")"
                  | "contradiction"
                  | "thesis" ;
```

定量化式は二項接続子の右端オペランドとして現れます。これは、量化子の結合力が最も弱く右方向へ延びることを反映します。`P implies for x being T holds Q(x)` は `P implies (for x being T holds Q(x))` として構文解析されます。

### 14.3.1 接続詞 (`&`)

二項結合は論理的な「and」を表します。左連想。

**例**：

```mizar
x is Nat & x > 0;
x > 0 & y > 0 & z > 0;   :: parsed as ((x > 0) & (y > 0)) & (z > 0)
```

> [！注記]
> `&` 演算子は **数式レベル** の論理積です。 **ステートメントレベル**の条件連鎖で使用される `and` キーワードとは異なります (例: `assume that ... and ...`、`such that ... and ...`)。 `and` キーワードについては、「[第15章 (文), §15.3.2](./15.statements.md#1532-集団的仮定-assume-that)」を参照してください。

**繰り返し形式** (`& ... &`): 整数定数の範囲にわたって変化する一連の同様の結合の短縮形。

```mizar
assume m <> 0 & ... & m <> 3;
:: expanded to: m <> 0 & m <> 1 & m <> 2 & m <> 3
```

> [！注記]
> 繰り返し形式 (`& ... &`、`or ... or`) は、左右の formula が alpha-equivalent で、
> ちょうど 1 箇所の閉じた integer literal だけが異なる場合に限って受理されます。
> 2 つの endpoint literal は有限でなければならず、その差は unit-step の閉区間を
> 決定します。展開では、その inclusive range の各整数を代入した formula を生成します。
> 複数の可変位置、symbolic endpoint、非整数 endpoint、または zero-length で曖昧な
> pattern は拒否されます。

### 14.3.2 論理和 (`or`)

二項論理和は論理「または」を表します。左連想。

**例**：

```mizar
x = 0 or x = 1;
x = 0 or y = 0 or z = 0;  :: parsed as ((x = 0) or (y = 0)) or (z = 0)
```

**繰り返しフォーム**:

```mizar
x mod 2 = 0 or ... or x mod 7 = 0;
```

### 14.3.3 影響 (`implies`)

論理的な意味合い (if-then) を表します。慣例により右結合。

**例**：

```mizar
x > 0 implies x^2 > 0;
x > 0 implies y > 0 implies x + y > 0;
:: parsed as x > 0 implies (y > 0 implies x + y > 0)
```

### 14.3.4 二重含意 (`iff`)

論理的等価性 (if および if のみ) を表します。非結合 (連鎖には括弧が必要)。

**例**：

```mizar
x in A iff x is Element of A;
(P iff Q) iff (P implies Q) & (Q implies P);
```

### 14.3.5 否定 (`not`)

単項接頭辞演算子。どの二項接続詞よりも強く結合します。

**例**：

```mizar
not x in A;
not (x > 0 or y < 0);
not x > 0 & y > 0;      :: parsed as (not x > 0) & y > 0
```

> [！注記]
> 構文形式 `x is not finite` も、`not (x is finite)` の短縮形として使用できます。 §14.2.4 を参照してください。

### 14.3.6 特殊定数

|定数 |意味論 | FOL エンコーディング |使い方 |
| --- | --- | --- | --- |
| `thesis` |現在の証明の目標 | (コンテキスト依存) |証明の `thus thesis;` |
| `contradiction` |常に false | `⊥` |矛盾による証明 |

**例**：

```mizar
:: Proof by contradiction
assume not thesis;
...;
thus contradiction;

:: Goal reference
A1: P & Q;
thus thesis by A1;
```

## 14.4 定量化された式

定量化された数式は、論理的な普遍性（`for`）と存在性（`ex`）を表現します。

**構文 (EBNF)**:

```ebnf
quantified_formula  ::= universal_formula | existential_formula ;
universal_formula   ::= "for" quantified_vars [ "st" formula ] ( "holds" formula | quantified_formula ) ;
existential_formula ::= "ex" quantified_vars "st" formula ;

quantified_vars   ::= explicit_vars [ "," implicit_vars ] | implicit_vars ;
explicit_vars     ::= qualified_segment { "," qualified_segment } ;
qualified_segment ::= var_list ( "being" | "be" ) type_expression ;
implicit_vars     ::= var_list ;
var_list          ::= identifier { "," identifier } ;
```

### 14.4.1 ユニバーサル定量化 (`for`)

`for ... holds` 構造は、定量化された変数のすべての値に対して式が成立することを表します。

**例**：

```mizar
:: Simple universal
for x being Nat holds x + 1 > x;

:: With condition (restricted quantification)
for x being Real st x > 0 holds x^2 > 0;

:: Multiple variables
for x, y being Real holds x + y = y + x;

:: Mixed typed and reserved variables
for a being Nat, b holds a + b = b + a;  :: b typed by reserve
```

### 14.4.2 ネストされた定量化

複数の `for` 量指定子は、`holds` を繰り返さずにネストできます。最も内側の `holds` はチェーン全体に適用されます。

**例**：

```mizar
:: Nested quantification
for x being Real st x > 0
  for y being Real st y > x
    holds x + y > 0;

:: Equivalent expanded form
for x being Real st x > 0 holds
  for y being Real st y > x holds x + y > 0;
```

### 14.4.3 存在の定量化 (`ex`)

`ex ... st` 構造は、条件を満たす値の存在をアサートします。

**例**：

```mizar
:: Simple existence
ex n being Nat st n > 100;

:: Multiple variables
ex x, y being Real st x^2 + y^2 = 1;

:: Nested quantifiers with existence
for x being Real st x > 0
  ex y being Real st y^2 = x;
```

### 14.4.4 修飾された変数の構文

数量化された式内の変数は、明示的に型指定することも、暗黙的に型指定することもできます。暗黙的な変数は、以前の `reserve` 宣言に依存します。

|フォーム |意味 |例 |
| --- | --- | --- |
| `x being T` |明示的な型 | `for x being Nat holds ...` |
| `x be T` |明示的な型 (省略表現) | `for x be Nat holds ...` |
| `x` (無資格) | `reserve` からの暗黙的な型 | |

```mizar
:: reserve declares implicit types
reserve n for Nat;
reserve x, y for Real;

:: Implicit typing: n is Nat, x is Real
for n holds n + 1 > n;
for x, y holds x + y = y + x;
```

## 14.5 特殊な数式形式

### 14.5.1 適格な属性テスト

属性が構造階層のコンテキストで定義されている場合、曖昧さを解決するために属性を修飾する必要がある場合があります。

**例**：

```mizar
:: Qualified attribute: specify the structure context
G is Group.associative;

:: Using qua to specify the type context
(G qua Magma) is associative;
```

> [！注記]
> `qua` 演算子の構文と意味論については、「[第13章 (項式), §13.6](./13.term_expression.md#136-型式認定-qua)」を参照してください。検証器によって発行される FOL 有効性条件については、[§13.8.7](./13.term_expression.md#1387-qua-演算子のエンコーディング) を参照してください。

### 14.5.2 平等と不平等

組み込みの等号 (`=`) 述語と不等号 (`<>`) 述語は、特別なステータスを持つアトミック式です。

```mizar
x = y;                 :: equality
x <> y;                :: inequality (syntactic sugar for not x = y)
```

### 14.5.3 メンバーシップの設定

組み込みのメンバーシップ述語 `in` はアトミック式です。

```mizar
x in X;
x in dom f;
[a, b] in [:A, B:];
```

ここで `[a, b]` は第13章の bracket functor term 形式です。`[` は term primary が期待される位置に現れるため、template引数としては解析されません。

### 14.5.4 実際の複合式

複数の数式構成を組み合わせた完全な例:

```mizar
:: Theorem statement: injectivity of a function
for f being Function st f is one-to-one holds
  for x, y being object st x in dom f & y in dom f
    holds f.x = f.y implies x = y;

:: Axiom of extensionality
for X, Y being set holds
  X = Y iff for z being object holds z in X iff z in Y;

:: Existence with nested structure
ex G being strict Group st
  G.carrier = NAT &
  ex e being Element of G st
    for a being Element of G holds G.binop(e, a) = a;
```

## 14.6 優先順位と結合性

数式演算子には固定の優先順位があります。量化子は結合力が最も低くなります。

|優先順位 |オペレーター |結合性 |例 |
| --- | --- | --- | --- |
| 1 (最高) |原子式 | — | `x is Nat`、`a divides b` |
| 2 | `not` |単項 (接頭辞) | `not x > 0` |
| 3 | `&` |左 | `a & b & c` → `(a & b) & c` |
| 4 | `or` |左 | `a or b or c` → `(a or b) or c` |
| 5 | `implies` |右 | `a implies b implies c` → `a implies (b implies c)` |
| 6 | `iff` |非関連 | `a iff b` (チェーンはエラーです) |
| 7 (最低) | `for`、`ex` | — |最も外側のスコープにバインドする |

**解析例**:

```mizar
:: Precedence demonstration
not x > 0 & y > 0;
:: parsed as: (not (x > 0)) & (y > 0)

a or b implies c;
:: parsed as: (a or b) implies c

a & b or c;
:: parsed as: (a & b) or c

for x being Nat holds x > 0 implies x >= 1;
:: parsed as: for x being Nat holds (x > 0 implies x >= 1)

:: Use parentheses for clarity
(x implies y) implies z;
(a or b) & (c or d);
```

> [！ヒント]
> 優先順位が読者にとって明らかでない場合は、括弧を使用してください。

## 14.7 論理エンコーディングの詳細

このセクションでは、数式構成がどのように一階論理 (FOL) にマップされるかについて説明します。

### 14.7.1 原子式

|Mizar source | FOL エンコーディング |
| --- | --- |
| `x in X` | `x ∈ X` |
| `x = y` | `x = y` |
| `x <> y` | `¬(x = y)` |
| `x divides y` | `divides(x, y)` |
| `x does not divide y` | `¬divides(x, y)` |
| `IsSmall(n)` (インライン) | `IsSmall(n)` (置換により拡張) |

### 14.7.2 型と属性のアサーション

|Mizar source | FOL エンコーディング |
| --- | --- |
| `x is Nat` | `is_Nat(x)` |
| `x is Element of A` | `is_Element_of(x, A)` |
| `x is positive even Integer` | `is_Integer(x) ∧ is_positive(x) ∧ is_even(x)` |
| `G is non empty` | `¬is_empty(G)` |
| `G is Group.associative` | `G` が `Group` として適用可能であることを検査した後、`is_Group_associative(G)` |
| `x is not finite` | `¬is_finite(x)` |

> [！注記]
> 形容詞連鎖の `non` は属性述語の否定 (`¬is_empty`) を生成しますが、`x is not finite` の `not` はアサーション全体の式レベルの否定 (`¬is_finite(x)`) を生成します。どちらも論理否定になりますが、`non` はチェーン内の単一の形容詞を対象とするのに対し、`not` は resolution 後の attribute assertion 全体を否定します。

### 14.7.3 論理接続子

|Mizar source | FOL エンコーディング |
| --- | --- |
| `P & Q` | `P ∧ Q` |
| `P or Q` | `P ∨ Q` |
| `P implies Q` | `P → Q` |
| `P iff Q` | `P ↔ Q` |
| `not P` | `¬P` |
| `contradiction` | `⊥` |

### 14.7.4 繰り返しフォーム

`& ... &` および `or ... or` 形式は、有限の論理積/論理和に拡張されます。

```
m <> 0 & ... & m <> 3
⟹  m ≠ 0 ∧ m ≠ 1 ∧ m ≠ 2 ∧ m ≠ 3

x mod 2 = 0 or ... or x mod 5 = 0
⟹  x mod 2 = 0 ∨ x mod 3 = 0 ∨ x mod 4 = 0 ∨ x mod 5 = 0
```

### 14.7.5 定量化された式

|Mizar source | FOL エンコーディング |
| --- | --- |
| `for x being T holds P(x)` | `∀x. (is_T(x) → P(x))` |
| `for x being T st Q(x) holds P(x)` | `∀x. (is_T(x) ∧ Q(x) → P(x))` |
| `ex x being T st P(x)` | `∃x. (is_T(x) ∧ P(x))` |
| `for x being T, y holds P(x, y)` | `∀x, y. (is_T(x) → P(x, y))` (リザーブの y のタイプ) |

**ネストされた数量化エンコーディング**:

```
for x being Real st x > 0
  for y being Real st y > x
    holds x + y > 0

⟹  ∀x. (is_Real(x) ∧ x > 0 →
      ∀y. (is_Real(y) ∧ y > x → x + y > 0))
```

### 14.7.6 特殊定数

|Mizar source | FOL エンコーディング |メモ |
| --- | --- | --- |
| `thesis` | (現在の目標式) |証明アシスタントによって解決されました |
| `contradiction` | `⊥` |ファルサム定数 |

### 14.7.7 複合エンコーディングの例

完全な定理エンコード:

```mizar
for X, Y being set holds
  X c= Y iff for z being object holds z in X implies z in Y;
```

以下にエンコードします:

```
∀X, Y. (is_set(X) ∧ is_set(Y) →
  (subset(X, Y) ↔ ∀z. (is_object(z) → (z ∈ X → z ∈ Y))))
```

## 14.8 完全な構文 (EBNF)

すべての数式構文の概要。

```ebnf
(* Formulas *)
formula              ::= quantified_formula | iff_formula ;

(* Atomic Formulas *)
atomic_formula       ::= predicate_application
                       | inline_predicate_application
                       | is_assertion ;

predicate_application        ::= user_predicate_application
                               | builtin_predicate_application ;
user_predicate_application   ::= predicate_segment { predicate_chain_segment } ;
predicate_segment            ::= [ term_list ] [ negation ] predicate_head [ term_list ] ;
predicate_chain_segment      ::= [ negation ] predicate_head term_list ;
predicate_head               ::= predicate_symbol [ template_args ] ;
builtin_predicate_application ::= term_expression builtin_pred term_expression ;
inline_predicate_application ::= inline_pred_name "(" [ term_list ] ")" ;
is_assertion                 ::= term_expression "is" [ "not" ] is_assertion_body ;
is_assertion_body            ::= type_expression | attribute_test_chain ;

attribute_test_chain         ::= [ "non" ] attribute_ref { [ "non" ] attribute_ref } ;
attribute_ref                ::= [ param_prefix ] [ struct_name "." ] attribute_name
                                 [ "(" argument_list ")" ] ;
negation                     ::= "does" "not" | "do" "not" ;
term_list                    ::= term_expression { "," term_expression } ;
argument_list                ::= term_expression { "," term_expression } ;
builtin_pred                 ::= "in" | "=" | "<>" ;
(* For predicate definitions, see Chapter 9 (Predicates) *)
(* For defpred, see Chapter 15 (Statements) *)

(* Compound Formulas *)
iff_formula          ::= implies_formula
                         [ "iff" ( implies_formula | quantified_formula ) ] ;
implies_formula      ::= or_formula
                         [ "implies" ( implies_formula | quantified_formula ) ] ;
or_formula           ::= and_formula
                         { "or" ( and_formula | quantified_formula )
                         | "or" "..." "or"
                           ( and_formula | quantified_formula ) } ;
and_formula          ::= not_formula
                         { "&" ( not_formula | quantified_formula )
                         | "&" "..." "&"
                           ( not_formula | quantified_formula ) } ;
not_formula          ::= "not" ( not_formula | quantified_formula )
                       | atomic_formula
                       | "(" formula ")"
                       | "contradiction"
                       | "thesis" ;

(* Quantified Formulas *)
quantified_formula   ::= universal_formula | existential_formula ;
universal_formula    ::= "for" quantified_vars [ "st" formula ]
                         ( "holds" formula | quantified_formula ) ;
existential_formula  ::= "ex" quantified_vars "st" formula ;

(* Quantified Variables *)
quantified_vars      ::= explicit_vars [ "," implicit_vars ] | implicit_vars ;
explicit_vars        ::= qualified_segment { "," qualified_segment } ;
qualified_segment    ::= var_list ( "being" | "be" ) type_expression ;
implicit_vars        ::= var_list ;
var_list             ::= identifier { "," identifier } ;
```
